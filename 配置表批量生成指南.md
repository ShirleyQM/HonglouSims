# 配置表批量生成指南

供 AI 批量产出内容后，粘贴进 `js/config.js` 对应字段。  
生成顺序建议：**状态 → 互动后气泡 → 需求气泡 → 情绪传染 → 互动模板 → 冲突剧场 / 记忆浮现 / 关系阈值 / 旁观反应**。

**快速复制：** 每张表有独立 Prompt 文件 → [`prompts/batch-gen/`](prompts/batch-gen/)  
（`01`–`12` 对应下文各节，`00-master-all-in-one.txt` 为整包一次生成）

---

## 通用约束（所有表共用）

| 项 | 规则 |
|----|------|
| 文风 | 红楼梦大观园，古风白话，口语化，不用网络梗 |
| 气泡字数 | 每条 ≤30 字（内心/说话气泡）；对白 `lines` 可略长 |
| 时间单位 | `duration` / `cooldown` / `cooldownGameMin` 均为**游戏分钟** |
| 状态 id | 运行时 camelCase（`awkward`）或 CSV 编号（`S003`），**不要混用同一状态** |
| 占位符 | 见各表说明；未列出的占位符会被原样显示 |
| 输出格式 | **纯 JSON 数组** 或 **CSV 行**，不要 markdown 代码块包裹 |

### 枚举速查

**互动类别 `category`：** `xujiu` 叙旧 · `lundao` 论道 · `tiaoxiao` 调笑 · `weijie` 慰藉 · `chuanqing` 传情 · `zhengchi` 争执

**关系四轴 `axis`：** `affection` 情分 · `trust` 信任 · `friendship` 交情 · `submission` 服从

**需求 `needKey`：** `hunger` · `energy` · `fun` · `hygiene`

**性格 `trait`：** `lazy` · `qinggao` · `qingjie` · `haoke` · `letian` · `duoqing` · `kebo` · `zhongcheng`

**气泡 `style`：** `thought` 内心 · `speech` 说话 · `exclaim` 惊呼

**状态冲突组 `conflictGroup`：** `mood_positive` · `mood_negative` · `social_atmo` · `drunk_state` · `love_reaction`

**技能 `blockedSkills` / `skillReq.skill`：** `move` · `talk` · `poetry` · `serve` · `manage` · `finance`

**effect `system`：** `need` · `relation` · `state` · `memory`

**effect `target`：** `both` · `initiator` · `target` · `a` · `b`

**旁观 `triggerSignal`：** `interaction` · `status_gained` · `action_tag` · `scene_enter` · `scene_enter_unauthorized` · `absent_near_baoyu`

**旁观 `reactionType`：** `send_bubble` · `force_interaction` · `issue_quest` · `add_status` · `boost_action` · `take_over_task`

## 1. 互动模板 `interactionTemplates`（对话面板内容）

**位置：** `config.js` → `interactionTemplates`（键为数字 id，现有约 101–733）  
**作用：** 互动菜单项、兜底台词、LLM 话题标签、效果与软锁收场

### 空白模板

```json
{
  "id": 0,
  "category": "xujiu",
  "name": "",
  "type": "dialogue",
  "relMin": -50,
  "relMax": 100,
  "minScore": 0,
  "duration": 8,
  "cooldown": 120,
  "once": false,
  "llmPrompt": "",
  "lines": [
    "{A}：「」",
    "{B}：「」"
  ],
  "effects": [
    { "target": "both", "system": "need", "key": "fun", "delta": 8 },
    { "target": "both", "system": "relation", "delta": 3 }
  ],
  "axisEffects": [
    { "axis": "affection", "delta": 3 },
    { "axis": "trust", "delta": 1 }
  ],
  "axisReq": {},
  "onLowScore": {
    "effects": [{ "system": "state", "stateId": "awkward", "target": "initiator" }],
    "axisEffects": [{ "axis": "affection", "delta": -2 }]
  },
  "skillReq": null,
  "unlock_conditions": {},
  "contact_type": "",
  "is_risky": false,
  "risky_details": {
    "base_success_rate": 0.6,
    "witness_penalty": 0.12,
    "fail_status": "awkward",
    "success_status": "secretCrush"
  }
}
```

### 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `type` | ✓ | `dialogue` 可走 LLM；`action` 多为动作描述一句 |
| `lines` | ✓ | 至少 2 条；`{A}` 发起方 `{B}` 对象方 |
| `llmPrompt` | 建议 | 与 `prompts/interaction-topics.txt` 中【标题】一致，或英文 tag |
| `minScore` | 可选 | 综合分门槛；缺省走 `interactionSocialConfig.categoryMinScore` |
| `effects` | 建议 | `state` 可带 `prob`；`memory` 用 `text`+`tag` |
| `onLowScore` | 可选 | 情分不足时的收场状态与轴变化 |

### 给 AI 的 Prompt

> 完整可复制版：`prompts/batch-gen/01-interaction-templates.txt`

```
你是《红楼梦》大观园模拟游戏的策划。请批量生成互动模板 JSON 数组。

【任务】
生成 N 条 interactionTemplates，category 限定为：{category}
每条包含：id（从 {startId} 递增）、name、type、lines（2–4 条）、effects、axisEffects。
dialogue 类必须写 llmPrompt，与互动名称或【{topicName}】话题对应。

【角色语境】
主要角色：宝玉、黛玉、宝钗、袭人、晴雯、凤姐、贾母、王夫人、刘姥姥等。
关系可亲可疏，台词符合身份礼法（主仆、辈分、闺阁）。

【数值参考】
- duration 5–15，cooldown 60–240
- fun delta：寒暄 5–8，传情 10–20，争执 5–12
- axisEffects：小 +1~3，中 +4~6，大 +7~10；争执可负
- 传情类成功可 prob:0.3–0.5 附加 stateId: heartFlutter / secretCrush
- 情分不足收场：onLowScore → awkward，affection -1~-3

【输出】
纯 JSON 数组，不要 markdown。每条 id 唯一。
```

## 2. LLM 话题文案 `prompts/interaction-topics.txt`

**位置：** `prompts/interaction-topics.txt`（非 JSON，按节追加）  
**作用：** 每个 `llmPrompt` / 互动名对应一段创作说明，拼进 Ollama 请求

### 空白模板

```
【互动名称】
一句话定位；2–4 句写作要点：语气、禁忌、可写话题、不宜写什么。
```

### 示例

```
【耳语】
极私密场合，声气极轻，只够对方听见；可写试探、叮嘱、秘事，不可大声张扬或粗俗。
```

### 给 AI 的 Prompt

> 完整可复制版：`prompts/batch-gen/02-interaction-topics.txt`

```
为《红楼梦》大观园游戏编写 interaction-topics 文案。

【格式】
每个互动一节，标题用【全角括号】包裹互动中文名，下接 2–4 句指导语。
不要编号，节与节之间空一行。

【待写列表】
{互动名列表，如：耳语、赔罪、冷嘲、护短、求情、禀报、查问、奉承}

【要求】
- 符合清代贵族礼法与闺阁分寸
- 写明「宜写什么」「不宜写什么」
- 每条指导语总共不超过 80 字

【输出】
纯文本，可直接追加到 interaction-topics.txt
```

      "effect": { "type": "relation", "delta": -2 }
    }
  ]
}
```

### 占位符

### 给 AI 的 Prompt

> 完整可复制版：`prompts/batch-gen/04-conflict-theaters.txt`

```
生成大观园游戏「冲突小剧场」JSON 数组。

生成 {count} 个 conflictTheaters，性格对立组合示例：
洁癖 qingjie vs 慵懒 lazy、好客 haoke vs 清高 qinggao、话多 haoke vs 寡言 qinggao。

【规则】
- 每个剧场 3–5 行 lines，speaker 交替 char_a / char_b

## 7. 状态定义 `stateDefs`（含情绪）

**位置：** `config.js` → `stateDefs`；设计稿 CSV：`状态总表_结构化版.csv` → 注入 `js/systems/state-defs.js`

### 空白模板（完整机制版）

```json
{
  "stateId": {
    "name": "",
    "duration": 30,
    "desc": "",
    "category": "情绪类",
    "stackable": true,
    "conflictGroup": "mood_negative",
    "needMods": {
      "fun": { "decay": 1.0, "grow": 1.0 }
    },
    "blockedSkills": [],
    "aiModifiers": {
      "boost": { "chuanqing": 1.0 },
      "cut": { "tiaoxiao": 1.0 }
    },
    "trigger": { "need": "fun", "op": "lt", "value": 18 }
  }
}
```

### trigger 形态

- 需求：`{ "need": "fun", "op": "lt|gt", "value": 18 }`
- 行为：`{ "action": "wine" }`

### aiModifiers 键（互动类）

`xujiu` `lundao` `tiaoxiao` `weijie` `chuanqing` `zhengchi` — 倍率 >1 提升权重，<1 降低

### CSV 设计稿列（批量填表用）

```
状态ID,状态名称,原分类,一级分类,二级分类,三级分类,描述,触发条件,效果类别,作用对象,影响维度,促进动作,促进技能,限制动作,限制技能,机制说明,持续时间,可叠加
```

### 给 AI 的 Prompt（CSV 设计稿）

> 完整可复制版：`prompts/batch-gen/07-state-defs-csv.txt`

```
生成《红楼梦》大观园游戏「状态设计表」CSV 内容（无表头）。

【任务】
生成情绪类状态 S{start}–S{end}，共 {count} 行。

【列顺序】
状态ID,状态名称,原分类,一级分类,二级分类,三级分类,描述,触发条件,效果类别,作用对象,影响维度,促进动作,促进技能,限制动作,限制技能,机制说明,持续时间,可叠加

【规则】
- 原分类填「情绪类」；二级分正向/负向/波动/其他
- 持续时间用中文：30分钟、1.5小时、2小时（程序会换算）
- 机制说明写清需求衰减/社交权重变化，供策划转 needMods
- 正向情绪 conflictGroup 对应 mood_positive，负向对应 mood_negative
- 描述 15–30 字，触发条件具体可Gameplay化

【输出】
CSV 数据行，逗号分隔，不要表头，不要 markdown
```

### 给 AI 的 Prompt（runtime JSON）

> 完整可复制版：`prompts/batch-gen/08-state-defs-json.txt`

```
将情绪/社交状态转为大观园游戏 stateDefs JSON 对象（非数组）。

【输入状态列表】
{名称与简述列表}

【任务】
每个状态一个 key（camelCase 英文 id，如 ganshang、awkward），写全：
name, duration（游戏分钟 10–120）, desc, conflictGroup（情绪必填）,
needMods（至少改 fun 或 energy 一项 decay/grow 倍率 0.3–2.0）,
blockedSkills（可选）, aiModifiers.boost/cut（可选，键用六类互动 id）,
trigger（可选，仅适合自动触发的）

【参考】
愤怒：fun decay×1.3；心动：fun grow×1.3；尴尬：blockedSkills 无，偏 cut chuanqing

【输出】
单个 JSON 对象 { "id1": {...}, "id2": {...} }
```
