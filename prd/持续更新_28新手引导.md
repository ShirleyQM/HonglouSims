# 新手引导 PRD：林黛玉进贾府

> 更新时间：2026-06-26
> 文档定位：定义 v0.1-alpha 的第一个明确玩家目标和新手教程链。
> 关联文档：`项目目标.md`、`持续更新_03任务.md`、`持续更新_17_UI总览.md`、`持续更新_20_主界面与人物面板信息架构.md`、`持续更新_22人物作息.md`、`持续更新_25传令与上下级管理面板.md`。

## 1. 目标定位

新手引导不是独立演出脚本，而是玩家第一次接触真实系统的轻量目标链。

第一版目标：

```text
林黛玉进贾府
```

玩家完成后应理解四件事：

- 怎么移动。
- 怎么与人物社交。
- 怎么通过传令进行管理。
- 怎么观察或安排日常起居。

## 2. 设计原则

- 必须使用真实系统，不做假按钮、假任务、假结算。
- 每一步只教一个动作，不堆解释。
- 玩家可以跳过，但默认推荐跟随。
- 失败要给可理解提示，比如目标太远、人物不在、权限不足。
- 引导目标只面向普通玩家，不开放后台。
- 教程完成后，玩家能继续自由玩，不进入强制剧情轨道。

## 3. 新手流程

### 3.1 第一段：移动

目标：让玩家知道点击地面或目标地点可以移动。

剧情包装：

```text
黛玉初入贾府，先随人入内拜见长辈。
```

玩家动作：

- 选中林黛玉。
- 点击指定地点或目标人物附近。
- 黛玉成功移动到目标区域。

验收：

- 地图上有明确目标提示。
- 行动队列出现移动行动。
- 到达后任务进入下一步。

### 3.2 第二段：社交

目标：让玩家知道点击人物可以打开互动菜单并选择社交互动。

剧情包装：

```text
黛玉拜见贾母，初叙亲情。
```

玩家动作：

- 点击贾母或迎接人物。
- 选择一个低风险社交互动，例如请安、问候、叙礼。
- 互动完成后关系或日志有反馈。

验收：

- 互动菜单不越界。
- 可用互动清楚显示。
- 完成后出现日志/气泡/任务进度反馈。

### 3.3 第三段：管理

目标：让玩家理解传令是管理和下发任务的入口。

剧情包装：

```text
黛玉安顿行李，需要请随侍或丫鬟帮忙。
```

玩家动作：

- 打开传令面板。
- 在点名传令中选择一个可传令对象。
- 下发一个轻量任务，例如引路、取物、整理行李、请人通报。

验收：

- 传令面板展示可传令对象。
- 候选卡展示服从、信任或预计接受。
- 发布后生成任务或行动反馈。
- 若当前人物无权限，教程应切换到有权限的发布者或给出明确解释。

### 3.4 第四段：日常

目标：让玩家理解人物日常由起居、需求、家具和 AI 共同驱动。

剧情包装：

```text
初入府中，黛玉需要安顿、梳洗或歇息。
```

玩家动作二选一：

- 打开起居，看黛玉当天日常安排。
- 或点击可用家具，让黛玉完成一次梳洗、歇息、饮茶、用点心等日常活动。

验收：

- 起居面板能打开并展示当天安排。
- 若走家具，家具互动完成后需求或状态有变化。
- 教程结束时显示“黛玉已初步安顿”之类完成反馈。

## 4. UI 入口

推荐第一版使用轻量目标面板：

```text
当前目标：林黛玉进贾府
1. 前往贾母处
2. 与贾母请安
3. 传令安排随侍
4. 安顿日常
```

规则：

- 面板默认显示在任务区域，不另开复杂新 UI。
- 每步完成后打勾，并提示下一步。
- 可折叠，避免挡住画布。
- 玩家跳过后不再强弹，但可以从帮助或任务面板重新打开。

## 5. 数据和系统接入

| 系统 | 接入方式 |
| --- | --- |
| 任务系统 | 新手目标作为一组 `tutorial` / `event` 任务实例。 |
| 行动队列 | 移动、家具、互动都走真实队列。 |
| 社交互动 | 使用现有互动模板和关系效果。 |
| 传令 | 使用统一传令面板，不新开“教程派活”。 |
| 起居 | 读取黛玉当天起居；日常活动可通过家具或起居块完成。 |
| 日志/气泡 | 每步给短反馈，避免大量说明文字。 |
| 存档 | 保存教程进度，刷新后不应重置已完成步骤。 |

## 7. 验收标准

- 新玩家 5-10 分钟内能完成整条目标。
- 四步分别教到移动、社交、管理、日常。
- 中途刷新后教程进度不丢。
- 任何一步失败都能给出下一步建议。
- 教程不依赖后台、不要求懂配置、不要求打开控制台。
- 完成后玩家可以继续自由模拟。
