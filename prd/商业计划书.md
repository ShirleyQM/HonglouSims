# 商业计划书

## 定位
古代版模拟人生，一款以中国古代为背景的“关系与命运”模拟游戏。
玩家将置身于预设或架空的地图，操控历史名人或自创角色，体验有因果、有记忆、有温度的古代人生。
完全由玩家自由书写的沙盒模式，或根据游戏提供预设的故事线在情境下书写自己的故事。
- 背景：中国古代，首期聚焦“红楼梦”架空背景，贵族家庭生活，后续以资料片扩展至皇家、修仙、市井等主题。
- 角色：支持操控历史名人（如宝玉、黛玉）或完全自创角色。角色有性格、习惯、需求、关系、个人成长、职业等系统。
- 玩法边界：核心是生活与关系、大世界模拟、换装和建筑。首期先做自由沙盒模式，先不做故事线。


## 市场机会
- 题材空白：EA《模拟人生》与《inZoi》均聚焦现代生活，古代生活模拟与东方玄幻题材在全球范围内存在明显市场缺口。
- 玩法差异：现有国产古风游戏多侧重战斗（燕云、古剑）或垂直玩法（种田、经营、修仙宗门管理），缺少一款专注“生活与关系”本身的模拟游戏。
- 文化红利：古代背景电视剧长期有市场，《红楼梦》《西游记》等IP无版权壁垒，古典家具、朝代故事素材丰富。
- 兼容性：当前为古代题材，但可以通过“可插拔”的系统或资料片形式，接入现代元素或玩法。



## 基础玩法

首期已落地的基础闭环：人物因需求与作息行动，用家具满足生活，点人互动改关系，上位者可传令差遣，性格让各人决策不同。对应 PRD 01～07。

- **性格（01）**：通用性格（性情/习惯/处世）+ 对偶 + 人物专属性格；统一修正 AI 权重、需求、状态、关系、社交、任务、记忆、礼法意愿等。
- **关系（02）**：好感 / 信任 / 友谊 / 服从四轴；关系面板、阶段与综合标签；主仆服从↔体恤、亲缘孝道↔慈爱；贴身照拂可因第四轴判「合礼」。
- **任务 / 传令（03）**：点名与群体传令、任务面板、仆从职责契约、日课与随侍轮值、跟随靠近推进、质量结算（出色～失败）与忠诚/积怨。
- **家具（04）**：一家具多动作、环形菜单选择；需求/状态/技能结算；书案、饭桌、茶案、琴台、花圃等首批；旁观反应基础版。
- **Utility AI（05）**：需求驱动的效用机 + 起居节律；家具/社交/漫步/归巢/任务等候选；关系牵引与短时意图（追人、安慰、争执等）。
- **基础需求与状态（06）**：饥 / 倦 / 趣 / 洁 / 交游 / 心绪六需求；分档与组合状态；自然衰退与性格修正。
- **社交互动（07）**：宽慰 / 闲谈 / 论艺 / 传情 / 玩笑 / 争执六类普现；关系与礼法软锁（风险/失败，不硬藏）；失败深度链；LLM 对白（失败回退模板）。

仍粗/待补：关系阶段事件、任务深层管理（如二段传话）、家具邀人共做、更 Generative 的社交 AI。

## 创新点和差异性

### 人物层面
- P0 AI npc：有深度的情绪和人物长线：怎么样做出好玩的AI npc，让用户在“掌控欲”和“惊喜感”做到平衡。
- P0 “传令”系统。模拟人生和inzoi目前都缺乏和其他npc的管理，损失了很大一部分乐趣——作为家主，我的家庭家风如何？每日秩序如何？默认的红楼背景，凤姐可以治理经济、发放月钱，贾政为各子弟安排学业日程等。相应也可以诞生，例如皇宫资料片，皇帝管理官员，三省六部升迁路途，后宫嬉闹日常，等等。

### 世界层面
- P0 “秩序”或“社会”系统。模拟的不只是个人，个人要影响到社会。在某个朝代或社会背景下，遵循这个社会的法则，个人也可以改变这个法则。在贾府，有人心、温情，腐败。人心决定传令是否会被遵守，如果让不服众的人来管事，可能人心就日益降低。温情决定人与人相处的和谐程度，抄检大观园后“温情”肯定大幅下降。一直让凤姐理家而无监管，腐败增加，各府到手的月银都变少了，丫鬟婆子吃不饱饭偷偷出去打零工……
- P0 “礼法”“身份”系统。古代社会有阶级有礼法，士农工商等级森严。而为了让现代人更易懂易玩、也摒弃一些封建糟粕，设定不同等阶的身份，并赋予动态关系，例如父对子有慈爱，子有孝道；主对仆有体贴，仆有忠诚。身份所天然带来的限制，会将一些互动判定为逾矩或冒犯，例如仆对主发起传情可能会判定冒犯，被第三人目击则容易形成流言，等等。


### 在线模式
- 可以做。但是要抽象一个线上环境。有点类似元宇宙。玩家可造访好友的府邸，和好友捏的小人互动，参与诗社、宴席等异步社交活动。一样，要突出的是“生活”+轻社交玩法，满足探索和分享欲，而不是纯“游览”。
- P1 故事线。设置几个故事的重要节点，预设好人物关系和故事情节，让玩家前来体验。但复杂度较高，希望能模拟出《底特律》或者《八方旅人》等多线、npc互动的局面，又要符合故事走向，必然无法做到用户纯自由。


## 目标用户
- 用户画像：喜欢古风文化的休闲玩家、模拟经营爱好者、故事驱动型玩家。全性向，偏女性向审美。
- 需求验证：《模拟人生》在国内有大量玩家自制古代服饰和建筑Mod，证明古代生活模拟需求真实存在。
- 竞争格局：燕云、古剑、天刀等游戏的休闲玩法（拍照、家园、穿搭）已培养大量古风模拟用户，但无一款产品将“休闲模拟”作为核心体验。
- 文化资产：中国古典文学、建筑、服饰、节气民俗等均可作为免费素材库，极大降低世界观构建成本与用户教育成本。

### 实现层
- 为了高扩展性，需要一部分自研引擎资源。模拟人生有庞大的mod和建房社群，依赖自研引擎和3d建模软件的高适配性。
- 生活模拟所需的美术资源。模拟人生类型的游戏几乎是美术尤其是动画资源堆出来的。但在AI出现后，尤其是类似燕云中视频生成动画的能力出现后，可复制性大大提升。
- 场景与性能优化问题。切换场景、大地图、多人。inzoi目前好像还可以，参考方案，确保中低配设备可流畅运行。
- 系统耦合度高，新增一个系统或者玩法，都涉及多个系统联调，模拟人生一更新就是很多恶性连锁bug，很难规避，必须前期设计好核心循环的测试方案。当前已经用AI设计一套资源生成和测试机制。
