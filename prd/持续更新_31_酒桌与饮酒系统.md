# 酒桌与饮酒系统 PRD

> 日期：2026-07-09  
> 文档定位：定义酒案/宴饮/独酌等家具互动、酒量技能、饮酒偏好性格、酒后状态、健康影响和 AI 选择边界。  
> 设计目标：酒不是“点一下就醉”的惩罚按钮，而是礼俗、社交、诗兴、失态和长期身体代价的综合系统。

## 0. 当前落地状态

2026-07-09 已先按 Activity Pack 底座落地第一版样板：

- 新增 `CONFIG.activityPacks.drinking`。
- 新增 `jiuliang` 酒量技能。
- 新增 `haoyin` 好饮 / `shenyin` 慎饮 性格对偶。
- `酒案` 移除默认 `extraEffects: drunk`。
- 酒案动作由 Activity Pack 自动挂载：小酌、对酌、行酒令、失意独酌、酩酊大醉。
- `酩酊大醉` 标记 `ai.allowAutonomous:false`，玩家可点，普通自主 AI 不会随机选择。

尚未落地：真实 `AlcoholSystem` 酒精负荷、长期健康结算、心绪条件解释、多人行酒令。

Activity Pack 底座见：[持续更新_30_ActivityPack主题玩法包.md](持续更新_30_ActivityPack主题玩法包.md)。

## 1. 文化口径

古代饮酒不只是摄入酒精，而是宴席礼法、诗文游戏、人情往来和情绪宣泄的场景。

红楼梦里的酒桌尤其适合做成戏剧系统：

- 刘姥姥进大观园时，大吃大喝、行酒令和醉卧怡红院，把阶层差异、嬉闹和失态都放在同一场宴饮里。
- 芦雪庵割腥啖膻、联诗争胜一类场景里，酒食、雪景、诗兴和人物性情互相推高。
- 酒令、猜枚、酒令牌本质是有规则的社交游戏：输了喝、喝了兴致高，也可能酒后失言。
- 因此“酒案”不应默认给 `drunk`，而应先提供小酌、对酌、行酒令等低到中风险动作；严重醉态只来自累计饮酒、情绪触发或玩家主动选择。

## 2. 当前问题

现有 `酒案` 家具模板结束时默认添加 `drunk` 状态，而 `drunk` 会禁用移动。

这导致：

- 玩家点击一次酒案就可能进入“不能移动”的强惩罚。
- AI 选择酒案时缺少程度区分，无法表达“小酌”“宴饮”“借酒浇愁”的差异。
- 酒量、性格、健康、场景和心绪没有进入结算。
- 酒桌没有承担红楼式社交玩法，只像一个负面状态触发器。

## 3. 核心设计

### 3.1 新技能：酒量 `jiuliang`

酒量是全技能系统的一项基础生活技能。

| 字段 | 设计 |
|---|---|
| ID | `jiuliang` |
| 名称 | 酒量 |
| 范围 | 0-100 |
| 定义 | 对酒力的耐受、饮酒节制和酒桌规矩的综合能力。 |
| 成长来源 | 小酌、对酌、行酒令、宴饮、被罚酒、醉后恢复。 |
| 不等于 | 不等于“越爱喝越好”，也不等于健康。 |

成长原则：

- 少量或中量饮酒给正常经验。
- 酩酊大醉也可给少量经验，但健康代价更高，且不应成为最优刷法。
- 酒量越高，同等酒量带来的醉意越低，酒后失言概率越低。
- 酒量太低的人被劝酒、罚酒时更容易进入微醺/酒醉。

建议经验公式：

```text
酒量经验 =
  动作基础经验
× 饮酒剂量
× 性格成长倍率
× 状态效率倍率
× 边际递减
```

边际递减沿用技能系统：高酒量后需要更长期的宴饮经验，而不是几次大醉刷满。

### 3.2 新对偶性格：好饮 <-> 慎饮

新增通用性格对偶，归入“习惯”。

| 性格 | ID | 说明 | 运行影响 |
|---|---|---|---|
| 好饮 | `haoyin` | 爱热闹、爱酒食，酒桌上更容易起兴。 | `wine` / `banquet` 行动权重提高；酒量经验略高；更愿意接受对酌、行酒令；情绪低时更可能选择独酌。 |
| 慎饮 | `shenyin` | 对饮酒克制，重体面和清醒。 | 酒桌强饮权重降低；更倾向茶、饭、谈话；面对高剂量饮酒可拒绝或只小酌；醉酒状态持续略短。 |

注意：

- “好饮”不是“酒鬼”，只是偏好酒桌与酒兴；是否失控还要看酒量、心绪、场景、礼法和行动剂量。
- “慎饮”不是不能喝，而是 AI 不会轻易选择高风险喝法。

### 3.3 酒精负荷

给人物运行时增加一个轻量酒精负荷，不作为基础需求展示。

```js
char.alcohol = {
  load: 0,          // 当前醉意负荷，随时间下降
  dailyDose: 0,     // 当日累计剂量，跨日结算健康
  lastDrinkAt: 0,   // 最近饮酒游戏时间
  lastHeavyDrinkDay: 0
}
```

单次动作产生 `dose`，再由技能和性格修正：

```text
有效醉意 =
  dose
× 性格剂量倍率
× 状态倍率
÷ (1 + 酒量 / 80)
```

`alcohol.load` 随游戏时间缓慢下降；`dailyDose` 到次日清晨进入健康结算。

## 4. 酒后状态分层

饮酒状态使用一个互斥组 `drunk_state`，从轻到重逐层覆盖。

| 醉意负荷 | 状态 | ID 建议 | 极性 | 效果 |
|---:|---|---|---|---|
| 0-0.25 | 清醒 | 无 | neutral | 不展示状态。 |
| 0.25-0.55 | 微醺 | `tipsySocial` | mixed | 谈兴、胆量和娱乐略升；礼法判断略降。 |
| 0.55-0.85 | 醺然 | `wineWarmth` | mixed | 诗兴/口才略升，办事/规矩略降；可触发酒后真言。 |
| 0.85-1.20 | 酒醉 | `drunk` | negative | 移动变慢、互动失误率上升；不应默认完全禁止移动。 |
| >=1.20 | 酩酊 | `deadDrunk` | negative | 可短时禁用移动，需要休息或被人扶回；次日高概率宿醉。 |
| 次日 | 宿醉 | `hangover` | negative | 精力恢复下降、健康扣减、办事效率下降。 |

关键改动：

- 当前 `drunk` 的移动硬锁应下沉到 `deadDrunk`，或只在 `drunk` 极短时间内触发。
- 小酌、对酌默认最多进入 `tipsySocial` / `wineWarmth`，不会一次喝到不能移动。

## 5. 酒案家具动作

酒案从单一默认动作改为多动作菜单。

| 动作 | ID | 剂量 | 条件 | 主要收益 | 风险 |
|---|---|---:|---|---|---|
| 小酌 | `wine_sip` | 0.15 | 无 | 意趣 +，心绪小幅恢复，少量酒量经验 | 低酒量连续多次会微醺 |
| 佐酒 | `wine_with_food` | 0.20 | 附近饭桌/点心或进食时段 | 饥饿/意趣/社交小幅收益 | 累计剂量 |
| 对酌 | `wine_pair` | 0.35 | 目标同场且关系意愿不极低 | 社交、关系、小幅心绪恢复；更容易出对话 | 微醺、失言 |
| 行酒令 | `wine_game` | 0.30-0.60 | 同场 2 人以上，宴饮/热闹更适合 | 社交、口才/文墨/酒量经验；生成旁观反应 | 输者罚酒、失礼 |
| 宴饮周旋 | `wine_banquet` | 0.45 | 宴席、长辈/主家发起或任务需要 | 关系、口才、规矩、办事经验 | 过量、逾矩、健康扣减 |
| 失意独酌 | `wine_sorrow` | 0.55 | 心绪 < 40 或有感伤/受责/失落状态 | 短期心绪回升、诗兴上升 | 情绪回落、健康扣减、易醉 |
| 借酒浇愁 | `wine_drown_sorrow` | 0.85 | 心绪 < 25 或严重负面事件后 | 强烈戏剧表现，可能触发诗/哭/诉苦 | 酒醉、宿醉、关系误伤 |
| 酩酊大醉 | `wine_dead_drunk` | 1.20 | 玩家主动、好饮/放诞/宴饮高潮或极低心绪 | 强戏剧节点 | 禁移动/需照看/健康明显下降 |

AI 默认边界：

- 普通空闲只允许 `小酌`、`佐酒`、低剂量 `对酌`。
- `失意独酌` 需要心绪或状态条件。
- `酩酊大醉` 不进入普通家具候选，只能由玩家主动、宴饮事件、极端心绪或特定性格触发。
- 仆从、晚辈、客人是否能饮酒，要叠加身份礼法和场景权限；但礼法不硬锁菜单，只影响 AI 意愿、旁观评价和后果。

## 6. 健康结算

饮酒对全局健康走“当日累计 + 长期频率”。

| 当日剂量 | 健康影响 |
|---:|---|
| 0-0.4 | 无直接扣减。 |
| 0.4-0.8 | 若心绪明显恢复，可只记录不扣；体弱者小扣。 |
| 0.8-1.5 | 次日健康 -1 到 -3，可能宿醉。 |
| 1.5+ | 立即健康 -2 到 -5，次日宿醉；体弱者额外扣。 |

长期规则：

- 连续 7 日中有 3 日以上 `dailyDose >= 0.8`，触发“纵饮”统计，健康上限或恢复效率小幅下降。
- `haoyin` 只提高选择倾向，不免除健康代价。
- `shenyin` 可降低高剂量行为概率，但如果被任务/剧情强迫饮酒，仍按体质和酒量结算。

## 7. 与关系、礼法、互动的关系

酒桌互动不是单纯家具恢复，而是社交互动的入口。

| 系统 | 接入方式 |
|---|---|
| 关系 | 对酌、行酒令、宴饮可按互动意愿公式影响接受率和结果强度。 |
| 礼法 | 身份/场景决定“合不合礼”，影响旁观评价、声望、秩序，不默认锁菜单。 |
| 叙事气泡 | 微醺、醉后失言、被劝酒、罚酒、酒后哭笑都应有气泡。 |
| 秩序 | 大宴有序、主客尽欢可增秩序；失态、越礼、闹酒可降秩序。 |
| 任务 | 备宴、陪客、劝酒、看顾醉者可作为传令/职业任务。 |

## 9. 后台配置

后台需要给策划可调项：

- 酒案动作表：动作名、剂量、条件、收益、状态、AI 权重、是否允许自主 AI。
- 酒量技能：成长倍率、边际递减、状态效率。
- 饮酒状态：阈值、持续时间、效果、是否阻塞移动。
- 性格对偶：好饮/慎饮的 actionWeights、成长倍率、拒绝阈值。
- 健康参数：当日剂量区间、扣减值、宿醉概率、长期纵饮阈值。

## 10. 实施路径

1. 配置层：新增 `jiuliang` 技能、`haoyin/shenyin` 性格、饮酒状态阈值。
2. 家具层：把 `酒案` 默认 `extraEffects: drunk` 移除，改成多动作 `actions`。
3. 运行层：新增 `AlcoholSystem`，维护 `load/dailyDose`、状态分层和跨日健康结算。
4. AI 层：家具 Candidate Provider 读取 action 剂量、条件、性格和心绪，过滤高风险自主动作。
5. UI 层：酒案菜单展示动作程度、触发条件和风险提示；不要把饮酒动作硬锁掉。
6. 埋点层：接入 `wine:*` 事件，用一周模拟看每人饮酒频率、醉酒次数和健康变化。

## 11. 验收标准

- 普通人物点击 `酒案 -> 小酌` 不会直接酒醉，也不会禁用移动。
- 连续多次小酌会累计微醺/醺然，低酒量人物更容易醉。
- `酒量` 随饮酒动作增长，但酩酊刷经验不是最优。
- 过量饮酒会降低健康，并可能次日宿醉。
- `好饮` 人物更愿意去酒桌，`慎饮` 人物更少主动饮酒或只选择小酌。
- `失意独酌`、`借酒浇愁`、`酩酊大醉` 有明确心绪/状态/玩家主动条件，不会在普通自主 AI 中乱跳。
- 一周模拟日志能看出：谁喝了、喝多少、为什么喝、是否醉、健康和关系如何变化。
