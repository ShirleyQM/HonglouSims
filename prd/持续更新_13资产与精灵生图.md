# 资产与精灵生图 PRD

> 更新时间：2026-06-21
> 来源：`0618_AI精灵生图工作流.md`、当前资产代码。
> 主要代码：`js/assets.js`、`assets/manifest.json`、`scripts/process_portraits.py`、`scripts/sprite_workflow.py`。

## 1. 总目标

资产系统负责把人物头像、HUD 半身立绘、房间背景、家具图和 64x64 精灵稳定加载进游戏。

精灵生图工作流的目标是：

- 保持同一角色在多帧、多方向、多动作中的身份一致。
- 让 AI 只做服饰迁移，不重新设计姿势和比例。
- 产物能直接进入 `assets/manifest.json` 并被运行时加载。

## 3. 精灵规格

| 项 | 当前值 |
|---|---|
| 单帧尺寸 | 64 x 64 |
| 游戏绘制尺寸 | 48 x 48 |
| sheet 尺寸 | 832 x 1344 |
| 行数 | 21 |
| 最大列数 | 13 |
| 脚底锚点 | `footFrac = 0.92` |

关键行映射：

| 动画 | 方向 | sheet row，0 基 | 使用列 |
|---|---|---:|---|
| `useHands` | up / left / down / right | 0-3 | 0-5 |
| `idle/walk/rest` | up / left / down / right | 8-11 | idle 0，walk 1-8 |
| `useWipe` | up / left / down / right | 12-15 | 1-4 |

## 4. 生图流程

```text
选择角色立绘
  -> 写 Style Bible
  -> 选择裸模动作条
  -> 生成单方向 pilot
  -> 检查脚底、比例、主色、发型、服饰识别点
  -> 扩四方向
  -> 扩 useHands / useWipe / rest
  -> 拼 sheet
  -> 更新 manifest
  -> 游戏内验证
```

## 5. Style Bible 必填

| 字段 | 含义 |
|---|---|
| `charId` | 角色 ID。 |
| `bodyRatio` | 头身比，建议 1:2.2 到 1:2.5。 |
| `hairSilhouette` | 发型剪影。 |
| `mainColors` | 外衣、内裙、腰带、头发等主色。 |
| `fixedDetails` | 每帧必须保留的识别点。 |
| `allowedMotion` | 袖摆、裙摆、发梢可随动作轻微变化。 |
| `forbidden` | 禁止换衣、换发色、加道具、加背景、变 Q 版等。 |

## 6. 验收标准

- 透明背景，无文字、水印、场景。
- 每帧严格 64x64，动作条总宽正确。
- 脚底 y 基本一致，不漂浮。
- 人物高度约 42-56 px。
- 四方向服饰、发型、腰带、主色一致。
- 左右方向可以镜像，但不能变成不同衣服。
