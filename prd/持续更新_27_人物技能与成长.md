# 人物技能与成长 PRD

> 日期：2026-07-09  
> 文档定位：定义大观园人物技能、技能提升路径、互动接入方式、状态/性格成长倍率与满级里程碑。  
> 参考方向：借鉴《模拟人生》的“行动即成长、状态影响效率、等级解锁玩法”机制，但服务于红楼生活模拟、秩序管理、传令和任务执行。

## 0. 一页结论

人物技能不做“战斗数值”，而是回答：

1. 这个人能不能把府内事务办好。
2. 这个人适不适合被派去传话、服侍、周旋、管账、处理危机。
3. 玩家让角色过什么日子，角色就自然长成什么样。

当前建议保留秩序原型里的 9 项技能，但重新划清边界：

| 技能 | 最短定义 | 边界 |
|---|---|---|
| 规矩 `ritual` | 懂名分、礼法、旧例 | 判断“合不合礼”，不负责说服别人。 |
| 文墨 `literacy` | 读写、诗文、账册理解 | 判断文字与才学，不等于办事能力。 |
| 察言 `insight` | 常驻读人、看眼色 | 日常识人能力，长期生效。 |
| 办事 `affairs` | 把事情落地 | 执行与调度，不负责阴谋和话术。 |
| 心计 `scheming` | 利益博弈和布局 | 只处理权衡、试探、压制、反制。 |
| 口才 `eloquence` | 把话说成 | 说服、圆场、辩解，不等于看懂对方。 |
| 勤勉 `diligence` | 持续执行和盯梢 | 耐劳与稳定，不等于聪明。 |
| 藏锋 `subtlety` | 低调、不露痕迹 | 暗中做事与避嫌，不等于心计。 |
| 通变 `adaptability` | 危机救场和临时改路 | 只在突发、破例、危机、临时变化时吃权重。 |
| 酒量 `jiuliang` | 饮酒耐受、酒桌节制和醉后自持 | 只影响酒桌/宴饮/醉态风险，不代表健康，也不鼓励酩酊刷经验。 |

与秩序册的关系：

- 全技能系统是主数据源。
- 秩序册只读取全技能中的“管理事务子集”，即上表 9 项。
- 其他技能如行走、交谈、诗词、侍奉、算学、女红、金融、医药、茶道等可以继续存在，作为基础能力或二级专精。
- 酒量是生活/社交技能，不属于秩序册管理事务子集；它服务于酒案、宴饮、行酒令和酒后状态。
- 秩序册不创建另一套技能，也不把所有技能都展示成管事能力。

酒量专项设计见：[持续更新_31_酒桌与饮酒系统.md](持续更新_31_酒桌与饮酒系统.md)。

## 1. 当前运行时调查

### 1.1 已存在的接口

当前主游戏已有统一效果入口：

```js
CharacterEffectSystem.apply({
  type: 'skillXp',
  charId,
  skill,
  delta
}, context)
```

已接入位置：

| 系统 | 当前状态 | 说明 |
|---|---|---|
| 家具/普通行动 | 已接入 | `world.js applyFurnEffects()` 支持 `extraEffects/effects` 里的 `type: 'skillXp'`。 |
| 任务系统 | 已接入 | `js/systems/quest.js applyEffects()` 支持 `rewards/penalties` 里的 `type: 'skillXp'`。 |
| 互动系统 | 未完整接入 | `action-queue.js applyInteractionEffects()` 目前处理 `need/state/memory/relation/axisEffects`，没有处理 `skillXp`。 |

### 1.2 当前技能表不统一

主游戏 `CONFIG.skillDefs` 当前是混合表：旧技能仍在，新 9 项技能也已补入。旧技能用于兼容既有家具、互动、职业、人物初始配置；新 9 项用于秩序、成长和新家具动作。

旧技能：

| 当前旧 ID | 中文 | 说明 |
|---|---|---|
| `move` | 行走 | 基础移动能力。 |
| `talk` | 交谈 | 基础互动能力。 |
| `poetry` | 诗词 | 书案填词。 |
| `serve` | 侍奉 | 茶酒伺候。 |
| `trade` | 算学 | 贸易账算。 |
| `craft` | 女红 | 针黹手艺。 |
| `manage` | 管理 | 理家管事。 |
| `finance` | 金融 | 放贷理财。 |

新 9 项：

```text
ritual / literacy / insight / affairs / scheming / eloquence / diligence / subtlety / adaptability
```

结论：

- `skillXp` 机制已有，但当前更像“等级 + delta”，还不是经验条。
- 新 9 技能已经写入主游戏 `CONFIG.skillDefs`，并已有部分家具动作开始发放新技能经验。
- 人物初始 `skills` 列表、职业条件、部分互动门槛仍主要使用旧技能，需要继续迁移或做兼容映射。
- 互动模板若直接写 `system: 'skillXp'`，当前不会生效，需要补解释器。

## 2. 技能数值与成长模型

### 2.1 数值口径

统一为一套技能数值，不再区分“人物技能等级”和“秩序管理技能分”两套显示：

- **运行时主值**：`0-100` 熟练度，用于任务、秩序、人岗适配、成长倍率、剧情条件和后台精细配置。
- **前台显示**：统一显示 `Lv0-Lv10`，每 10 点熟练度为 1 级。
- **本级进度条**：显示当前等级内的 `0-100%` 进度，例如熟练度 37 = `Lv3`，进度 70%。
- **兼容旧配置**：旧数据中 `skillLevels <= 10` 视为旧等级，运行时展示时自动换算为 `等级 × 10` 熟练度；`>10` 视为新熟练度值直接使用。
- **管理技能**：秩序册里的【人物管理技能】是全技能的子集，不创建第二套数值；同样显示 `Lv0-Lv10`，hover/后台可查看 `0-100` 熟练度。

| 熟练度 | 前台等级 | 段位 | 玩家感知 |
|---|---|---|---|
| 0-9 | Lv0 | 未入门 | 几乎不会，只能尝试。 |
| 10-19 | Lv1 | 生手 | 能做简单动作，常出错。 |
| 20-39 | Lv2-Lv3 | 会做 | 可完成日常，不适合负责。 |
| 40-59 | Lv4-Lv5 | 熟手 | 可独立处理普通事务。 |
| 60-79 | Lv6-Lv7 | 能手 | 可管理他人，稳定可靠。 |
| 80-95 | Lv8-Lv9 | 高手 | 可处理复杂局面，解锁高阶选项。 |
| 96-100 | Lv9-Lv10 | 绝活 | 获得里程碑，进入人物传闻和故事条件。 |

### 2.2 经验公式

参考《模拟人生》，每个相关动作都给经验，但速度受状态、性格、场景、难度影响：

```text
实际经验 =
  动作基础经验
× 时长系数
× 难度系数
× 状态倍率
× 性格倍率
× 导师/环境倍率
× 边际递减
```

边际递减：

| 当前技能 | 经验倍率 |
|---|---|
| 0-20 | x1.4 |
| 21-40 | x1.15 |
| 41-60 | x1.0 |
| 61-80 | x0.75 |
| 81-95 | x0.45 |
| 96-100 | x0.18 |

### 2.3 动作基础经验

| 动作来源 | 经验建议 |
|---|---|
| 普通日常动作 | 0.3-0.8 / 30 分钟 |
| 对口家具动作 | 0.6-1.2 / 30 分钟 |
| 对口互动成功 | 0.5-2.0 / 次 |
| 对口任务完成 | 2-8 / 次 |
| 高风险任务完成 | 6-15 / 次 |
| 故事关键节点 | 10-30 / 次 |
| 失败但复盘 | 成功经验的 20%-50%，仅限察言、藏锋、通变、心计等可从失败学习的技能 |

## 3. 技能提升动作

| 技能 | 低速日常 | 中速任务 | 高速事件 |
|---|---|---|---|
| 规矩 | 请安、观礼、翻旧例、听嬷嬷讲规矩 | 安排座次、接待来客、处理失礼 | 当众裁断冲突、替主子圆礼法场面 |
| 文墨 | 读书、抄书、写诗、看账册 | 评文、联句、写帖子、核账 | 诗社夺魁、查出账目错漏、写关键书信 |
| 察言 | 侍茶、陪坐、旁听、倾听 | 传复杂口信、判断来客亲疏、照料情绪 | 提前看出冲突苗头、拦住失礼事件 |
| 办事 | 洒扫、取物、跑腿、采买小物 | 排班、筹备宴席、调人手 | 限时补齐缺人缺物、救回办砸的差事 |
| 心计 | 听闲话、观察派系、复盘人事 | 安排人事、谈条件、压怨气 | 斗争事件中借力打力、反制栽赃 |
| 口才 | 寒暄、问安、闲谈、说笑 | 劝解、讨赏、催账、替人求情 | 当场圆谎、说服长辈改主意、公开辩解 |
| 勤勉 | 早起当值、按时日课、值守 | 连续完成差事、值夜、盯工 | 抱病完成、通宵赶工、救急补漏 |
| 藏锋 | 少说多做、低调传话、替主办事不邀功 | 暗中调停、私下递话、低调赏罚 | 压下丑闻、处理秘密差事、背锅不露人 |
| 通变 | 无常规训练，只能少量从变化中获得 | 临时改排、绕开阻碍、替补空缺 | 突发事故救场、危机中破例成功 |
| 酒量 | 小酌、佐酒、低剂量对酌 | 行酒令、宴饮陪客、被罚少量酒 | 大宴周旋、醉后自持、照看醉者并复盘 |

通变的设计口径：

- 不作为普通日常成长技能。
- 日常互动不默认提升通变。
- 只有“计划被打断、条件变化、时间不足、突发见证者、身份阻拦、任务临期、风险失败补救”时才判定通变。
- 通变高的人不是每天都聪明，而是“出事时能改路”。

## 6. 互动提升表

互动提升不应覆盖所有互动。只有“这个动作本身具有练习、判断、执行、周旋意义”时才给技能经验。

原则：

1. 娱乐性互动大多不给或只给很少经验，避免玩家刷“闲谈/嬉闹”练满技能。
2. 一次性强事件互动不按普通经验结算，交给任务里程碑或制度事件发大额奖励。
3. 逾矩/高风险互动只有成功且符合场景时给经验；失败默认不给，后续可做“失败复盘”。
4. 通变不挂在普通互动上，只在突发、限时、被撞见、身份阻拦、临场救场时判定。

### 6.1 一级大类口径

| 大类 | 默认是否给经验 | 说明 |
|---|---|
| 宽慰 `weijie` | 部分给 | 倾听、安慰、照拂这类有判断/服务意义的给；单纯陪坐不给或很少。 |
| 闲谈 `xujiu` | 少量给 | 问安、复杂寒暄、打听消息给；普通闲聊不给。 |
| 论艺 `lundao` | 主要给 | 文墨、评判、对弈、联句是核心成长来源。 |
| 传情 `chuanqing` | 精选给 | 试探、私语、赠物等给察言/口才/藏锋；强肢体亲密不作为刷技能来源。 |

### 6.4 家具动作提升

家具动作比互动更适合做稳定训练来源：它可重复、可排队、可被 AI 选择，也天然和场景、身份、日课绑定。当前 `world.js` 已支持家具模板 `extraEffects` 和家具动作 `actions[].effects`，其中 `type: 'skillXp'` 会在动作结算时通过 `CharacterEffectSystem.apply()` 生效。因此家具动作提升可以先做配置，不需要新增解释器。

设计原则：

1. 每个家具动作只给 1-3 项技能，避免“一次动作什么都涨”。
2. 日常动作给低额、稳定经验；任务完成、制度事件和高影响节点给中高额经验。
3. 可刷动作的单次经验要克制，建议 `0.2-0.8`；高阶或长时动作可到 `1.0-1.5`。
4. 家具动作可以带 `skill/skillLevel` 门槛，但门槛用于“能不能做”，经验仍写在 `effects`。
5. 现阶段主游戏是旧技能 + 新 9 技能混合表；新配置优先写新 9 技能，旧技能门槛和职业条件通过映射继续兼容。

当前可直接落地的配置格式：

```js
actions: [
  {
    id: 'check_account_book',
    name: '核对账册',
    text: '逐页核看账册，算盘珠声轻响',
    duration: 5,
    needRestores: [{ need: 'fun', ratePerGameMin: 3 }, { need: 'mood', ratePerGameMin: 1 }],
    effects: [
      { timing: 'end', type: 'skillXp', skill: 'literacy', delta: 0.5 },
      { timing: 'end', type: 'skillXp', skill: 'affairs', delta: 0.4 },
      { timing: 'end', type: 'skillXp', skill: 'scheming', delta: 0.2 },
    ],
    tags: ['desk', 'account', 'skill'],
  }
]
```

当前运行时兼容旧写法 `{ type: 'skillXp', param: 'poetry', delta: 1 }`；迁移到新 9 技能后推荐统一使用 `skill` 字段。

落地状态（2026-07-06）：6.4.1 和 6.4.2 的可配置家具动作已录入 `js/config.js`。主游戏暂时保留旧技能表，同时补入新 9 技能 ID，家具动作经验使用新 9 技能结算；6.4.3 标注的双人/风险/导师类动作仍按后续系统处理，不作为普通家具动作录入。

## 7. 接入方式

### 7.1 当前可直接使用的格式

任务奖励已支持：

```js
rewards: [
  { type: 'skillXp', target: 'assignee', skill: 'literacy', delta: 2 }
]
```

家具/行动已支持：

```js
extraEffects: [
  { timing: 'end', type: 'skillXp', skill: 'literacy', delta: 1 }
]
```

### 7.2 互动需要补的解释器

建议互动模板使用同一套 `effects` 风格：

```js
effects: [
  { target: 'initiator', system: 'skillXp', skill: 'eloquence', delta: 0.5 },
  { target: 'target', system: 'skillXp', skill: 'insight', delta: 0.5 }
]
```

`action-queue.js applyInteractionEffects()` 需要补：

```js
if (ef.system === 'skillXp' && ef.skill) {
  CharacterEffectSystem.apply({
    type: 'skillXp',
    charId: who.id,
    skill: ef.skill,
    delta: ef.delta || 1,
  }, {
    source: `interaction:${tpl.id}`,
    reason: tpl.name,
    otherId: who.id === initiator.id ? target.id : initiator.id,
  });
}
```

低分收场 `runLowScoreInteraction()` 当前只处理 `state` 和 `axisEffects`。若以后想让失败也能提升察言/藏锋/通变，需要额外支持 `onLowScore.effects` 中的 `skillXp`。

### 7.3 技能 ID 迁移

秩序技能已经进入主游戏 `CONFIG.skillDefs`，但旧技能仍被人物初始配置、职业条件、互动门槛和部分家具模板引用。需要建立兼容映射，逐步把“成长口径”统一到新 9 技能：

| 旧 ID | 处理建议 |
|---|---|
| `move` | 保留为基础能力，不进入成长主技能。 |
| `talk` | 可并入口才 `eloquence`，或保留为隐藏基础能力。 |
| `poetry` | 映射到文墨 `literacy`。 |
| `serve` | 拆到办事 `affairs` + 察言 `insight` + 勤勉 `diligence`。 |
| `trade` | 映射到文墨 `literacy` + 办事 `affairs`。 |
| `craft` | 暂保留为专业技能；后续可作为二级专精，不进 9 项基础。 |
| `manage` | 拆到办事 `affairs` + 心计 `scheming` + 规矩 `ritual`。 |
| `finance` | 作为账房专精，底层读取文墨/心计/藏锋。 |

## 9. UI 与反馈

### 9.1 人物面板

人物技能应与族谱/身份信息合并展示，不单独做一张重复大表给玩家长期看。

当前展示规则：

- 前台人物详情：用细条展示人物技能和人物管理技能，默认显示 `Lv0-Lv10`，hover 显示熟练度 `0-100` 与说明。
- 技能完整面板：分成【人物技能】和【人物管理技能】两组；两组都使用同一套 `Lv0-Lv10 + 本级进度` 呈现。
- 秩序面板：事务分配时展示“相关技能匹配度”，读取同一套 `0-100` 熟练度。
- 后台/调试：保留 `0-100` 熟练度，允许精细改数；如导入旧 `0-10` 等级，展示层自动兼容换算。

### 9.3 秩序册技能展示

秩序册不是另一套技能系统。它只在“事务归口 / 管事评估”里展示全技能中的【人物管理技能】子集：

- 数值来源：同人物技能主数据。
- 底层口径：`0-100` 熟练度。
- 展示口径：统一为 `Lv0-Lv10`；需要解释时显示 `0-100` 熟练度。
- 当前子集：规矩、文墨、察言、办事、心计、口才、勤勉、藏锋、通变。

| 事务 | 推荐展示技能 | 示例解释 |
|---|---|---|
| 礼法 | 规矩、口才、察言、文墨 | 懂旧例、能圆场、看得懂长辈脸色。 |
| 功课 | 文墨、规矩、勤勉、口才 | 能督学、能讲明白、能坚持盯进度。 |
| 随侍 | 察言、办事、勤勉、藏锋 | 看眼色、手脚稳、少出风头。 |
| 雅集 | 文墨、口才、规矩、察言 | 有才情，也懂场面分寸。 |
| 账房 | 文墨、办事、心计、藏锋 | 看得懂账，办得下去，也不招人眼红。 |
| 洒扫 | 办事、勤勉、口才、规矩 | 会排班、肯盯工、能压住粗使。 |
| 门禁 | 规矩、察言、口才、藏锋 | 知道谁该拦、谁该报、话怎么传。 |
| 人事 | 心计、察言、口才、藏锋 | 知人、会周旋、能避嫌。 |

非管理技能不固定展示，但可作为专项加成：

- 诗词高：雅集、评文、诗社加成。
- 女红高：针线、衣物、随侍细活加成。
- 医药高：侍药、病中照料加成。
- 茶道高：待客、陪坐、慢社交加成。
- 行走高：只影响抵达和跑腿效率，不参与秩序技能主评分。

## 10. Runtime Gaps

| 缺口 | 影响 | 优先级 |
|---|---|---|
| 主游戏 `CONFIG.skillDefs` 仍是旧技能 + 新 9 技能混合表 | 行动门槛、人物初始技能、职业条件和成长技能口径暂不完全统一 | P0 |
| `CharacterEffectSystem.changeSkillXp()` 当前是等级加法，不是 0-100 经验条 | 难以做 Sims 式成长速度和边际递减 | P0 |
| 互动 `effects` 不解释 `skillXp` | 互动无法直接涨技能 | P0 |
| 家具动作已有一批新 9 技能经验，但互动和场景动作仍不足 | 玩家可通过部分家具成长，但社交和场景成长闭环还不完整 | P0 |
| 秩序册还未接入主游戏顶部入口 | 管理技能无法在事务归口和管事评估里被玩家理解 | P0 |
| 低分/逾矩失败不解释技能经验 | 失败复盘无法提升察言/藏锋/通变 | P1 |
| 状态倍率尚未接入技能经验计算 | `zizai/teaHeart/ganshang` 等状态不能影响成长速度 | P1 |
| 性格倍率尚未接入技能经验计算 | 性格只能影响行动倾向，不能影响成长速度 | P1 |
| 满级里程碑没有实体 | 无法用于故事条件、传闻、特殊选项 | P2 |

## 11. 分期实现建议

### P0：打通最小闭环

1. 保留 `CONFIG.skillDefs` 混合表，先建立旧技能到新 9 技能的兼容映射。
2. 修改 `CharacterEffectSystem`，支持 `0-100` 熟练度或 `skillXp + skillLevel` 双字段。
3. 给 `applyInteractionEffects()` 补 `system: 'skillXp'`。
4. 先给第 6 节精选互动加经验，不一次性铺满全部模板。
5. 继续补场景训练动作，让不社交的日常行动也能成长。
6. 秩序册顶部入口接入后，在事务评估里展示新 9 技能中的管理子集。

### P1：加入成长速度

1. 增加 `SkillGrowthSystem`，集中计算状态/性格/场景/难度倍率。
2. 接入任务质量：任务完成质量越高，技能经验越多。
3. 接入失败复盘：低分、逾矩、危机失败可少量提升察言/藏锋/通变。

### P2：做角色养成感

1. 技能跨段反馈。
2. 满级里程碑标签。
3. 传令、秩序、任务质量和制度事件读取里程碑。
4. 支持专业二级技能，如女红、诗词、账房、医药、茶道。
