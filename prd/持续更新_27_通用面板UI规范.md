# 通用面板 UI 规范

> 更新时间：2026-07-10
> 适用范围：主游戏外层面板，包括【人物】、【起居】、【关系】、【传令】、【更多-技能】、【更多-背包】、存档、帮助等 `#panel-overlay / #panel-content` 承载的玩家前台面板。
> 主要代码：`index.html`、`js/ui.js`。

## 1. 目标

主游戏外层面板必须像同一套产品，而不是每个入口各自弹出不同尺寸。

统一目标：

- 所有主游戏外层面板默认使用【起居】面板尺寸。
- 美学风格对齐【人物】档案和【设置】-【人物设定】：宣纸底、轻玻璃、淡金线、玉色/青绿点缀。
- 面板外壳统一，内容区按模块自适应滚动。
- 不为了“看起来满”而扩大单个小功能面板；背包、技能等未完整系统也保持同尺寸。
- 前台面板只做玩家可读展示；后台人物设定是同口径的可编辑版本。

## 2. 统一尺寸

桌面端：

| 项 | 数值 |
| --- | --- |
| 面板宽度 | `86vw` |
| 面板高度 | `78vh` |
| 最大宽度 | `92vw` |
| 最大高度 | `85vh` |
| 外层定位 | `#panel-overlay` 居中 |

移动/窄屏：

| 项 | 数值 |
| --- | --- |
| 面板宽度 | `92vw` |
| 面板高度 | `84vh` |

说明：

- 【起居】、【秩序册】等复杂面板可使用 `overflow:hidden`，内部自己分区滚动。
- 普通面板可由 `.panel-box` 外层滚动。
- 不再允许人物、关系、技能、背包、传令各自定义外层宽高。

## 3. 外层视觉

`.panel-box` 是玩家前台通用面板壳。

视觉口径：

- 背景：宣纸/米白底，叠轻微径向高光与低饱和青绿光。
- 边框：细白玻璃线，不用厚重黑框。
- 阴影：柔和大阴影加内侧白线，表现纸面浮层。
- 字体：沿用项目全局字体，不单独切字体。
- 标题：16-17px，正常字重，2-3px 字距，底部淡金分隔线。
- 按钮：沿用 `.sys-btn`，主按钮使用淡金渐变。

## 4. 内容规范

### 4.1 人物

【人物】是前台信息美学基准：

- 以卡片分区承载人物总览、状态、身份、声望、梦想、职业。
- 性格、身份、状态等使用小标签，不直接暴露配置 JSON。
- 和后台【人物设定】保持信息口径一致，但前台更漂亮、后台可编辑。

### 4.2 起居

【起居】是复杂编辑面板尺寸基准：

- 使用固定外层尺寸。
- 内部分为摘要、满意度、工具栏、48 刻网格、右侧活动面板。
- 外层不滚动，网格/列表在内部区域消化。

### 4.3 关系

【关系】使用同一外壳，不再因关系卡数量改变外层尺寸。

- 关系卡可在内容区滚动。
- 卡片样式对齐人物面板：浅底、淡边、轻阴影。
- 图表可以保留，但不应撑破面板。

### 4.4 传令

【传令】使用同一外壳。

- 左侧点名传令，右侧群体传令。
- 月银/公账/失败策略等附属信息放入卡片区。
- 不因可传令对象多少改变外层面板尺寸。

### 4.5 技能与背包

技能和背包即使暂未完整实现，也使用同一外壳。

- 技能列表在内容区滚动。
- 背包未开放时展示空状态卡，不使用小弹窗。
- 未来背包接入后沿用同尺寸，内容区分栏即可。

## 5. 后台人物设定对齐

后台【设置】-【人物设定】是“可编辑版人物总览”。

对齐原则：

- 前台展示什么信息，后台应能找到对应配置或解释。
- 后台可出现输入框、表格、实际生效系数。
- 前台不展示消耗/恢复系数、JSON、高级配置字段。
- 两者共用同一视觉方向：轻玻璃、宣纸底、淡金线、青绿点缀。

## 6. 新增面板接入规则

新增主游戏面板时：

1. 必须通过 `openPanel(html)` 进入 `#panel-overlay / #panel-content`。
2. 不允许在内容 HTML 上写外层 `width/height`。
3. 如果需要复杂布局，只给内容根节点加模块类，例如 `.routine-panel`。
4. 如果需要内部滚动，在内容区或子列表滚动，不改 `.panel-box` 尺寸。
5. 标题使用 `h2` 或 `h3`，保持通用标题样式。
6. 卡片、按钮、表格优先复用现有 `.sys-btn`、浅底卡片和淡边线。

## 7. 当前已落地

已在 `index.html` 中统一：

- `.panel-box` 默认 `86vw × 78vh`。
- 移动端 `.panel-box` 默认 `92vw × 84vh`。
- 【起居】和【秩序册】保留同尺寸但内部滚动。
- 关系卡、传令卡、技能列表等前台面板内容采用更接近人物档案的浅玻璃卡片底。

后续如果某个面板仍然单独指定外层尺寸，应迁回本规范。
