## 1. 总目标

UI 是大观园模拟的操作层和解释层。它要让玩家能在同一屏内完成观察、选择人物、安排行动、查看关系、下发任务、阅读日志、切换 LLM 对话和进入后台配置。

设计目标：

- 第一屏就是可玩的模拟界面，不做落地页。
- 游戏画布保持主舞台，HUD 和面板只承载必要信息。
- 所有复杂系统都要有可解释入口：任务、关系、需求、状态、性格、身份、路径。
- 玩家操作和 AI 调试共用尽量一致的状态文案。
- 古典题材视觉要克制、清晰、可读，不让装饰压过信息。

## 5. 交互原则

- 点击地面：移动或加入行动队列。
- 点击家具：打开家具互动菜单，选择动作。
- 点击人物：打开人物互动菜单，可社交、差遣或查看相关入口。
- Shift 操作：用于追加或插队，具体由当前行动入口决定。
- 关系不足：不隐藏互动，显示意愿强弱和风险提示。
- 模型对白：必须可暂停、继续、终止；失败回退模板。
- 后台操作：保存状态要明确，批量导入要尽量可预检。

## 6. 视觉规范

| 项 | 口径 |
|---|---|
| 色彩 | 以纸色、青绿、金色、深墨为主；避免全屏单色调。 |
| 字体 | 正文小而清晰，标题克制，不使用负字距。 |
| 按钮 | 功能按钮使用清楚文字；图标按钮必须有 title。 |
| 面板 | 信息面板可以紧凑，但避免卡片套卡片。 |
| 气泡 | 文案短，优先不遮挡人物和队列。 |
| 状态色 | 正面、负面、混合、风险要区分，但不过度鲜艳。 |

## 7. 重点子系统

### 7.1 环形互动菜单

独立文档：`UI_环形互动菜单.md`。

维护重点：

- 菜单不能超出屏幕边缘。
- 中心区展示目标和返回/关闭。
- 第一层为类别，第二层为具体互动。
- 禁用项、风险项、LLM 项、意愿强弱要有视觉区分。

### 7.2 后台 UI

独立文档：`持续更新_15后台与人物配置.md`。

维护重点：

- v2 后台走左侧分组 + 主表 + 详情抽屉。
- 旧后台保留作为全量配置兜底。
- 表格编辑和 JSON 编辑的保存口径必须一致。

### 7.3 LLM 对话 UI

维护重点：

- `waiting`、`generating`、`replying`、`paused`、`error` 状态要可感知。
- 换话题只显示当前情境可用互动。
- 对话浮层不能遮住关键菜单和错误信息。
