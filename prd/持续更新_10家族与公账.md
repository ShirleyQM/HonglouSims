> 更新时间：2026-07-15

## 1. 总目标

家族系统负责把人物组织成可切换、可结算、可承担声望和公账变化的家庭单位。

它需要支持：

- 玩家按家庭理解人物位置、职责、居所和财政。
- 公账、个人钱、月银、工钱和任务奖励有统一入口。
- 家族事件能影响资金、声望、关系和人物状态。
- 仆从契约、随侍轮值、身份权限能找到人物所属家庭。

## 3. 数据口径

### 3.1 家庭 `familyConfig.families`

| 字段 | 含义 |
|---|---|
| `id`、`name` | 家庭标识与名称。 |
| `fund` | 初始公账。 |
| `residenceKind` | 住所类型：`onscene`、`offmap`、`nonresidential`。 |
| `residenceSceneId` | 地图内主要住所；仅 `onscene` 家庭需要。 |
| `residenceLabel` | 玩家可见住所名；地图外住所也必须有。 |
| `activitySceneId` | 可选。家庭常用办公/会客/活动场景，不等于居住地点。 |
| `members` | 成员列表，包含人物 ID 和家庭角色。 |
| `reputation` | 家庭声望初值。 |

### 3.1.1 家庭住所、地图外住所和搬家口径

家庭要有“住在哪里”，但不能把所有家庭都硬绑到一个可见地图场景。后续搬家、家庭管理、归巢都统一读 `ResidenceSystem`，不要让各系统直接猜 `family.residenceSceneId`。

住所类型：

| 类型 | 含义 | 例子 | 系统行为 |
|---|---|---|---|
| `onscene` | 住在当前可见地图中的场景。 | 怡红院、潇湘馆、蘅芜苑、秋爽斋。 | 归巢、睡觉、居家闲步、住所任务都可导航到该场景。 |
| `offmap` | 有住所，但当前地图未做出来。 | 宁国府、刘家庄、凤姐院中住处、荣国府内宅。 | UI 可显示“地图外”；归巢和住所任务先回退当前场景或跳过，后续接 OffMapCharacterPool。 |
| `nonresidential` | 场景可活动/办公/会客，但不住人。 | 荣禧堂、沁芳庭。 | 不能作为默认归巢和睡觉地点；可作为 `activitySceneId` 被任务、传令或事件使用。 |

当前实现：

- 稻香村已从当前地图移除，不再作为 `sceneId: 6` 的可见场景。
- 荣禧堂、沁芳庭按“非居住场景”理解：可以办公、会客、礼仪、聚会，但不作为家庭默认住所。
- 没有地图内住所的家庭，先配置为 `offmap`，如宁国府、刘家庄、协理院。
- 贾珍原先在稻香村，当前临时放到荣禧堂活动；他的真实住所仍属于宁国府地图外。

后续搬家：

```js
ResidenceSystem.moveFamily(familyId, {
  residenceKind: 'onscene',
  residenceSceneId: 4,
  residenceLabel: '怡红院'
})
```

搬家不是改人物当前坐标，而是改家庭住所配置；人物是否立刻搬过去，由传令/任务/过夜结算/家庭管理面板决定。

后续地图外归巢：

- P0：地图外住所只展示和解释，不把人物真的移出世界。
- P1：增加 `OffMapCharacterPool`，人物可进入“地图外/不在当前场景”状态，暂停普通寻路、保留经济/关系/日程结算。
- P2：地图外住所可被做成新场景包后迁入 `onscene`，旧家庭配置只需要改住所，不需要改人物和任务逻辑。

### 3.2 钱与交易

| 系统 | 职责 |
|---|---|
| `MoneySystem` | 人物现金、家庭公账低层变更、路径标记、代理人。 |
| `FamilySystem` | 家庭公账展示、交易流水、家庭事件和家庭关系修正。 |
| `EconomySystem` | 饭钱、工资、月银、上班候选。 |

## 4. 经济事件流程

```text
游戏小时事件
  -> EconomySystem 检查工作时间
  -> 生成上班 / 回家候选或结算工资

游戏日事件
  -> 月银周期检查
  -> 家庭每日事件
  -> 公账和声望变化
```

饭钱当前只在完成进食家具行为时扣除，不按时间自动扣。

## 5. 与任务和身份的关系

- 家庭角色决定场景权限、任务下发范围和仆从直属关系。
- 公账可作为任务奖励、月银、采办成本和惩罚来源。
- 家庭声望后续应影响外客来访、婚恋、路径晋升和重大事件。

## 5.1 家庭切换面板 UI 口径

入口：顶部【切换家庭】、左侧家庭栏小按钮、快捷键 `F`。

面板第一屏必须先回答“当前家庭是什么状况”：

| 字段 | 说明 |
|---|---|
| 当前家庭 | 家庭名、家主、家庭标签。 |
| 住所 | 读取 `ResidenceSystem.residenceOfFamily()`；地图外住所必须显示“地图外”。 |
| 账面 | 当前家庭公库余额。 |
| 人口 | 当前家庭成员数，附带家庭声望。 |
| 常用场所 | 可选，读取 `activitySceneId/activityLabel`，用于荣禧堂这类办公/会客场景。 |

下方家庭列表只做快速切换，保持紧凑小卡：

- 每个家庭显示：图标、名称、家主、人口、住所简写、公库、标签。
- 当前家庭高亮。
- 点击后切换家庭、刷新底部人物栏，并触发家庭成员自我介绍气泡。
