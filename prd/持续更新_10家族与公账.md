# 家族与公账 PRD

> 更新时间：2026-06-21
> 来源：当前代码与经济/职业/任务文档。
> 主要代码：`js/systems/family.js`、`js/systems/economy.js`、`js/systems/money.js`、`js/systems/servant-relations.js`、`js/admin.js`。

## 1. 总目标

家族系统负责把人物组织成可切换、可结算、可承担声望和公账变化的家庭单位。

它需要支持：

- 玩家按家庭理解人物位置、职责、居所和财政。
- 公账、个人钱、月银、工钱和任务奖励有统一入口。
- 家族事件能影响资金、声望、关系和人物状态。
- 仆从契约、随侍轮值、身份权限能找到人物所属家庭。

## 3. 数据口径

### 3.1 家庭 `familyConfig.families`

| 字段 | 含义 |
|---|---|
| `id`、`name` | 家庭标识与名称。 |
| `fund` | 初始公账。 |
| `residenceSceneId` | 主要住所。 |
| `members` | 成员列表，包含人物 ID 和家庭角色。 |
| `reputation` | 家庭声望初值。 |

### 3.2 钱与交易

| 系统 | 职责 |
|---|---|
| `MoneySystem` | 人物现金、家庭公账低层变更、路径标记、代理人。 |
| `FamilySystem` | 家庭公账展示、交易流水、家庭事件和家庭关系修正。 |
| `EconomySystem` | 饭钱、工资、月银、上班候选。 |

## 4. 经济事件流程

```text
游戏小时事件
  -> EconomySystem 检查工作时间
  -> 生成上班 / 回家候选或结算工资

游戏日事件
  -> 月银周期检查
  -> 家庭每日事件
  -> 公账和声望变化
```

饭钱当前只在完成进食家具行为时扣除，不按时间自动扣。

## 5. 与任务和身份的关系

- 家庭角色决定场景权限、任务下发范围和仆从直属关系。
- 公账可作为任务奖励、月银、采办成本和惩罚来源。
- 家庭声望后续应影响外客来访、婚恋、路径晋升和重大事件。

## 6. 后续待办

- 家庭面板增加长期账本：按天汇总收入、支出、月银、任务奖励。
- 公账不足时触发管家任务、裁撤开销或冲突事件。
- 家庭声望要接入更多行为：逾矩被目击、待客、诗社、管家成效。
- 个人钱、家庭公账和经济系统的字段命名需要再统一一轮。
