# 人物作息可视化面板 PRD

## 一、核心定位

起居面板用于给角色配置一周内的行为倾向。它不是强制脚本，而是给 Utility AI 提供“这个时段更偏向哪类互动”的权重锚点。

活动最终仍走统一底层：需求恢复、家具互动、社交互动、职业身份与未来技能系统都通过同一套活动模板、标签、能力要求和候选评分接入。

新起居是当前时段的显式覆盖层；旧基础作息与旧日程窗口只作为空白时段的兜底，不再与新起居重复叠加权重。

职业身份、每日任务和起居的关系：

- 职业身份决定人物每天“应该承担什么职责”，例如丫鬟、小厮、管事、先生、商人、医者。
- 每日固定职责应投影到起居职业行，例如晨起伺候、洒扫、陪读、随侍、伺候就寝、门房值守、巡查。
- 起居职业块不是临时任务实例，而是每日职责的时间投影；到点后由任务系统生成或推进对应 `daily` 任务。
- 临时差遣、剧情突发、AI 临时派活不进入起居，只在执行层作为插队/打断显示。
- 随侍属于职业日课中的目标跟随型起居块，可以显示在职业行，并拥有高于普通起居活动的执行优先级。
- 整体系统依赖图单独维护在 `prd/系统架构依赖图.md`。

## 二、主界面结构

面板顶部只保留人物标题，不显示“48刻网格（每刻30分钟）”说明文案。

标题下方展示三张信息卡：

| 信息卡 | 内容 |
| --- | --- |
| 性格 | 展示人物所有已配置性格标签，按性情、习惯、专属性格等分类合并展示。 |
| 职业 | 展示当前人物职业/身份相关标签。 |
| 技能 | 展示当前人物技能标签与等级。 |

信息卡下一行展示整体作息满意度：

```text
作息满意度 86%  尚可 · 当前刻 08:00
```

满意度是整份起居配置的总体评价，不再在每个活动块上显示单独满意度。

## 三、工具栏

工具栏分左右两组。

左侧从左到右：

| 控件 | 说明 |
| --- | --- |
| 星期日历 | 下拉选择星期日到星期六，用于切换当天作息草稿。 |
| 复制昨日 | 将昨日实际/配置作息复制到当前日。 |
| 应用到本周 | 将当前日配置应用到整周。 |
| 作息模板选项 | 下拉选择预设模板，如默认作息、读书人作息、商人作息等。 |
| 应用模板 | 将所选模板应用到当前日。 |

右侧从左到右：

| 控件 | 说明 |
| --- | --- |
| 保存 | 保存当前起居配置。 |
| 关闭 | 关闭面板；若存在未保存修改，必须提示确认。 |

关闭提示：

```text
当前起居尚未保存，确认不保存并关闭吗？
```

## 四、网格规则

主网格为 `3 x 48`：

| 行 | 含义 | 活动来源 |
| --- | --- | --- |
| 需求 | 饮食、睡眠、梳洗、休闲、社交、心绪等基础生活倾向。 | 常用/需求类活动模板。 |
| 技能 | 读书、吟诗、女红、侍奉、理务等技能训练或技能驱动活动。 | 技能模板，以及带技能要求但不是职业模板的活动。 |
| 职业 | 上朝、开店、授课、坐诊等职业身份活动。 | 职业模板。 |

横向为全天 48 刻，每刻 30 分钟，从 00:00 到 24:00。

点击任意空格会立即生成一个 30 分钟空白活动块，并选中该活动。活动块只能在当前行内左右拖拽，拖动时吸附到 30 分钟刻度。左右边缘可调整时长。

同一角色同一天的活动全局不允许重叠，不按需求/技能/职业分行放宽：

| 操作 | 规则 |
| --- | --- |
| 创建 | 目标刻已被任意活动占用时不创建，并提示该时段已有作息。 |
| 移动 | 优先保留原时长；若目标起点后的连续空档不足，自动裁剪到最大可用时长。 |
| 拉伸 | 左右拉伸都必须避开其他活动，超出可用空档时自动裁剪。 |
| 不足 1 刻 | 裁剪后不足 30 分钟时不执行创建/移动/拉伸，活动回到原状态。 |

支持跨午夜活动。活动数据仍保存为一个块，例如 `22:00-06:00` 表示为：

```json
{ "startSlot": 44, "durationSlots": 16 }
```

UI 渲染时将跨日活动拆成右侧末尾段和左侧开头段展示，但两个视觉片段共享同一个活动 id、选中态、拖拽状态和右侧活动面板。拖动跨午夜活动时按 48 刻环形日历循环滑动，不再被限制在 `48 - durationSlots` 之前。

## 五、活动面板

右侧固定活动面板与当前选中格所在行对齐：

| 当前行 | 右侧面板内容 |
| --- | --- |
| 需求 | 需求/常用活动。 |
| 技能 | 技能活动。 |
| 职业 | 职业活动。 |

选择右侧活动后，当前空白活动块被替换为对应活动模板。不会再额外新建第二个活动块。

活动块在网格内竖排显示活动名，不显示 `common`、`skill`、`profession` 等内部分类文本，也不显示单块满意度图标。

## 六、活动模板与底层接口

每个活动块本质上是一个活动模板引用，模板可包含：

| 字段 | 用途 |
| --- | --- |
| `needs` | 影响需求权重。 |
| `completeBy.categories/tags` | 用于匹配家具、社交或其他互动完成口径。 |
| `requiredSkills` / `skillReq` | 技能系统接入点。 |
| `requiredProfessions` | 职业身份接入点。 |
| `requiredFurniture` | 家具互动接入点。 |
| `boost/cut/completeCut` | AI 评分与完成后的抑制规则。 |

起居不单独实现一套执行逻辑。需求、技能、职业都应最终落到统一候选评分链，由家具互动、社交互动、任务/职业行为等 provider 生成候选，再由起居锚点调整权重。

职业行活动分为两类：

| 类型 | 说明 | 示例 |
| --- | --- | --- |
| 职业活动模板 | 直接由职业身份解锁的日常行为，主要影响 Utility AI 候选。 | 上朝、开店、授课、坐诊。 |
| 职业日课投影 | 由职业/职责系统生成的稳定每日任务窗口，到点生成或推进 `daily` 任务。 | 晨起伺候、洒扫、陪读、随侍黛玉、伺候就寝。 |

职业日课块建议字段：

```js
{
  sourceType: 'dutyQuest',       // dutyQuest / followRotation / lifePathDaily / manualRoutine
  questTemplateId: 3005,
  contractId: 'daiyu_zijuan',
  masterId: 'daiyu',
  role: '贴身侍女',
  locked: true,
  priority: 'duty',
  completeConditionType: 'FOLLOW_CHARACTER'
}
```

随侍块是特殊职业日课：

- 起居上显示为 `随侍黛玉`、`随侍宝玉` 等目标跟随型块。
- 执行上不绑定固定格子，而是持续生成靠近目标、保持距离、跟随目标的候选。
- 进度由 `FOLLOW_CHARACTER` 判定，满足同场景且距离阈值内才增长。
- 饥饿、洁净、精力过低时可请辞，解决后回归随侍。
- 随侍优先级高于普通读书、闲逛、普通社交，但低于严重需求危机和强制剧情。

Utility AI 接入顺序：

| 层级 | 生效规则 |
| --- | --- |
| 新起居 profile block | 当前时段存在新起居块时，直接返回新起居锚点并覆盖兜底。 |
| 旧 `routineAnchors` | 当前时段没有新起居块时，作为第一层基础作息兜底。 |
| 旧 `scheduleWindows` | 当前时段没有新起居块且没有旧基础作息锚点时，转换为宽泛起居锚点补充空档。 |

`calcScheduleFactor` 不再独立乘入最终权重。旧日程窗口只通过统一后的 `routineFactor` 生效，`scheduleWindow` 字段仅保留为兜底来源诊断。

旧 `scheduleWindows` 转换后的兜底锚点不参与“活动完成”结算，避免一次家具或社交行为让整段日程窗口失效。

需求活动通过 `completeBy.categories/tags` 匹配实际候选。家具候选的 `category/tags/needRestores`，以及社交候选的 `category/social`，都进入同一套 `candMatchesRoutineAnchor` 与 `calcRoutineFactor` 判断。职业身份和未来技能系统继续通过 `requiredProfessions`、`requiredSkills`、`requiredFurniture` 等字段接入。

## 七、保存与周历

面板支持星期切换。每个星期日到星期六都可以保留独立草稿；保存时将当前星期草稿与周历草稿一起写入角色起居配置。

当前 AI 默认读取角色当前生效的 `blocks`，周历数据作为后续按星期自动切换的扩展接口保留。

## 八、验收标准

| 项 | 标准 |
| --- | --- |
| 顶部说明 | 不再显示“48刻网格（每刻30分钟） · 需求 / 技能 / 职业”。 |
| 信息卡 | 显示性格、职业、技能三张大卡。 |
| 满意度 | 只显示整体作息满意度，不显示单活动满意度。 |
| 工具栏 | 左侧为星期日历、复制昨日、应用到本周、模板选择、应用模板；右侧为保存、关闭。 |
| 关闭 | 未保存关闭必须确认。 |
| 网格 | 3 行 x 48 刻完整展示。 |
| 创建 | 点击格子直接创建 30 分钟空白活动。 |
| 不重叠 | 同一天任意活动全局不重叠；冲突创建失败，移动/拉伸自动裁剪或回弹。 |
| 跨午夜 | `22:00-06:00` 等活动保存为单个跨日块，UI 拆成两段展示并可循环拖动。 |
| 活动选择 | 右侧面板按当前行展示对应活动，并替换当前活动块。 |
| 活动块 | 竖排显示活动名，不展示内部分类文本。 |
| AI 合并 | 新起居覆盖当前时段，旧基础作息和旧日程窗口只作为空白时段兜底，最终只通过 `routineFactor` 加权。 |
