# 后台与人物配置 PRD

> 更新时间：2026-07-08
> 来源：当前后台 v2、人物设定页、配置保存和试玩发版状态。
> 主要代码：`js/admin.js`、`js/config.js`、`js/ui.js`、`js/assets.js`。

## 1. 总目标

后台是开发者和策划维护游戏内容的主入口，不是普通玩家试玩入口。

后台目标：

- 常用配置表格化，减少大段 JSON 暴露。
- 页面风格统一为“性格系统”那种紧凑表格式后台。
- 人物设定是“可编辑版人物总览”：与前台人物档案信息口径一致，但后台可改。
- 暂时不用或容易误导的配置页先隐藏，不删除底层业务。
- 配置保存和游玩存档分离。

## 3. 人物设定口径

后台【人物设定】展示：

- 立绘缩略预览，不占据主要编辑空间。
- 立绘下方只展示人物自定义描述，不放实时状态。
- 基础信息第一层为 5 个自适应小框：性别、年龄、家庭、家庭身份、血缘。血缘暂未开放，只占位。
- 身份、声望、职业单独成行：身份编辑 `socialRank`；职业编辑 `lifePath/currentStage`；声望展示五格解释。
- 性格四框置于基础需求上方：性情、习惯、处世、特殊。特殊读取人物专属性格 `profiles[charId].specialties`。
- 梦想可选择长期梦想类型，保存到 `charSpecialtyConfig.dreamProfiles[charId].type`；标题、说明和达成条件读取梦想配置，不在人物设定页自定义。
- 基础需求系数：每个需求横向一行，后台编辑 `baseNeedCoeffs`，实际运行进入 `calcNeedCoeffs`。表格同时展示配置值和实际值。
- 基础需求顺序固定为：饥饿、精力、清洁、社交、意趣、心绪。
- 基础需求旁展示需求层次金字塔：底层饥饿/精力/清洁，中层社交/意趣，顶层心绪。
- 健康为 0-100，可在人物设定页直接调试配置；运行时仍会被日结、状态、照顾事件继续影响。

前台【人物】展示同一口径，但只读、简约、美观，不展示 JSON 和后台字段。

## 3.1 声望五格口径

人物设定页不直接把所有历史声望域平铺给策划，而是展示五格解释：

| 显示名 | 底层来源 | 权重 | 说明 |
| --- | --- | ---: | --- |
| 声望 | 面子、口碑、才名、功名加权派生 | 1.00 | 总体展示值，人物设定页按下方四项加权计算。 |
| 面子 | `family` | 0.25 | 长辈评价、差事完成度、是否逾矩、人际关系。小范围、熟人圈评价。 |
| 口碑 | `outside`，仆从角色可参考 `servant` | 0.20 | 善行、恶行、流言、与外界互动。 |
| 才名 | `scholarly` | 0.15 | 技能等级、作品质量、同行评价。 |
| 功名 | `official` | 0.40 | 仕途、科第、科举结果、官场表现、上司考核；与职业系统联动。 |

当前兼容边界：

- `reputationDomainConfig.domains.general` 仍保留为旧系统兼容声望，用于旧任务、梦想、人生路径条件兜底。
- 人物设定页的“声望”显示为派生展示值，不强行覆盖旧 `general` 存量，避免旧条件突然变化。
- `servant` 暂命名为“仆从口碑”，仍用于仆从圈内部评价，暂不进入主五格，后续可在仆从/主仆专页展开。

## 4. 配置保存口径

| 数据 | localStorage key | 说明 |
| --- | --- | --- |
| 配置 | `dgy_config` | 后台保存的完整 `CONFIG`。 |
| 进度 | `dgy_save` | 当前游戏进度，由存档系统序列化。 |
| 行为埋点 | `dgy_behavior_telemetry_v1` | AI、任务、行为调试数据。 |

后台保存配置调用 `saveConfigToStorage()`，游玩进度调用 `saveGameToStorage()`。

## 5. 页面分组

| 分组 | 当前页面 | 说明 |
| --- | --- | --- |
| 人物配置 | 人物设定、基础需求和状态、性格、梦想 | 人物总览和人格/需求配置。 |
| 关系与身份 | 关系、身份礼法、声望相关表 | 场景权限暂不作为常用页暴露。 |
| 职业与故事 | 职业系统、故事系统 | 职业管身份职责；故事管路径节点。 |
| 任务与传令 | 任务模板、传令/上下级相关配置 | 临时任务不进入起居。 |
| 行为规则 | 家具、互动模板/多人互动、Utility AI | 家具/社交/任务共享行动底层。 |
| AI 与调试 | 行为埋点、导入导出、高级 JSON | 服务开发和调参。 |
