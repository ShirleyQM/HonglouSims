# 后台与人物配置 PRD

> 更新时间：2026-06-21
> 来源：当前后台实现与各系统配置页。
> 主要代码：`js/admin.js`、`js/config.js`、`js/ui.js`、`js/assets.js`。

## 1. 总目标

后台是策划维护游戏内容的主入口。它要让人物、性格、关系、状态、互动、任务、家具、身份、场景、人生路径和多人反应都能被查看、编辑、导入和保存。

设计目标：

- 常用配置表格化，少让策划直接改大段 JSON。
- 高级 JSON 保留，方便批量替换和快速修复。
- 人物配置按“总表”和“单人物设定”分层。
- 每个配置页都能说明它会影响哪些运行系统。

## 3. 配置保存口径

| 数据 | localStorage key | 说明 |
|---|---|---|
| 配置 | `dgy_config` | 后台保存的完整 `CONFIG`。 |
| 进度 | `dgy_save` | 当前游戏进度，由存档系统序列化。 |
| 行为埋点 | `dgy_behavior_telemetry_v1` | AI 和任务调试数据。 |

后台保存配置时调用 `saveConfigToStorage()`，游玩进度调用 `saveGameToStorage()`。
