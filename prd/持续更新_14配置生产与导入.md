# 配置生产与导入 PRD

> 更新时间：2026-06-25
> 主要代码：`js/config.js`、`js/admin.js`、`prompts/batch-gen/`、`import systems/`。

## 1. 总目标

配置生产与导入体系用于把大量策划内容稳定写入游戏，而不是手工散改 `config.js`。

覆盖内容：

- 状态、互动模板、需求气泡、情绪传染、旁观反应。
- 关系阈值事件、冲突剧场、记忆浮现。
- 性格、人物配置、身份礼法、职业系统、故事系统。
- Excel / CSV / JSON 生成、检查、导入和后台编辑。

## 3. 生成顺序

推荐顺序：

```text
基础需求和状态
  -> 性格 / 人物专属性格
  -> 互动后气泡 / 需求气泡
  -> 互动模板 / 多人互动
  -> 任务模板 / 家具动作
  -> 职业系统
  -> 故事系统
  -> 身份礼法 / 声望规则
```

原因：后续表会引用前面的状态 ID、互动类别、需求 key、关系轴、任务模板和效果类型。

## 4. 通用约束

| 项 | 规则 |
| --- | --- |
| 文风 | 红楼梦大观园，古风白话，口语化，不用网络梗。 |
| 时间单位 | `duration`、`cooldown`、`cooldownGameMin` 均为游戏分钟。 |
| 状态 ID | camelCase 或 CSV 编号均可，但同一状态不要混用。 |
| 关系轴 | `affection`、`trust`、`friendship`、`submission`。 |
| 需求 key | `hunger`、`energy`、`fun`、`hygiene`、`social`、`mood`。 |
| 需求系数 | 人物基础系数写入 `baseNeedCoeffs`。 |
| effect system | `need`、`relation`、`state`、`memory`。 |
| 输出 | 纯 JSON 数组或 CSV 行，不要包 markdown。 |

## 5. 导入前检查

- ID 唯一，且不覆盖非本次目标内容。
- 引用的状态、需求、关系轴、技能、互动类别存在。
- 冷却和持续时间单位是游戏分钟。
- 气泡文本长度适合头顶显示。
- LLM prompt 名称能在 `prompts/interaction-topics.txt` 找到或有兜底。
- `effects` 不直接引用不存在的人物或场景。
- 职业日课与故事节点不要混用：日常职责进职业系统，剧情节点进故事系统。
