## 2026-06-22 置顶修订：从“把柄”升级为正负都有的秘密系统

“搞事情”不应只等于陷害、告密和毁人。更适合大观园的底层系统是：**秘密、隐私、信任与揭露**。

秘密可以是负面的把柄，也可以是正面的心事、恩情、暗恋、才华、病症、身世、私房钱、未公开计划。玩家和 NPC 的乐趣不只是“害人”，也包括：

- 帮人保守秘密。
- 用秘密成全一段关系。
- 用秘密安慰或救人。
- 以秘密换取信任。
- 揭露秘密制造风波。
- 误解秘密导致反噬。

因此第一阶段不做纯负向“把柄系统”，而做统一的 `SecretSystem`。其中 `leverage` 只是秘密的一种用途。

### 一、秘密类型

| 类型 | 性质 | 示例 | 可带来的玩法 |
|---|---|---|---|
| `affection` 心意 | 正 / 中性 | 暗恋、惦念、赠物心意 | 成全、撮合、传话、误会 |
| `favor` 恩情 | 正 | 某人曾暗中相助 | 报恩、护短、提升信任 |
| `talent` 才华 | 正 | 私下写诗、会管账、会医术 | 举荐、任务机会、声望提升 |
| `illness` 病症 | 中性 / 负 | 体弱、隐疾、心病 | 照顾、隐瞒、误传、求医 |
| `fear` 忌惮 | 中性 | 怕长辈、怕被逐、怕失宠 | 安慰、利用、保护 |
| `plan` 计划 | 中性 | 准备搬屋、求职、筹钱、送礼 | 协助、阻挠、提前布局 |
| `wealth` 私财 | 中性 / 负 | 私房钱、借贷、放利钱 | 借钱、资助、勒索、查账 |
| `misconduct` 逾矩 | 负 | 私会、偷盗、失礼、违禁物 | 告密、要挟、搜查、惩罚 |
| `frame` 伪证 | 负 | 伪造书信、栽赃物 | 查证、反坐、洗冤 |

### 二、秘密不是普通记忆

普通记忆回答“我经历过什么”。秘密回答“谁知道了什么、能不能被用、被谁相信、公开后会怎样”。

建议新增 `SecretSystem`，但与记忆系统互通：

```js
{
  id: 'secret_001',
  type: 'affection',
  polarity: 'positive',       // positive / neutral / negative / mixed
  holderId: 'xiren',          // 谁知道
  subjectIds: ['baoyu'],      // 秘密关于谁
  targetIds: ['daiyu'],       // 可选：牵涉对象
  source: 'overhear',         // overhear / witness / confession / gift / rumor / forged
  sceneId: 4,
  title: '宝玉惦念黛玉',
  content: '听见宝玉私下问起黛玉近日心绪。',
  strength: 60,               // 重要程度
  credibility: 80,            // 可信度
  intimacy: 30,               // 私密程度，越高越不该公开
  expiresAt: 12345,
  exposed: false,
  usedCount: 0
}
```

### 三、秘密的使用方式

| 使用方式 | 倾向 | 说明 |
|---|---|---|
| 保守秘密 | 正 | 提升信任、友谊；适合忠诚、厚道、温和人物。 |
| 私下提醒 | 正 | 告诉当事人“我知道，但我帮你遮掩”。 |
| 成全 / 撮合 | 正 | 把心意、恩情、计划转成任务或互动机会。 |
| 举荐 | 正 | 把才华秘密转成声望、任务、职位机会。 |
| 求助 | 中性 | 病症、困境、债务秘密触发照顾或筹钱任务。 |
| 交换 | 中性 | 用秘密换钱、换帮忙、换承诺。 |
| 传闲话 | 负 / 中性 | 秘密变流言，可信度下降但扩散更快。 |
| 告密 | 负 | 向权威报告，引发搜查、禁足、惩罚。 |
| 要挟 | 负 | 逼迫对方给钱、办事、远离某人。 |
| 洗白 / 辟谣 | 正 | 公开澄清伪证，提升信任或反坐陷害者。 |

### 四、第一阶段最小闭环

先做温和版，不先做下药、死亡、冤魂。

第一批互动：

| 互动 | 作用 | 正负性 |
|---|---|---|
| 偷听 | 获得秘密，但被发现会降关系 | 中性偏负 |
| 私下询问 | 对目标温和确认秘密，可能转为信任 | 中性偏正 |
| 保守秘密 | 消耗一次秘密使用机会，提升信任/友谊 | 正 |
| 私下提醒 | 让当事人避开风险，可能触发感谢 | 正 |
| 传话成全 | 将心意/计划类秘密转成互动机会 | 正 |
| 告密 | 将负向秘密报告给权威 | 负 |
| 要挟 | 将负向秘密转成任务或金钱压力 | 负 |
| 辟谣 | 对伪证/流言类秘密进行澄清 | 正 |

第一批 UI：

- 【秘密面板】显示“我知道的秘密”。
- 每条秘密展示：性质、可信度、私密度、有效期、可用行动。
- 不直接告诉玩家所有真相；对“别人掌握我的秘密”只显示推测或风险提示。

第一批系统接入：

- `CharacterEffectSystem` 支持 `type: 'secret'`。
- `NarrativeBubbleSystem` 可读取秘密生成气泡，例如“这话若传出去……”或“我替你瞒着便是”。
- `QuestSystem` 支持由秘密触发轻任务：提醒、送信、遮掩、求医、搜查。
- `AiDrama` 后续读取秘密，生成保护、撮合、报复、告密等意图。

### 五、分期建议

P0：正负都有的 `SecretSystem`，加秘密面板和 4 个温和互动：偷听、私下询问、保守秘密、私下提醒。
P1：加入成全、传话、告密、要挟，接任务和关系后果。
P2：加入伪证、栽赃、搜查、辟谣，形成阴谋链。
P3：再考虑巫蛊、下药、死亡、冤魂等重戏剧事件。

这样“搞事情”会更像大观园：不是纯恶作剧，而是每个人都有不能说出口的心事。玩家既可以拆台，也可以护人，系统会更有层次。

---

模拟人生的“猎奇”本质是给玩家一套**制造混乱、窥探隐私、打破规则**的工具。
我们把它翻译成大观园语境，就是：**私通、陷害、巫蛊、争宠、落水、抄家**——全是红楼梦里现成的。

基于现有系统，不需要推翻重来，只需要新增一套**阴谋互动模板** + **秘密目击扩散机制**，就能让游戏从“过日子”变成“搞事情”。

## 一、新增阴谋互动类别：**暗流**

在现有六大类（叙旧/论道/调笑/慰藉/传情/争执）之外，增加第七类：**暗流**。
它的特征是：**隐蔽性强、后果严重、常涉及第三方**。

| 互动名 | 描述 | 条件 | 私密要求 | 效果 | 暴露后果 |
|--------|------|------|----------|------|----------|
| **栽赃** | 将一件违禁品（如巫蛊人偶、情书）偷偷放入目标房间 | 拥有可栽赃物品，与目标关系≤-30 | 独处目标私室或公共区域无人时 | 目标被搜查时触发惩罚事件 | 若被目击，自身反坐，声望-50 |
| **下咒** | 请马道婆或自己动手施巫蛊，让目标生病或精神失常 | 拥有咒术物品，或雇佣懂巫术的NPC | 极私密（自己房间或寺庙） | 目标获得“中邪”状态，行为失常数日 | 被查出则抄家级丑闻 |
| **偷听** | 故意藏在屏风后、窗外偷听他人私密对话 | 目标在进行私密互动 | 躲藏在家具后（屏风/假山） | 获得“把柄”信息，可用于日后要挟 | 被发现则关系-20，被驱逐 |
| **告密** | 将偷听或目击的把柄汇报给有权者（王夫人、贾母） | 持有把柄信息 | 私室，单独对话 | 被告受罚，告密者获赏或信任 | 若被告知是诬告，反坐 |
| **要挟** | 以把柄威胁对方做某事（给钱、办事、远离某人） | 持有把柄 | 私下二人 | 对方被迫接受一个任务或放弃某行动 | 对方可能鱼死网破 |
| **引诱** | 用言语或物品引诱对方做出逾矩行为，然后记录 | 自己魅力或口才高，目标好感＞30 | 私室 | 对方进入“被引诱”状态，下次逾矩成功率+50% | 若对方不上钩，关系-10 |
| **伪造书信** | 模仿他人笔迹写一封情书或密信，放入某人处 | 文学或书法技能高 | 私室 | 制造虚假把柄，可用于栽赃 | 笔迹被识破则自身被追查 |
| **下药** | 在饮食中放入迷药或毒药，使对方昏睡或生病 | 拥有药物 | 厨房或饭桌无人时 | 目标精力归零或获“病中”状态 | 致死则触发命案调查 |

这些互动直接调用现有的**逾矩系统**（成功率+目击判定）、**观察-反应**（被看见则流言扩散）、**冲突引擎**（受害者报复），不增加新架构。

## 二、把柄与秘密扩散机制

猎奇的核心是“我知道你的秘密，我可以毁了你”。
需要在记忆系统中增加一种类型：**把柄**。

- 当玩家或NPC通过偷听、目击、引诱等手段获得他人逾矩证据时，生成一条“把柄记忆”，标注目标、内容、强度。
- 持有把柄者可随时执行**告密**或**要挟**互动。
- 把柄有“时效性”：太久不用，对方可能销毁证据，把柄失效。
- NPC也会互相掌握把柄，形成复杂的利益网络。例如赵姨娘可能掌握凤姐放贷的把柄，凤姐掌握贾琏偷娶的把柄。

**把柄控制台（玩家界面）**：
一个隐藏的“秘密”面板，列出你掌握的所有人的把柄，以及谁可能掌握你的把柄（推测）。这给玩家一种“情报战”的掌控感。

## 三、意外死亡与鬼魂（轻量超自然）

红楼梦有秦可卿托梦、贾瑞风月宝鉴、尤二姐吞金等情节。我们可以引入“冤魂不散”作为极少数事件的后续。

- 如果某角色因陷害、冤屈而死（如尤二姐），其鬼魂可能出现在涉事者的梦境或特定场景（如天香楼），导致涉事者心绪持续下降、出现幻觉。
- 鬼魂不能被互动，只是环境叙事和状态减益。
- 可通过做法事（请僧道）或完成遗愿来平息。

这不需要复杂系统，只需在角色死亡后生成一个“冤魂”状态附着于特定场景，对进入者施加情绪效果。

## 四、玩家“捣乱”的乐趣整合

结合现有系统和新增暗流，玩家的猎奇玩法路径如下：

1. **选一个目标**：比如看不惯的赵姨娘。
2. **收集把柄**：操控丫鬟偷听她与贾环的私语，或偷看她房间。
3. **制造把柄**：伪造一封她与马道婆的通信，放入她柜中。
4. **引爆**：在王夫人面前告密，触发搜查事件。
5. **旁观连锁反应**：赵姨娘被禁足、贾环受牵连、探春愤怒又无奈——全府流言四起，观察-反应系统自动产生大量围观和议论。
6. **反噬风险**：如果伪造书信被识破，自己就成了全府公敌。

整个过程玩家感受到的是：**策划阴谋—执行—赌上风险—观看风暴**，这正是模拟人生“拆掉泳池梯子”的快感，但完全包裹在红楼梦的家族政治语境中。
