# 运行时、行动队列与存档 PRD

> 更新时间：2026-06-21
> 来源：当前运行时代码、服务脚本和存档实现。
> 主要代码：`js/runtime.js`、`js/action-queue.js`、`js/world.js`、`js/main.js`、`js/systems/quest.js`、`serve.mjs`。

## 1. 总目标

运行时系统负责把所有玩法系统接在同一个模拟循环上：时间推进、行动排队、移动、家具使用、互动执行、事件日志、自动保存和本地服务。

目标：

- 玩家和 AI 都通过同一行动队列执行行为。
- 行动失败能发出可诊断事件，不静默卡死。
- 游戏进度能序列化和恢复。
- 本地服务支持静态资源、局域网访问和 Ollama 代理。

## 3. 行动队列口径

常见行动项：

| 类型 | 来源 | 执行内容 |
|---|---|---|
| `move` | 玩家点击、AI 候选、任务 | 寻路并移动到目标格。 |
| `furniture` | 家具候选、任务候选 | 使用家具动作，恢复需求或产生效果。 |
| `interaction` | 互动菜单、AI 社交、多人物反应 | 靠近目标并执行互动模板或 LLM 对话。 |
| `quest` / 派生项 | 任务系统 | 根据完成条件转成移动、家具、跟随、交互等行为。 |

队列失败必须发 `queue:failed`，带上人物、候选 key、原因、目标场景或家具，供 AI 冷却和调试面板使用。

## 4. 存档范围

当前存档包含：

- 游戏时间、选中人物、队列页码、行动 ID。
- 人物位置、需求、状态、行动队列、记忆、技能、路径、钱。
- 关系数据。
- 家族、公账、经济、任务、任务下发、仆从关系、场景权限、人生路径、运势等系统状态。

配置不写入进度存档，而是单独存在 `dgy_config`。

## 5. 本地服务与 LLM

| 命令 | 用途 |
|---|---|
| `npm run start` | 启动静态服务，默认监听 `0.0.0.0:8765`。 |
| `npm run start:local` | 只监听 `127.0.0.1:8765`。 |
| `node serve.mjs` | 直接启动服务。 |
| `npm run tunnel` | 通过 cloudflared 暴露本地服务。 |

`serve.mjs` 还提供 Ollama 相关能力：

- `/ollama` 代理模型请求。
- 可尝试启动 `ollama serve`。
- 文件模式打开游戏时，LLM 对话会提示改用本地服务。

## 6. 后续待办

- 存档版本迁移规则需要文档化，每次新增字段都写迁移说明。
- 行动队列应增加统一 item schema，减少不同系统各造字段。
- 队列失败原因需要枚举化：不可达、权限不足、家具占用、目标离开、需求打断、任务阻塞。
- 服务脚本可增加健康检查页，显示资源、Ollama 和当前端口状态。
