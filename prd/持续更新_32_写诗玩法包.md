## 1. 玩法定位

写诗不是单纯“恢复意趣”的家具动作，它在红楼生活模拟里承担四件事：

- 表达人物性情：黛玉的伤怀、宝钗的稳妥、湘云的豪爽、探春的组织力，都可以通过诗作和诗社表现。
- 连接社交关系：联句、评诗、命题、诗社排名，本质是雅集社交。
- 形成可留存产物：诗稿、题帕、题壁、诗社名次、被人记住的一句好诗。
- 触发长线情绪：得句会开心，苦吟会耗精力，败兴会烦乱，极端情境下可能写出危险文本。

因此写诗玩法包不应只做成“书案 + poetry 经验”，而应逐步变成：独处创作、多人诗社、情绪排遣、才名增长和关系波动的组合玩法。

## 2. 当前系统基础

可复用现有能力：

| 系统 | 当前可用点 |
|---|---|
| 家具 actions | 书案、花圃、亭台类家具可挂写诗动作。 |
| 技能 | 旧技能 `poetry`、新技能 `literacy` 可共同使用；`eloquence / insight / diligence` 可少量参与。 |
| 性格 | `caihua`、`duochou`、`qinggao`、`gupi`、`xijing`、`haosheng`、`qianrang` 等都能影响诗作倾向。 |
| 状态 | `ganshang`、`melancholy`、`zizai`、`teaHeart`、`elated` 等可作为触发或结果。 |
| Activity Pack | 可声明家具动作并挂入菜单和 AI 候选。 |

当前不足：

- 没有诗作产物系统。
- 没有诗社多人活动。
- 没有“作品质量”结算。
- 没有才名/口碑和诗作的关联。
- 没有统一 `conditions` 解释器，暂时只能把触发条件写在 PRD 和配置草案里。

## 3. 设计原则

1. 写诗应有“普通消遣”和“强表达”两层，不要让 NPC 每次都写得像人生绝笔。
2. 日常写诗可以自主 AI；高戏剧动作必须由玩家、事件或极端状态触发。
3. 写诗要有失败和低质结果：俗套、苦吟不得、走题、被人笑话。
4. 多人诗社不只是多人同时写诗，而是命题、限韵、评诗、排名和关系反馈。
5. 诗作要能沉淀为记忆/作品，后续接才名、梦想、人物传闻。

## 5. 技能设计

首版不新增技能，复用现有技能：

| 技能 | 用途 |
|---|---|
| `poetry` | 旧技能，表达诗词专精；短期继续保留。 |
| `literacy` | 文墨底层能力，影响用典、句法、改稿质量。 |
| `eloquence` | 联句、评诗、诗社场面表达。 |
| `insight` | 即景、读人、借物寄情。 |
| `diligence` | 改稿、苦吟、抄录。 |

后续如果诗词玩法变重，可以考虑把 `poetry` 从旧技能迁成正式专精，或在技能系统里明确“文墨是基础，诗才是专精”。

## 6. 家具动作表

| 动作 | ID | 家具 | 自主 AI | 主要收益 | 风险/限制 |
|---|---|---|---|---|---|
| 抄录旧诗 | `poetry_copy_old` | 书案 | 是 | 意趣小回升，`literacy/diligence` 经验 | 低戏剧，适合日常。 |
| 起兴得句 | `poetry_find_line` | 书案/花圃 | 是 | 意趣、心绪小回升，`poetry/literacy` 经验 | 低概率得“佳句”。 |
| 即景题诗 | `poetry_scene_poem` | 花圃/园景 | 是 | 心绪、意趣、`insight/poetry` 经验 | 需要场景氛围，户外更合适。 |
| 苦吟推敲 | `poetry_revise_hard` | 书案 | 是，但低权重 | `poetry/literacy/diligence` 经验较高 | 精力下降，可能烦乱。 |
| 题帕题笺 | `poetry_write_token` | 书案 | 否 | 生成私密诗稿，关系/记忆入口 | 需要目标或事件，容易越界。 |
| 联句唱和 | `poetry_couplet` | 书案/亭台 | 条件 AI | 社交、口才、诗才，关系小幅变化 | 需要 2 人以上。 |
| 诗社命题 | `poetry_club_prompt` | 书案/亭台 | 否 | 多人活动、排名、才名 | 需要组织者/诗社事件。 |
| 葬花成诗 | `poetry_bury_flower` | 花圃 | 条件 AI | 黛玉式情绪表达，诗才和记忆 | 需要感伤/低心绪，不应泛化。 |
| 写险诗 | `poetry_desperate` | 书案/花圃 | 否 | 强叙事节点 | 极低心绪或剧情，必须禁止普通 AI。 |

首版 P0 已按上表只接前四个日常动作；`题帕题笺 / 联句唱和 / 诗社命题 / 葬花成诗 / 写险诗` 仍放后续阶段。

## 7. 状态设计

建议新增或规范以下状态：

| 状态 | ID 建议 | 极性 | 来源 | 效果 |
|---|---|---|---|---|
| 得句 | `goodLine` | positive | 起兴得句、即景题诗成功 | 心绪/意趣上升，短时提高写诗权重。 |
| 苦吟 | `hardComposing` | mixed | 苦吟推敲 | `poetry/literacy` 成长略高，精力恢复下降。 |
| 诗兴 | `poeticMood` | positive | 茶、花、雪、月、琴、微醺等触发 | 写诗/联句权重提高。 |
| 败兴 | `poetryBlocked` | negative | 苦吟失败、被打断、被嘲笑 | 写诗权重短时下降，心绪下降。 |
| 佳作流传 | `poemPraised` | positive | 诗社高分/被称赞 | 才名、心绪、关系正向。 |

注意：

- `写险诗` 不应该直接复用普通 `poeticMood`，要走心绪崩溃、重大事件或剧情层。
- 诗兴可以被其他玩法包触发，例如饮酒微醺、品茶、赏花、听曲。

## 8. 性格接入

首版不新增性格，先复用现有性格。

| 性格 | 影响 |
|---|---|
| `caihua` 才华 | 写诗、联句、佳作概率提高；诗作质量加成。 |
| `duochou` 多愁 | 情绪化写作、葬花成诗、苦吟权重提高；负面状态下更容易写。 |
| `qinggao` 清高 | 独处写诗、评诗、拒绝俗套题目。 |
| `gupi` 孤僻 | 独处写诗权重提高，诗社加入降低。 |
| `xijing` 喜静 | 书案、独处、改稿权重提高。 |
| `haosheng` 好胜 | 诗社、联句、争胜权重提高。 |
| `qianrang` 谦让 | 诗社争名权重下降，但评诗更温和。 |

未来可新增对偶性格，但不在本轮草案引入：

- 爱诗 <-> 务实
- 用典 <-> 白描
- 苦吟 <-> 天成

## 9. 作品质量结算

写诗玩法最好有一个轻量质量分，不需要立刻生成真实诗文。

建议公式：

```text
作品质量 =
  基础动作难度
+ poetry * 0.45
+ literacy * 0.25
+ insight * 0.15
+ 状态加成
+ 性格加成
+ 场景氛围
- 精力不足惩罚
- 心绪过低惩罚或转化为险诗风险
+ 随机波动
```

质量分只用于系统判断：

| 分数 | 结果 |
|---:|---|
| 0-30 | 不成篇，只得几句。 |
| 31-55 | 平平一首，可自娱。 |
| 56-75 | 有可观处，可被人称赞。 |
| 76-90 | 佳作，进入记忆/才名。 |
| 91+ | 名句，进入传闻或特殊节点。 |

首版可以先不生成完整文本，只生成一条作品记录：

```js
{
  id: 'poem_xxx',
  title: '未题',
  authorId: 'daiyu',
  activityId: 'poetryWriting',
  quality: 78,
  mood: 'ganshang',
  sceneId: 12,
  createdAt: { day: 3, hour: 17 },
  tags: ['flower', 'sorrow'],
  excerpt: '花影低回，似有旧愁'
}
```

## 10. AI 边界

可自主：

- 抄录旧诗
- 起兴得句
- 即景题诗
- 苦吟推敲，但需低权重和疲劳限制

条件自主：

- 联句唱和：需要同场 2 人以上，且社交意愿不低。
- 葬花成诗：需要花圃/落花氛围 + 感伤/低心绪/特定人物专属性格。

禁止普通自主：

- 题帕题笺：涉及对象和私密关系。
- 诗社命题：需要组织者、时间和多人事件。
- 写险诗：只能由极端心绪、剧情或玩家主动触发。

对应配置：

```js
{ id: 'poetry_desperate', ai: { allowAutonomous: false } }
```

## 13. 后台配置

后台【主题玩法包】里，写诗包需要能配：

- 动作表：动作名、挂载家具、时长、AI 权重、是否允许自主。
- 技能收益：每个动作给哪些技能经验。
- 状态结果：成功/失败/低心绪对应状态。
- 质量公式参数：技能权重、状态权重、随机波动。
- 作品标签：花、月、雪、病、离别、宴饮、题帕、诗社。
- 诗社模板：命题、限韵、参与人数、评分方式。

P0：日常写诗包

- 已完成：配置 `activityPacks.poetryWriting`。
- 已完成：上 4 个动作：抄录旧诗、起兴得句、即景题诗、苦吟推敲。
- 已完成：写入技能经验、需求变化、得句/苦吟/败兴状态。
- 已完成：高戏剧动作尚未配置进代码，因此不会进入普通 AI。

P1：作品系统

- 新增轻量 `WorkSystem` 或 `CreativeWorkSystem`。
- 写诗完成后生成作品记录。
- 作品质量进入人物记忆、前台人物面板和才名。

P2：诗社

- 联句唱和、诗社命题、多人评分。
- 支持组织者、命题、限时、排名、称赞/嘲笑。
- 接入关系、声望、梦想。

P3：LLM 文本

- 规则层继续输出同一套作品结构。
- LLM 层只负责按结构生成诗句/题名/评语。
- 保持可解释：质量分、状态、关系变化仍由规则决定。

## 15. 验收标准

- 点击书案可以看到至少 4 个写诗相关动作。
- 普通 NPC 会偶尔抄诗/起兴，但不会随机写险诗。
- 黛玉、湘云、宝钗等角色因性格不同，写诗动作权重明显不同。
- 苦吟会消耗精力或心绪，不是无成本刷技能。
- 一次高质量诗作能进入日志/记忆，后续可被关系、梦想、才名系统读取。
- 诗社动作不会在无人同场时出现或被 AI 乱选。
