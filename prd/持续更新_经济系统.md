> 当前原则：先完成“上班挣钱 → 家庭入账 → 发月银 → 吃饭扣钱”的最小闭环，其余内容均为后续设计。

## 0. 当前已实现

### 0.1 账户口径

- 工资、月银、饭钱全部使用 `FamilySystem` 的家庭公账。
- 角色原有 `money` 个人私房钱仍保留，但本轮不参与吃饭、工资和月银。
- `MoneySystem.getFamilyFund/changeFamilyFund` 已统一代理到 `FamilySystem`，不再修改配置对象中的静态 `fund`。
- 旧的笼统“每日开销”已设为 `0`，当前唯一固定消费是角色实际完成进食行为后的饭钱。

### 0.2 每日上班与工资

| 角色 | 上班 | 下班 | 工作 | 每日工资 | 入账家庭 |
| --- | ---: | ---: | --- | ---: | --- |
| 贾政 | 07:00 | 17:00 | 衙署当值 | 30 两 | 荣禧堂 |
| 贾琏 | 08:00 | 17:00 | 外务办差 | 20 两 | 协理院 |
| 贾珍 | 08:00 | 16:00 | 料理府务 | 24 两 | 宁国府 |
| 来旺 | 07:00 | 18:00 | 在外跑腿 | 8 两 | 协理院 |

结算规则：

1. 到上班时辰，角色获得当值状态，并将工作案几作为高权重 AI 行为。
2. 到下班时辰，仅当天确实开始过当值的角色结算一次工资。
3. 工资直接进入角色所属家庭公账，不进入个人私房钱。
4. 下班后角色排队返回所属家庭住所。
5. 当值状态、当日是否结薪会进入存档，防止读档重复发钱。

### 0.3 月银

游戏按每 30 日为一个月，每月初一自动执行配置表：

| 发放人 | 来源家庭 | 目标家庭 | 金额 |
| --- | --- | --- | ---: |
| 王夫人 | 荣禧堂 | 怡红院 | 30 两 |
| 王夫人 | 荣禧堂 | 潇湘馆 | 24 两 |
| 凤姐 | 协理院 | 蘅芜苑 | 18 两 |
| 凤姐 | 协理院 | 秋爽斋 | 15 两 |
| 凤姐 | 协理院 | 刘家庄 | 6 两 |

- 月银是家庭公账之间的真实转账，发放方余额不足则该笔不执行。
- 同一游戏月只自动结算一次，结算标记进入存档。
- 对外接口：`EconomySystem.distributeAllowance(issuerId, targetFamilyId, amount, note)`。
- 当前只允许 `wangfuren` 和 `xifeng` 调用发放接口。

### 0.4 饭钱

| 家具类别 | 例子 | 每次完成扣款 |
| --- | --- | ---: |
| `meal` | 饭桌 | 3 两 |
| `snack` | 点心案 | 1 两 |
| `kitchen` | 厨房灶台 | 2 两 |
| `table` | 茶案等 | 1 两 |

- 使用前检查角色所属家庭公账。
- 余额不足时，家具不可使用，并显示所需金额。
- 完成进食后才扣款；中途取消或行动失败不收费。
- 每次扣款写入家庭流水，并发出 `economy:food_paid` 事件。

### 0.5 埋点

行为埋点新增：

- `economy:shift_started`
- `economy:shift_ended`
- `economy:food_paid`
- `economy:allowance_paid`

仍遵循 `BehaviorTelemetry` 的观察角色名单；默认只记录黛玉，需要分析其他角色时先调用 `BehaviorTelemetry.watch(charId, true)`。

### 0.6 第一版数值思路

- 数值只追求闭环可运行，不追求历史银价还原。
- 一顿正餐 3 两，普通角色一日三餐约消耗 9 两。
- 贾政日工资 30 两，可以覆盖一个小家庭的基本饮食。
- 贾琏与来旺合计每日 28 两，使协理院既能吃饭也能承担少量月银。
- 月银显著低于月度工资总额，先避免发放方快速破产。

### 0.7 已知风险与后续整理项

- 当前工作地点仍在现有地图内，用“工作案几”表现，没有独立衙署或外务场景。
- 当值期间 AI 会强烈偏好办公，但紧急需求、玩家队列和互动仍可能打断工作。
- 工资只按是否开始当值结算，暂未统计迟到、早退和实际工作时长。
- 饭钱按家具类别收费，没有区分主仆、菜品、宴席和食材库存。
- 月银目前发到家庭公账，不是逐人发到个人私房钱。
- 家庭流水只保留最近 20 条，长期经济分析应另做专门账本或扩大存储。
- 当前没有欠账、赊饭、物价波动、田庄收入、税费、采购和财务危机。
- 后续调数应优先观察“每日家庭净变化、每人进食次数、月银失败率、家庭归零天数”。

## 1. 后续通用框架

以下章节是长期 PRD，除上方“当前已实现”明确列出的部分外，均不代表已经上线。

## 1. 概述

金钱系统是支撑角色经济行为的基础模块，驱动大观园内外的一切交易、赏赐、投资与贪贿行为。本系统设计为**通用框架**，支持个人私库与家族公库双线并行，同时预留放贷、经营、投资等扩展接口，后续可接入商贾路径、科举贿赂、选秀打点等场景。

**核心原则：**
- **双线并行**：个人账户（私房钱）与家族账户（府库公银）独立运作，各有收支规则。
- **配置驱动**：所有价格、薪水、利率、罚金均由数据表定义，易于调整经济平衡。
- **来源多样**：月钱、赏赐、任务报酬、放贷利息、贪贿所得、经营收入等。
- **消耗明确**：购物、打赏、投资、贿赂、罚金、日常消费等。
- **风险机制**：借贷有坏账风险，贪贿有被查风险，过度消费有亏空风险。

## 2. 数据结构

### 2.1 账户类型

| 类型 | 说明 | 持有人 | 收支来源 |
|------|------|--------|----------|
| 个人私库 | 角色的私人积蓄 | 每个角色独立 | 月钱、赏赐、任务报酬、放贷、贪贿 |
| 家族公库 | 贾府公共银两 | 家庭为单位 | 家族收入（田庄、俸禄）、支出（月钱发放、宴席、修缮） |
| 店铺账户 | 商贾路径角色拥有 | 店铺主人 | 经营收入、进货支出 |

### 2.2 个人账户数据结构

```js
PersonalAccount {
  character_id: int,
  balance: float,              // 当前余额
  total_income: float,         // 累计收入
  total_expense: float,        // 累计支出
  monthly_salary: float,       // 月钱（由身份决定）
  loans_out: [Loan],           // 放出的贷款
  debts: [Debt],               // 自己的欠债
  transaction_log: [Transaction],  // 最近流水
}
```

### 2.3 家族账户数据结构

```js
FamilyAccount {
  family_id: int,
  balance: float,
  monthly_income: float,       // 固定月收入（田庄、房租、俸禄）
  monthly_expense: float,      // 固定月支出（月钱、饮食、修缮）
  emergency_fund: float,       // 应急储备（低于此值触发警告）
  investments: [Investment],   // 投资记录
  transaction_log: [Transaction],
}
```

### 2.4 交易记录

```js
Transaction {
  id: string,
  timestamp: float,
  type: enum,           // income/expense/loan/debt/transfer
  amount: float,
  source: string,       // 来源描述（如“月钱发放”“赏赐-贾母”）
  target_account: enum, // personal/family/shop
  related_character_id: int,
  note: string,
}
```

## 5. 核心机制

### 5.1 月钱发放

每月初一自动触发，由系统从家族公库中扣除总额，分别打入各角色个人账户。若公库余额不足，优先发放主子，仆从可能被拖欠（引发不满状态）。

### 5.2 交易系统（购买/贩卖）

玩家进入商店场景（预留）或与货郎 NPC 互动时，打开交易界面。物品价格由 `ItemTemplate.price` 决定，支持议价（需口才技能判定）。购买后物品进入角色背包（预留物品系统）。

### 5.3 放贷机制

- **放出贷款**：通过心腹 NPC（如来旺）将银两借给他人，约定利息和期限。
- **收回**：到期后借款人需归还本息。若对方无力偿还，可延期、逼债（降低关系/声望）或坏账。
- **风险**：借款人可能赖账（由其信任值和关系决定），无抵押品。

### 5.4 贪贿与风险

- **收受贿赂**：在特定事件中接受他人金钱，同时增加“把柄”计数。
- **风险累积**：把柄≥3时，御史弹劾概率大幅增加，触发抄家事件链。
- **打点**：可通过贿赂官员降低风险，形成“贪→打点→再贪”循环。

### 5.5 家族危机机制

- 家族公库余额低于警戒线时，触发“财务危机”状态。
- 危机状态下，全府仆从服从下降，部分任务无法执行。
- 需通过削减开支、增加收入（如变卖家产）来缓解。

## 6. 与现有系统集成

| 系统 | 集成方式 |
|------|----------|
| 任务系统 | 任务报酬自动打入个人账户；任务可能要求花费金钱 |
| 关系系统 | 金钱往来影响信任（借债、送礼）、服从（克扣月钱） |
| 身份系统 | 身份决定月钱基准和消费水平；晋升后月钱提升 |
| 声望系统 | 挥霍/吝啬影响声望；救济穷人增加声望 |
| 物品系统 | 购买/贩卖物品需要金钱；物品有价格属性 |
| 场景系统 | 商店、当铺、银号等经济场景 |
