# AI 精灵生图工作流

更新时间：2026-06-18

## 1. 目标

把人物立绘稳定转成游戏可用的 64x64 帧精灵表，解决三个问题：

- 不要每帧都像“另一个人”。
- 不要过度 Q 版，保留更修长、更古风的人物比例。
- 能批量覆盖待机、走路、用手、擦拭、休息等游戏动作。

本流程的核心原则：

> AI 只负责“给同一套裸模穿上同一个人的衣服”，不能让 AI 重新设计姿势、比例和动作。

## 3. 三层模板

### 3.1 动作裸模模板

职责：

- 决定姿势、朝向、肢体动作、脚底锚点和主体大小。
- 每帧都必须严格继承裸模轮廓。
- 不允许 AI 自己改动作。

输入：

- 单帧：`sprite_r04_c03.png`
- 动作条：例如 `r11 c01-c09`
- 完整裸模 sheet：`female copy.png`

### 3.2 角色 Style Bible

每个角色先生成/整理一份角色小人设定，而不是直接全量出动画。

字段：

| 字段 | 说明 |
|---|---|
| `charId` | 角色 ID，例如 `zijuan` |
| `portrait` | 立绘参考 |
| `bodyRatio` | 头身比，建议 `1:2.2` 到 `1:2.5`，避免过 Q |
| `hairSilhouette` | 发型剪影 |
| `mainColors` | 主色、辅色、点缀色 |
| `fixedDetails` | 必须每帧保持的识别点 |
| `allowedMotion` | 袖摆、裙摆、发梢允许随动作变化 |
| `forbidden` | 禁止换发色、换腰带位置、换衣服大形、额外道具 |

紫鹃示例：

```json
{
  "charId": "zijuan",
  "bodyRatio": "1:2.3",
  "hairSilhouette": "black hair, neat maid hairstyle, small hair ornament, soft side locks",
  "mainColors": {
    "outerRobe": "pale blue green",
    "innerSkirt": "light warm pink",
    "waistSash": "muted lavender gray",
    "hair": "near black"
  },
  "fixedDetails": [
    "pale blue-green hanfu outer robe",
    "light pink inner skirt visible below robe",
    "muted lavender-gray waist sash",
    "small delicate hair ornament",
    "refined but simple servant styling"
  ],
  "allowedMotion": [
    "sleeves may swing slightly",
    "skirt hem may shift with walking",
    "hair tips may move one or two pixels"
  ],
  "forbidden": [
    "do not make the body chibi or toddler-like",
    "do not change outfit colors between frames",
    "do not add weapons, fans, umbrellas, pets, text, watermark, scenery"
  ]
}
```

### 3.3 生成版式模板

不要一帧一帧完全独立生成。推荐小批量：

1. 先生成 `down idle + down walk 8 帧`。
2. 再生成 `left/right/up walk`。
3. 再生成 `useHands`、`useWipe`、`rest`。
4. 最后拼成完整 `832x1344` sheet。

每次生成必须包含：

- 同一个角色 Style Bible。
- 同一条裸模动作参考。
- 明确的输出尺寸和网格。
- 明确的禁止项。

## 4. 推荐试验顺序

### Pilot 1：向下待机 + 行走

目的：验证一致性和比例。

输入：

- 裸模行：`r11`
- 帧：`c01-c09`
- 角色：紫鹃
- 输出：一张 `9 x 64`、即 `576 x 64` 的透明 PNG 动作条，或 9 张 `64 x 64` 单帧。

验收：

- 9 帧脚底 y 基本一致。
- 人物高度在 42-56 px 之间。
- 头身比不要低于 `1:2`。
- 主色不漂。
- 衣带、发型、裙摆识别点不跳。

### Pilot 2：四方向行走

输入：

- `r09-r12`
- 每行 `c01-c09`

输出：

- `4 x 9` 动作格。

验收：

- 四方向发型、腰带、裙色一致。
- 左右方向可以镜像风格，但不能变成不同服装。

### Pilot 3：衣食住行动作

当前可先映射：

| 游戏动作 | 动画 clip | 对应家具/行为 |
|---|---|---|
| 待机 | `idle` | 空闲、站立 |
| 走路 | `walk` | 移动 |
| 用手 | `useHands` | 吃饭、读书、抚琴、换衣 |
| 擦拭 | `useWipe` | 打扫、梳洗、浇花 |
| 休息 | `rest` | 睡觉、坐榻、亭中小憩 |

后续如果要更细，可以增加：

- `eat`
- `read`
- `playQin`
- `sweep`
- `weep`
- `bow`
- `serveTea`
- `carryTray`

但第一版不要急着扩动作，否则一致性问题会放大。

## 5. 生图 Prompt 模板

### 5.1 单行动作条 Prompt

```text
Use case: style-transfer
Asset type: RPG pixel character animation strip

Primary request:
Create a transparent-background pixel-art animation strip for {charName}, using the provided nude/base sprite strip only as the exact pose, body silhouette, frame count, frame size, foot anchor, and motion reference. Dress the base sprite as the character from the portrait/style bible.

Input images:
Image 1: base sprite strip, exact pose and frame layout reference.
Image 2: character portrait/style reference.

Output format:
{frameCount} frames in one horizontal strip, each frame exactly 64x64 pixels, total size {frameCount*64}x64.
Transparent background.
Pixel-art game sprite, crisp edges, no antialias blur.

Character identity:
{styleBible}

Pose and motion constraints:
Preserve the pose, body silhouette, head position, limb position, and foot anchor of Image 1 in every frame.
Do not invent new poses.
Do not change the animation timing.

Style constraints:
Not toddler-like, not super-deformed chibi.
Elegant ancient Chinese game sprite proportions, about 1:2.2 to 1:2.5 head-to-body ratio.
Readable at 48x48 in game.

Avoid:
No background, no scenery, no props unless present in the base pose, no text, no watermark.
Do not change outfit colors between frames.
Do not make each frame a different costume.
```

### 5.2 单帧 Prompt

只用于打样，不推荐大量单帧独立生成。

```text
Create one 64x64 transparent-background pixel-art game sprite.
Use Image 1 as exact pose/silhouette/anchor reference.
Use Image 2 only for character appearance.
Dress the base sprite as {charName}.
Keep the same foot anchor, body size, and stance as Image 1.
Avoid chibi toddler proportions; use elegant ancient Chinese sprite proportions.
```

## 6. 质量检查

每次生成后检查：

| 检查项 | 通过标准 |
|---|---|
| 尺寸 | 每帧 `64x64`，sheet 与 manifest 一致 |
| 透明底 | 背景必须透明 |
| 脚底锚点 | 同一动作条脚底 y 误差建议 <= 2 px |
| 主体高度 | 同一角色不同帧高度误差建议 <= 4 px |
| 主色 | 主色漂移不能明显 |
| 轮廓 | 不能超出裸模太多，袖摆可适度放大 |
| 识别点 | 发型、腰带、衣服主色稳定 |
| 游戏预览 | 48x48 绘制时仍能认出角色 |

## 7. 入库方案

推荐目录：

```text
assets/sprites/source/
  zijuan/
    style-bible.json
    refs/
      portrait.png
      base-r11.png
    generated/
      pilot-down-walk.png
      pilot-down-walk.frames/
    final/
      zijuan.png
      report.json
```

最终接入 `assets/manifest.json`：

```json
"sheets": {
  "zijuan": {
    "src": "assets/sprites/chars/zijuan.png"
  }
}
```

并更新 `js/assets.js` 的角色映射：

```js
const CHAR_SHEET = { daiyu: 'daiyu', zijuan: 'zijuan' };
```

## 8. 脚本

新增：

```bash
python3 scripts/sprite_workflow.py inspect
python3 scripts/sprite_workflow.py plan --char zijuan --portrait assets/lihui/紫鹃.png --preset pilot-down
```

脚本职责：

- 读取裸模 `meta.json` 和 `index.txt`。
- 输出动作行与文件映射。
- 生成 prompt 包到 `reports/sprite_workflow/`。
- 检查 PNG 尺寸和 sheet 尺寸。

注意：当前脚本不直接调用生图模型。生图仍通过 imagegen 或外部工具完成；脚本负责把输入和验收标准固定下来。
