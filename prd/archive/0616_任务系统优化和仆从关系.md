> 更新日期：2026-06-16
> 合并来源：`0616_任务系统优化和仆从关系.md`、`0616_任务和仆从和跟随2.md`
> 当前状态：任务管理、仆从职责、任务质量、跟随轮值第一版已落地；跨场景传唤、后台表格化配置和更完整的任务调度仍待做。

## 一、目标

把任务系统从“有任务列表”推进到“能解释、能差遣、能长期影响关系”的家内事务系统。

核心目标：

- 主子可以向直属仆从下达职责内任务。
- 仆从是否接受、如何执行、执行质量如何，都能由身份、服从、信任、好感、负担和状态解释。
- 日常职责、随侍跟随、传话、洒扫、陪伴等都统一为任务模板。
- 黛玉测试时可以通过任务面板和埋点还原“一整天做了什么，为什么这样做”。
- 不另起新系统，复用 `QuestSystem`、`QuestIssueSystem`、`ServantRelationSystem`、AI 候选池、关系系统和底部 UI。

## 三、当前系统形态

### 3.1 任务来源

| 类型 | 用途 | 当前实现 |
| --- | --- | --- |
| 日常任务 | 固定时间执行读书、洒扫、服侍等日课 | 按每日时间槽或职责契约生成。 |
| 随侍轮值 | 贴身仆从按日跟随主子 | 由 `followRotations` 生成。 |
| 人物差遣 | 有权限的人向指定人物下发任务 | 可接受、拒绝、撤回。 |
| 群体任务 | 向符合身份条件的一组人物传令 | 支持批次与群体筛选。 |
| 剧情任务 | 由故事事件触发 | 可绑定前置任务和剧情结果。 |
| AI 差遣 | NPC 根据身份、性格和概率主动派任务 | 定时寻找候选任务并下发。 |

### 3.2 任务生命周期

```text
生成任务
  ↓
待接受 PENDING
  ├─ 接受 → ACCEPTED → 推进中 → COMPLETED / FAILED
  ├─ 拒绝 → DECLINED
  ├─ 发布者撤回 → REVOKED
  └─ 超时未处理 → EXPIRED
```

建议后续扩展但尚未完整实现的状态：

```text
IN_PROGRESS   明确进行中
PAUSED        被更高优先级行为打断
BLOCKED       缺少目标、家具、技能或路径
```

现在已有阻塞原因字段，但还没有把 `PAUSED` / `BLOCKED` 完整作为任务状态流转。

### 3.3 当前进度条件

| 条件 | 含义 |
| --- | --- |
| `USE_FURNITURE_DURATION` | 使用指定家具累计一定游戏时间。 |
| `USE_FURNITURE` | 完成一次指定家具行为。 |
| `INTERACT_WITH` | 与指定人物完成一次互动。 |
| `INTERACT_WITH_DURATION` | 与指定人物持续互动一定时间。 |
| `FOLLOW_CHARACTER` | 与目标处于同一场景且距离足够近后持续跟随；普通跟随默认 2.5 格内，`唤来报到` 默认 1.5 格内。 |
| `STAY_IN_SCENE` | 在指定场景停留一定时间。 |
| `AVOID_SCENE` | 进入禁区则失败。 |
| `AVOID_CHARACTER` | 接近指定人物则失败。 |
| `ENTER_SCENE` | 进入指定场景则失败。 |

### 3.4 派发权限

| 等级 | 典型身份 | 当前可支配关系 |
| --- | --- | --- |
| 0 | 老祖宗 | 晚辈、子女、仆从，可群体传令。 |
| 1 | 家主、主母 | 子女、仆从，可群体传令。 |
| 2 | 少爷、小姐 | 自己的仆从。 |
| 3 | 普通主子 | 自己的仆从。 |
| 4 | 管事、高等仆从 | 低等仆从，可群体安排杂务。 |
| 5 | 普通仆从 | 默认无派发权限。 |
| 6 | 外人 | 默认不参与家内支配关系。 |

身份关系包括：

- `master_to_servant`：主子对仆从
- `servant_to_master`：仆从对主子
- `senior_servant_to_junior`：高等仆从对低等仆从
- `junior_to_senior_servant`：低等仆从对高等仆从
- `parent_to_child` / `child_to_parent`
- `grandparent_to_child`
- `peer`
- `outsider`

## 四、仆从关系模型

### 4.1 职责契约

`servantConfig.contracts` 用来明确主仆服务关系。

```json
{
  "id": "daiyu_zijuan",
  "masterId": "daiyu",
  "servantId": "zijuan",
  "role": "贴身侍女",
  "scope": "personal",
  "authorityLevel": 80,
  "dutyCategories": ["起居", "陪伴", "传话", "取物", "照看", "跟随"],
  "sceneIds": ["xiaoxiang"],
  "salaryPerMonth": 200
}
```

`scope` 建议含义：

| scope | 含义 |
| --- | --- |
| `personal` | 只对直接主子负责。 |
| `household` | 可被家庭主母或管事调动。 |
| `yard` | 负责特定院落。 |
| `outer` | 外院、采购、传递等事务。 |

### 4.2 关系数值

| 数值 | 含义 | 当前状态 |
| --- | --- | --- |
| 好感 `affection` | 私人情感，喜欢或厌恶。 | 已有。 |
| 信任 `trust` | 是否相信对方公平可靠。 | 已有。 |
| 友谊 `friendship` | 亲近程度。 | 已有。 |
| 服从 `submission` | A 对 B 的服从程度。 | 已有，并已进入任务接受和质量。 |
| 忠诚 | 是否维护主子利益。 | 仆从运行态已做。 |
| 积怨 | 长期负面累积。 | 仆从运行态已做。 |
| 劳役压力 | 当前任务负担与过劳。 | 仆从运行态已做。 |

### 4.3 典型关系状态

| 状态 | 条件示例 | 行为 |
| --- | --- | --- |
| 忠心耿耿 | 忠诚、信任、服从均高 | 主动补位，保护主子。 |
| 亲近逾矩 | 好感高、服从低 | 关心主子但可能没大没小。 |
| 畏服 | 服从高、好感低 | 执行命令，但容易积怨。 |
| 敷衍塞责 | 服从、信任均低 | 拖延或低质量完成。 |
| 心怀怨怼 | 积怨高 | 私下抱怨，可能泄密。 |
| 不堪劳役 | 劳役压力高 | 疲倦、出错、拒绝额外任务。 |
| 得脸受宠 | 好感与信任高 | 获得更高权限或月银。 |

## 五、跟随与差遣流程

### 5.1 跟随轮值

随侍不再写死为单个仆从的固定日课，而是由 `servantConfig.followRotations` 配置。

```json
{
  "masterId": "baoyu",
  "templateId": 2002,
  "triggerHour": 7,
  "rotation": [
    { "servantId": "xiren", "weekday": 1 },
    { "servantId": "sheyue", "weekday": 2 },
    { "servantId": "qingwen", "weekday": 3 }
  ]
}
```

当前默认：

| 主子 | 当值仆从 |
| --- | --- |
| 宝玉 | 袭人、麝月、晴雯轮值。 |
| 黛玉 | 紫鹃、雪雁轮值。 |

### 5.2 同场景差遣

```text
玩家点击仆从
  ↓
互动菜单“差遣”
  ↓
选择可用任务模板
  ↓
生成 QuestInstance
  ↓
仆从接受或拒绝
  ↓
AI 根据任务权重推进
```

### 5.3 跨场景唤来

目标流程：

```text
玩家点击不在同场景的人
  ↓
选择“唤来”
  ↓
系统给当前跟随者生成传话任务
  ↓
跟随者去叫目标
  ↓
目标收到“被召唤/报到”任务
  ↓
目标前来回话
```

当前只完成了 `唤来报到` 任务模板，完整 UI 入口和传话执行链还未做。

### 5.4 跟随被基础需求打断

跟随中饥、洁、倦过低时，仆从可以临时离开。

| 条件 | 阈值 |
| --- | --- |
| 默认 | 20% |
| 高服从 | 10% |
| 高信任主子 | 25% |

触发时：

- 头上出气泡：“我去去就来。”
- 发出 `servant:follow_interrupted`。
- AI 会优先解决需求。
- 解决后因为跟随任务仍有权重，会回到主子附近。

显式“请辞中 / 归队中”状态机尚未做。

## 六、任务质量与结算

完成任务后计算 0-100 质量分。

```text
质量 =
  技能
  + 及时性
  + 正确目标与家具
  + 服从修正
  + 心绪修正
  - 疲倦
  - 阻塞或拖延
```

| 质量 | 结果 | 影响 |
| --- | --- | --- |
| 85-100 | 出色 | 信任、好感、能力经验上升。 |
| 60-84 | 合格 | 正常结算。 |
| 30-59 | 敷衍 | 主子可训斥，仆从可能积怨。 |
| 0-29 | 失败 | 任务失败或产生负面事件。 |

## 十一、待确认设计

- 仆从职责是严格绑定个人主子，还是允许家庭主母统一调度。
- 玩家是否可以强制执行被拒绝的命令，以及强制命令的关系代价。
- 日课是否全部显示为任务，还是低价值杂务只记录在后台。
- 仆从月银由直接主子、家庭公账还是管事统一发放。
- 忠诚和积怨后续是否继续独立存储，还是部分由关系轴推导。
- 任务质量是否直接影响场景洁净、饭食品质和家具效果。
- 跟随是否需要明确“当值时间段”和“休息时间段”，还是先按整日轮值处理。
