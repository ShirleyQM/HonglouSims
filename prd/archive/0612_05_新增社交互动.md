# 社交互动扩展：亲密度解锁与自由交互设计

## 1. 设计目标
- **亲密度驱动解锁**：互动不再全部开放，而是根据双方关系四轴（尤其是好感、亲密、信任）动态显示。
- **大幅提升自由度**：增加更多身体接触、情感表达动作，让角色互动更细腻。
- **身份与逾矩保留**：高风险行为仍受礼法约束，失败或目击会引发连锁反应。
- **配置化**：所有新增互动均为模板，可自由增删、调整条件。

## 2. 互动模板扩展字段
在现有互动模板基础上增加以下字段，以支持条件解锁和身份判定：

| 字段 | 类型 | 说明 |
|------|------|------|
| `unlock_conditions` | object | 解琐条件，包含亲密度要求、身份条件等 |
| `unlock_conditions.min_score` | int | 最低综合分（-100~100） |
| `unlock_conditions.min_affection` | int | 最低好感 |
| `unlock_conditions.min_intimacy` | int | 最低亲密 |
| `unlock_conditions.min_trust` | int | 最低信任 |
| `unlock_conditions.require_status` | string | 自身或对方需要持有的状态ID，如“心动” |
| `is_risky` | bool | 是否为逾矩行为（需走风险判定） |
| `risky_details` | object | 若`is_risky=true`，定义风险配置 |
| `risky_details.base_success_rate` | float | 基础成功率 |
| `risky_details.witness_penalty` | float | 旁观者惩罚 |
| `risky_details.fail_status` | string | 失败后施加的状态 |
| `risky_details.success_status` | string | 成功后额外状态 |
| `contact_type` | enum | `none` / `touch` / `embrace` / `kiss` 等身体接触等级 |
| `gender_constraint` | string | `any` / `male_female_only` / `same_gender_only` |
| `age_constraint` | enum | `any` / `peer_only`（仅平辈） / `senior_junior_only`（需辈分差） |
| `effects` | list | 关系四轴变化、需求变化、状态变化 |

## 4. 与身份系统的集成
- 所有身体接触互动在 `IdentityRulesConfig` 中根据 `contact_type` 和 `hierarchyRelation` 自动判定是否为逾矩。例如，`kiss` 类型在 `servant_to_master` 下始终逾矩，`embrace` 在 `peer` 间正常但在 `parent_to_child` 可能为关爱。
- 逾矩行为自动调用风险流程，成功与失败效果均配置在模板中。
- 界面上，逾矩互动旁显示⚠️标记，悬停提示风险详情。

## 5. 动态菜单展示
- 玩家点击目标后，系统根据双方当前关系值、状态、身份关系，**过滤出已解锁的互动**。
- 未解锁互动不显示（或灰显并显示解锁条件，如“需亲密50”）。
- 互动按类别分组：叙旧、论道、传情、慰藉、嬉闹等，逾矩互动在组内靠后排列但可加亮。
- 高亲密时，默认推荐的几个互动会出现在快捷气泡中（样式四），方便一键执行。

## 6. 扩展性与配置
所有新互动均作为 `InteractionTemplate` 条目存入数据库/JSON配置，策划可通过编辑器随时增删、调整亲密度门槛和效果，无需改动代码。

示例配置（拥抱）：
```json
{
  "id": "embrace",
  "category": "叙旧",
  "unlock_conditions": {
    "min_score": 60,
    "min_intimacy": 40
  },
  "contact_type": "embrace",
  "is_risky": false,
  "risky_details": null,
  "gender_constraint": "any",
  "age_constraint": "any",
  "effects": [
    { "axis": "intimacy", "delta": 10 },
    { "axis": "affection", "delta": 8 },
    { "system": "status", "status": "温馨", "duration": 60 }
  ]
}
```
身份越界时，`is_risky` 会在运行时被系统覆盖为 `true`，并套用默认风险配置（或模板内自定义的 risky_details）。

---

通过此设计，玩家可在亲密度足够时逐步解锁丰富的肢体语言和情感表达，大幅增强角色扮演的自由度与沉浸感，同时保留身份礼法的戏剧张力。
