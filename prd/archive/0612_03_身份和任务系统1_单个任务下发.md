# 身份分层任务下发系统 PRD

## 1. 概述

本系统将大观园中的任务下发权限与角色的**社会等级**、**身份关系类型**绑定，形成一个通用的分层指令框架。不再为某个主子单独写死任务逻辑，而是通过**身份层级**和**任务模板**的组合，允许任何符合条件的角色向符合条件的对象下发任务。系统支持：
- 长辈向晚辈、主子向仆从、体面仆从向普通仆从等常规指令。
- 特殊身份（至尊、执事）的跨级权限。
- 任务类型与身份层级的自动匹配（如长辈可罚抄书，仆从只能执行洒扫）。
- 玩家操控任意角色时，均可通过界面选择目标并下发符合身份的任务。

---

## 2. 身份层级与下发权限矩阵

### 2.1 社会等级（SocialRank）
沿用既有设计：0至尊、1长辈、2平辈主子、3晚辈主子、4体面仆从、5普通仆从、6外人。

### 2.2 身份关系类型（HierarchyRelation）
用于判定具体互动权限：
- `grandparent_to_any`：贾母对任何人（除长辈外）
- `parent_to_child`：长辈对晚辈主子
- `master_to_servant`：主子对仆人
- `senior_servant_to_junior`：体面仆从对普通仆从
- `peer`：平级（主子间、仆从间）
- `junior_servant_to_senior`：普通仆从对体面仆从（逆向，权限受限）

### 2.3 下发权限配置表（QuestIssuePermissionConfig）

每个身份层级对应一行，定义该层级可对**哪些身份关系类型**下发任务，以及可下发的任务类别。

```js
QuestIssuePermissionConfig = [
  {
    issuer_rank: 0,              // 至尊（贾母）
    target_relations: ['grandparent_to_any', 'parent_to_child', 'master_to_servant'],
    // 可跨越常规层级：直接对晚辈主子、仆从下发，甚至可对长辈（但极少）
    allowed_categories: ['功课','禁足','侍奉','跑腿','陪伴','传话','采办','家宴','训诫'],
    special_quests: ['family_event', 'summon'],   // 可发起家族事件、召见
  },
  {
    issuer_rank: 1,              // 长辈（贾政、王夫人）
    target_relations: ['parent_to_child', 'master_to_servant'],
    allowed_categories: ['功课','禁足','侍奉','跑腿','训诫'],
    special_quests: ['inspect'], // 可下令检查某人物或场景
  },
  {
    issuer_rank: 2,              // 平辈主子（宝玉、黛玉、宝钗等）
    target_relations: ['master_to_servant'],
    allowed_categories: ['侍奉','跑腿','陪伴','传话','采办','诗社'],
    special_quests: [],
  },
  {
    issuer_rank: 3,              // 晚辈主子（探春、惜春等）
    target_relations: ['master_to_servant'],
    allowed_categories: ['侍奉','跑腿','陪伴','传话'],   // 可下发的类别更少
    special_quests: [],
  },
  {
    issuer_rank: 4,              // 体面仆从（袭人、平儿、紫鹃）
    target_relations: ['senior_servant_to_junior'],      // 只能对普通仆从
    allowed_categories: ['洒扫','跑腿','传话','采办','照看'],
    special_quests: ['report'],  // 可向主子汇报（特殊任务）
  },
  {
    issuer_rank: 5,              // 普通仆从
    target_relations: [],        // 不能下发任务（除非特殊事件授权）
    allowed_categories: [],
    special_quests: [],
  },
];
```

> `allowed_categories` 中的值对应 `QuestTemplate.category`，确保任务模板的分类与权限匹配。

## 3. 任务模板通用化设计

所有任务均基于 `QuestTemplate`，不绑定具体下发者，只绑定**身份关系要求**。下发时由系统根据当前两人的身份关系自动判定能否使用该模板。

### 3.1 模板扩展字段
在原有 `QuestTemplate` 基础上增加：
```js
{
  issuer_relation_required: ['parent_to_child', 'master_to_servant'],  // 允许下发此任务的身份关系
  target_rank_range: { min: 0, max: 5 },   // 目标身份等级范围（可选）
  category: '侍奉',                         // 任务分类（用于权限匹配）
  is_special: false,                        // 是否为特殊任务（需要额外条件）
}
```

### 3.2 通用任务模板示例
| 模板ID | 名称 | 类别 | 下发关系要求 | 目标等级 | 说明 |
|--------|------|------|-------------|---------|------|
| Q001 | 晨昏定省 | 侍奉 | parent_to_child, master_to_servant | 0-5 | 晚辈向长辈请安 |
| Q002 | 罚抄书 | 功课 | parent_to_child | 2-3 | 长辈罚晚辈抄书 |
| Q003 | 禁足 | 禁足 | parent_to_child, master_to_servant | 2-5 | 长辈或主子限制对象自由 |
| Q004 | 跑腿买办 | 采办 | master_to_servant, senior_servant_to_junior | 4-5 | 主子或体面仆从差遣外出购物 |
| Q005 | 洒扫庭院 | 洒扫 | master_to_servant, senior_servant_to_junior | 4-5 | 日常清洁任务 |
| Q006 | 伴读 | 陪伴 | parent_to_child, master_to_servant | 2-5 | 陪主子读书 |
| Q007 | 传话 | 传话 | master_to_servant, senior_servant_to_junior | 4-5 | 传递口信 |
| Q008 | 备膳 | 侍奉 | master_to_servant | 4-5 | 准备饭食 |
| Q009 | 训诫 | 训诫 | parent_to_child, master_to_servant | 0-5 | 言语教训（非惩罚） |
| Q010 | 查账 | 特殊 | parent_to_child (长辈), master_to_servant (体面仆从) | 4 | 长辈让体面仆从查账 |

特殊任务（如“家宴筹备”）可设为 `is_special:true`，需额外检查 `special_quests` 权限。

## 4. 任务下发流程

### 4.1 玩家操控角色下发
1. 玩家点击目标人物，菜单中出现“差遣”或“任务”选项（根据双方身份关系动态生成）。
2. 系统查询 `QuestIssuePermissionConfig` 中当前操控角色的权限，过滤出可用的 `allowed_categories`。
3. 从 `QuestTemplate` 表中筛选满足以下条件的模板：
   - `category` 在允许列表中；
   - `issuer_relation_required` 包含当前两人身份关系；
   - 目标角色等级在 `target_rank_range` 内；
   - 若为特殊任务，检查 `special_quests` 权限。
4. 将可用任务以菜单形式展示，玩家选择后生成 `QuestInstance` 下发给目标。
5. 若玩家操控的角色不具备下发权限，菜单中不出现“差遣”选项（或灰显提示“身份不符”）。

### 4.2 AI自主下发
在AI决策中，具有下发权限的角色会根据需求（如长辈看晚辈不顺眼、主子需要服务）主动向符合条件的对象下发任务。流程类似，但由AI权重和冷却时间控制。

---

## 5. 任务接受与执行
- 指令任务默认强制接受（`auto_accept:true`），除非模板配置为可选（如某些陪伴类任务可拒绝，但会有惩罚）。
- 仆从的日常任务通过时间表自动生成，不属于下发流程，但保留主子可临时插入紧急任务并打断日常的机制。
- 任务进度追踪、奖励、惩罚沿用原任务系统。

---

## 6. 界面与交互

### 6.1 “差遣”菜单
当玩家操控角色点击可下发任务的目标时，弹出类似互动菜单的二级菜单，标题为“差遣”或“吩咐”，下方列出可用任务（附简要说明和时限）。若不可下发，不显示此菜单。

### 6.2 任务清单
目标角色的任务列表中会标注“下发者：某某”，并显示剩余时间。下发者可在任务面板中查看自己下发的任务状态，并可撤销（若未开始）。

### 6.3 身份不符提示
当玩家试图对不可下发任务的角色操作时，底部信息栏提示：“你的身份无法差遣此人”或“礼法不合”。

---

## 7. 扩展性设计
- **新增身份层级**：只需在 `SocialRank` 枚举中插入新等级，并添加对应的 `QuestIssuePermissionConfig` 行即可。
- **新增任务类别**：在 `QuestTemplate` 中增加模板，并更新相关权限行的 `allowed_categories`。
- **跨级授权**：某些特殊事件或关系（如“宠溺”状态）可临时赋予角色跨级下发权限，通过动态修改 `QuestIssuePermissionConfig` 的运行缓存实现。
- **与关系四轴联动**：若目标服从值极低，可能拒绝执行任务（即使强制接受，也会故意拖延），触发额外惩罚事件。

---

## 8. 总结
本系统将身份层级与任务模板解耦，通过一张权限矩阵和关系类型过滤，实现了通用的、可扩展的任务下发机制。无论是贾母举办家宴，还是袭人吩咐小丫鬟扫地，都通过同一套逻辑驱动，既符合古代礼法，又为玩家提供了清晰的操控反馈。
