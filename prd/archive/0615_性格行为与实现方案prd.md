## 性格、追求与动画表现扩展

要让每个角色真正“活”起来，需要三步并行：**细化性格**（让人物区分度更高），**定义个人追求**（给角色长远目标），**可视化行为**（脱离纯日志）。以下提供可落地的框架。

### 一、性格标签库（扩展版）

为方便“一个个钻研”，将原有性格标签拆分为**三类**：脾性（情绪反应）、社交（人际风格）、习性（生活习惯）。每个角色可组合多个，形成立体性格。

#### 1. 脾性（情绪与本能反应）
| 标签 | 说明 | 效果示例 |
|------|------|----------|
| 乐天 | 容易开心，难沮丧 | 愉悦阈值-30%，忧伤恢复快 |
| 悲观 | 凡事往坏处想 | 安全感需求永久+20%，易触发忧伤 |
| 急躁 | 容易发怒，缺乏耐心 | 等待类行动权重降低，愤怒阈值-20% |
| 温和 | 情绪稳定，不易激动 | 愤怒积累速度-50% |
| 敏感 | 心思细腻，容易受伤 | 关系值变化双倍效果，易触发尴尬 |
| 迟钝 | 神经大条，不记事 | 记忆生成概率-30%，不易记仇 |
| 多愁 | 见花落泪，对月伤怀 | 环境氛围影响大幅增加 |
| 豪爽 | 不拘小节，大开大合 | 娱乐需求恢复快，花钱大方 |

#### 2. 社交（人际互动风格）
| 标签 | 说明 | 效果示例 |
|------|------|----------|
| 热络 | 喜欢热闹，主动亲近 | 发起社交权重+30% |
| 孤僻 | 喜欢独处，回避社交 | 独处行动权重+50%，群聊参与率低 |
| 圆滑 | 见人说人话，不得罪 | 争执概率低，但信任增长慢 |
| 耿直 | 说话直，容易得罪人 | 信任初始高但容易因直言下降 |
| 多疑 | 不容易信任别人 | 信任增长效率-50% |
| 轻信 | 容易相信人，也容易失望 | 信任增长快，但受骗后崩溃 |
| 好胜 | 凡事要压人一头 | 论道、争执权重增加，认输概率低 |
| 谦让 | 不爱争抢，退一步海阔 | 冲突升级概率-40% |

#### 3. 习性（日常偏好与行为倾向）
| 标签 | 说明 | 效果示例 |
|------|------|----------|
| 贪嘴 | 好吃，对食物执着 | 饥饿阈值降低，容易去厨房 |
| 嗜睡 | 爱睡觉，起床困难 | 精力消耗快，早晨起床晚 |
| 爱洁 | 受不了一点脏 | 卫生需求高，频繁洗澡 |
| 邋遢 | 不拘小节，脏点无所谓 | 卫生需求低，舒适需求高 |
| 好动 | 坐不住，喜欢走动 | 户外活动权重增加 |
| 喜静 | 能不动就不动 | 室内活动权重增加，移动速度略慢 |
| 守时 | 严格遵守时间表 | 日常任务准时率+50% |
| 拖延 | 什么事都拖到最后一刻 | 任务截止前效率突增，但易逾期 |

**配置方式**：每个角色从三类中各选1-2个标签，形成唯一组合。例如黛玉：敏感+多愁+孤僻+耿直+喜静。这样即使性格总数不多，组合后也可产生上百种区分。

### 二、个人追求系统（长期目标）

在马斯洛需求层次之上，增加**个人追求**，作为角色的“人生任务线”。追求驱动角色的长期行为，并与利益冲突深度绑定。

#### 追求类型（可多选，但有优先级）
| 追求 | 描述 | 触发行为 | 冲突来源 |
|------|------|----------|----------|
| **求宠** | 获得某个主子的专宠 | 频繁侍奉、排挤竞争者 | 其他求宠者 |
| **求权** | 掌握管家权力 | 主动揽活、打压对手 | 现掌权者、同级竞争者 |
| **求名** | 在诗社或外面积累名声 | 参加诗社、出版作品、施舍穷人 | 同领域竞争者 |
| **求财** | 积累个人财富 | 放贷、经商、贪贿 | 债务人、竞争者 |
| **求爱** | 与心上人终成眷属 | 传情、私会、排除情敌 | 情敌、长辈阻挠 |
| **求安** | 安稳度日，不被撵走 | 谨小慎微、讨好长辈 | 威胁其安全者 |
| **求艺** | 精进某项技能（如绘画、医术） | 大量练习、拜师 | 很少，除非资源有限 |
| **求仕** | 科举入仕或提升官位 | 苦读、结交官员 | 同科考生 |
| **求道** | 修行解脱 | 静坐、诵经、远离俗务 | 几乎无 |

**动态效果**：追求进度影响角色情绪和决策权重。例如求宠的袭人若感到进度停滞，会焦虑并更激进地排挤晴雯。追求可被玩家或事件改变，如黛玉可能从“求爱”转为“求艺”。

### 三、简单动画系统（脱离纯日志）

通过少量动画状态，让玩家直接“看到”角色在做什么，而非只看日志。

#### 1. 动画状态机（轻量）
每个角色维护当前动画状态，根据行动类型自动切换。无需逐帧动画，用关键帧+图标即可。

| 动画状态 | 对应行动类型 | 视觉表现 |
|----------|--------------|----------|
| 闲站 | 空闲、等待 | 微微晃动，偶尔整理衣襟 |
| 行走 | 移动 | 按路径移动 |
| 坐 | 使用椅子、床、榻 | 坐下，可能看书/做针线/发呆 |
| 侍奉 | 服侍他人（倒茶、捶腿） | 弓身、端盘、靠近对方 |
| 争吵 | 争执互动 | 手指对方、摇头、跺脚 |
| 亲密 | 传情、拥抱、梳头 | 靠近、牵手、依偎 |
| 工作 | 洒扫、算账、做饭 | 扫地、拨算盘、炒菜 |
| 悲伤 | 忧伤、哭泣 | 低头、拭泪、蜷缩 |
| 高兴 | 愉悦、大笑 | 拍手、转圈、仰头笑 |
| 病弱 | 病中、精疲力竭 | 佝偻、缓慢移动、咳嗽 |

#### 2. 动画触发
- 行动开始时，角色切换到对应动画状态。
- 情绪爆发时，覆盖原有动画（如正在走路，突然进入“高兴”状态，会跳一下再继续走）。
- 观察-反应时，周围角色自动播放对应反应动画（如震惊、捂嘴、皱眉）。

#### 3. 资源最低实现
- 用简单骨骼动画或预制图标，区分性别和身份（主子/仆从）。后续可替换为精美动画。
- 仅需约10-15个基础动画模板，组合使用即可覆盖大量情景。

#### 4. 与现有系统结合
- 行动系统：执行行动时调用动画。
- 情绪系统：情绪状态可覆盖动画（如愤怒的人走路步伐更大）。
- 观察-反应：旁观者根据距离自动播放相应微动画。
