# 基础需求状态标签

> 更新时间：2026-06-16
> 主要配置：`needDefs[].stateBands`、`needCombinationStates`、`stateDefs`
> 主要代码：`js/systems/need-state.js`、`js/systems/state.js`

## 1. 分档规则

六项基础需求使用统一四档标签，中间 `40-79` 不生成标签：

| 需求 | `<20` 极低 | `20-39` 偏低 | `80-99` 满足 | `≥100` 超额 |
| --- | --- | --- | --- | --- |
| 饥饿 | 饿殍 | 饥肠 | 饱腹 | 肚圆 |
| 精力 | 力竭 | 困乏 | 焕发 | 亢奋 |
| 意趣 | 郁结 | 无趣 | 舒朗 | 陶然 |
| 洁净 | 臭烘烘 | 微尘 | 净爽 | 皎然 |
| 交游 | 独寂 | 疏淡 | 投契 | 人来疯 |
| 心绪 | 摧折 | 黯淡 | 美滋滋 | 酣畅 |

标签进入对应区间时获得，持续 30 游戏分钟；若 30 分钟后仍在该区间，会继续刷新。切换到同一需求的另一档时，利用 `conflictGroup` 替换旧标签。

需求标签属于额外状态，不覆盖人物当前行动文字，也不占普通状态的 6 个容量名额。

## 2. 正负效果

- 极低、偏低通常是负面状态，提高恢复对应需求的 AI 权重，并降低工作、移动、社交或心绪表现。
- 满足通常是正面状态，轻微提高工作、社交或需求稳定性。
- 超额可能是正面或混合状态。例如“肚圆”会抑制继续进食，但略微增加精力消耗；“人来疯”提高社交意愿，也更不愿独处。
- 状态效果统一写在 `stateDefs` 的 `needMods`、`aiModifiers`、`blockedSkills` 中。
- `polarity` 使用 `positive`、`negative`、`mixed`、`neutral`，控制后台分类和前台标签颜色。

## 3. 性格追加效果

状态可通过 `traitEffects` 对特定性格追加效果：

```json
{
  "traitEffects": {
    "aijie": {
      "needMods": {
        "mood": { "decay": 0.82 }
      },
      "aiModifiers": {
        "boost": { "task": 1.12 }
      }
    }
  }
}
```

当前“爱洁”人物：

- 洁净低时受到更明显的心绪/意趣负面影响，更积极寻找梳洗。
- 洁净达到 80 后获得“净爽”，达到 100 后获得“皎然”。
- 净爽、皎然对爱洁者有额外正面效果。

## 4. 多项需求组合

组合状态配置在 `needCombinationStates`。同一时刻只保留满足条件最多的一种：

| 状态 | 默认条件 |
| --- | --- |
| 心满意足 | 饥、闷 ≥80 |
| 神清气爽 | 倦、洁 ≥80 |
| 无欲无求 | 饥、倦、闷 ≥80 |
| 四大皆空（但吃饱了） | 饥 ≥100，倦、闷、洁 ≥80 |
| 周身通泰 | 饥、倦、闷、洁 ≥80 |
| 怡然自得 | 六项需求全部 ≥80 |

组合状态持续 30 游戏分钟，并使用统一冲突组避免多个近义标签同时堆叠。

## 5. 后台配置

### 【人物】需求

需求表显示：

`名称 / ID / 标签 / 摘要 / 默认增长系数 / 默认衰退系数 / 状态标签1-4`

高级 JSON 中：

- `defaultGrow`、`defaultDecay`：该需求的默认系数。
- `stateBands`：四档阈值与状态 ID。
- “组合满足状态”子表编辑多项需求条件。

### 【人物】状态

- 顶部“基础需求状态矩阵”按一项需求一行、四档一组展示。
- “多项需求组合状态”单独展示条件。
- “全部状态”仍可查看和编辑普通状态。
- 高级 JSON 可配置 `category`、`polarity`、`needBand`、`needMods`、`aiModifiers`、`traitEffects`。
- CSV 已扩展到交游、心绪、状态分类、正负类型、需求档位、AI 修正和性格追加效果。
