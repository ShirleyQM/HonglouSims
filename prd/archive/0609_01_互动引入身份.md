## 身份礼法分层互动系统 PRD

### 背景与目标

当前互动解锁仅靠关系值门槛，缺乏古代社会的身份维度。需要引入**身份礼法分层**，让互动受双方身份关系约束，同时保留"逾矩"的戏剧性空间——不是禁止，而是**风险与后果**。

### 核心设计：身份关系 + 礼法规则

#### 身份层级定义

每个角色有一个 `socialRank`，决定其在家族中的相对位置：

```
SocialRank:
    0. 至尊     —— 贾母（家族最高权威）
    1. 长辈     —— 贾政、王夫人
    2. 平辈主子 —— 贾宝玉、林黛玉、薛宝钗、王熙凤（已婚但同辈）
    3. 晚辈主子 —— 贾探春、贾惜春
    4. 体面仆从 —— 袭人、平儿、紫鹃（有头脸的大丫鬟/陪房）
    5. 普通仆从 —— 晴雯、麝月、雪雁、莺儿
    6. 外人/访客 —— 刘姥姥、清客相公
```

注意：rank 不是线性阶梯——平辈主子之间平等，但他们与长辈/仆从的关系不对称。

#### 双方身份关系判定

给定两个角色 A 和 B，判定他们的**身份关系类型**：

```
HierarchyRelation:
    parent_to_child      // A是B的长辈
    child_to_parent      // A是B的晚辈
    master_to_servant    // A是B的主子
    servant_to_master    // A是B的仆人
    senior_servant_to_junior  // 体面仆从 vs 普通仆从（有等级差）
    peer                 // 平等（同辈主子、同级仆从）
    outsider             // 一方是外人/访客
```

#### 礼法规则配置表

每种身份关系下，定义互动规则：

```
HierarchyRule {
    rule_id: string
    hierarchy_relation: HierarchyRelation
    interaction_category: string   // 互动大类，如"叙旧""传情""争执""差遣"

    // —— 基准规则 ——
    default_behavior: enum
        // allowed       —— 正常可用，无额外修正
        // conditional   —— 需满足额外条件
        // risky         —— 可用但被视为"逾矩"，有风险
        // forbidden     —— 系统阻止（极少数情况，如仆从主动斥责主子）

    // —— 条件（若 default_behavior = conditional） ——
    condition: ConditionGroup   // 如：独处时 / 关系值>70 / 对方情绪"开心"

    // —— 逾矩风险（若 default_behavior = risky） ——
    risk_profile: {
        base_success_rate: float       // 互动"成功执行不被阻拦"的概率
        fail_reaction: ReactionConfig  // 失败时的反应（被斥责/训斥/冷脸）
        success_reaction: ReactionConfig // 成功但被发现后的反应（关系特殊变化）
        witness_effect: {              // 若有旁观者，风险修正
            has_witness_penalty: float   // 有旁观者时成功率额外降低
            witness_reaction: ReactionConfig  // 旁观者的反应（告密/议论）
        }
    }

    // —— 效果修正 ——
    effect_modifiers: {
        affection_scale: float     // 好感变化倍率（逾矩行为好感变化放大）
        trust_scale: float         // 信任变化倍率
        intimacy_scale: float      // 亲密变化倍率
        extra_effects: List<Effect> // 额外效果（如触发特殊状态）
    }

    // —— 冷却与限制 ——
    cooldown_modifier: float        // 冷却时间倍率（逾矩行为冷却更长）
    daily_limit: int               // 每日最大次数（0=无限）

    // —— 气泡文案 ——
    bubble_on_risky_success: string  // 逾矩成功后的气泡
    bubble_on_risky_fail: string     // 逾矩失败后的气泡
}
```

### 身份关系判定

```js
getHierarchyRelation(idA, idB) {
    const rankA = getChar(idA).socialRank;
    const rankB = getChar(idB).socialRank;

    // 外人处理
    if (rankA === 'outsider' || rankB === 'outsider') return 'outsider';

    // 差 ≥ 3 级 → 主仆关系
    if (rankA <= 3 && rankB >= 4) return 'master_to_servant';
    if (rankB <= 3 && rankA >= 4) return 'servant_to_master';

    // 差 1-2 级但都在仆从层 → 体面 vs 普通
    if (rankA <= 4 && rankB === 5) return 'senior_servant_to_junior';
    if (rankB <= 4 && rankA === 5) return 'junior_servant_to_senior';

    // 差 1 级且同在主子层 → 长辈/晚辈
    if (rankA === 1 && rankB >= 2) return 'parent_to_child';
    if (rankB === 1 && rankA >= 2) return 'child_to_parent';
    if (rankA === 0) return 'grandparent_to_child';  // 贾母对所有人

    // 其他 → 平等
    return 'peer';
}
```

### 互动判定流程

```
当角色 A 发起对 B 的互动 I 时：

1. 获取 hierarchyRelation = getHierarchyRelation(A, B)
2. 查询 HierarchyRule WHERE relation = hierarchyRelation AND category = I.category
3. 若 default_behavior = forbidden → 拒绝，不显示选项
4. 若 default_behavior = conditional → 检查条件，不满足则灰显
5. 若 default_behavior = risky:
    a. 计算成功率 = rule.risk_profile.base_success_rate
    b. 若同场景有旁观者 → 成功率 += witness_effect.has_witness_penalty
    c. 若成功 → 执行互动 + success_reaction + effect_modifiers
    d. 若失败 → 执行 fail_reaction，互动中断
    e. 若有旁观者且成功 → 旁观者可能触发 witness_reaction
6. 若 default_behavior = allowed → 正常执行
7. 应用 effect_modifiers 到关系变化（affection/trust/intimacy 倍率）
8. 应用 cooldown_modifier 和 daily_limit
```

### 逾矩的戏剧张力

逾矩不是简单的"禁止/允许"，而是一个**风险-回报博弈**：

**仆从向主子传情：**
- 成功概率 50%
- 成功 → 亲密大幅上升，双方获得"暧昧氛围"状态，形成秘密
- 失败 → 好感暴跌，仆从受罚，可能引发观察-反应连锁
- 有旁观者 → 成功率再降 30%，且旁观者可能告密
- 每日限 3 次——玩家不能无限尝试

**这制造了：**
- 紧张感：玩家不知道这次逾矩会不会被发现
- 叙事性：成功与失败都产生不同的故事走向
- 策略性：要不要等独处时再试？要不要先搞好关系提升成功率？

### 与现有系统的对接

| 系统 | 对接方式 |
|------|----------|
| 三轴关系系统 | 效果修正中的 scale 直接作用于 affection/trust/intimacy 的 delta |
| 互动模板 | 模板中新增 `hierarchy_rule_ref` 字段，引用具体 rule_id |
| 观察-反应系统 | risky 行为触发旁观者判定，走已有观察-反应链路 |
| 叙事气泡 | fail_reaction / success_reaction 中的 bubble 直接推送 |
| 状态系统 | risk 结果可添加/移除状态 |
