## 1. 总目标

任务系统要从“一个可接受、可完成的列表”升级为“大观园家内事务与人物命运的核心驱动”。

最终目标：

- 任何任务都来自明确来源：日常、职责、差遣、群体传令、随侍轮值、剧情、人生路径或 AI 自主下发。
- 任务是否能下发，由身份等级、身份关系、任务类别、任务模板、仆从职责契约和特殊权限共同决定。
- 任务是否接受，由自动接受配置、服从、信任、好感、职责内/外、当前负担、心绪、疲倦和性格共同决定。
- 任务如何执行，由完成条件、AI 候选权重、场景权限、家具可用性、目标人物位置、需求打断和任务优先级共同决定。
- 任务结果不只分成功/失败，还要计算质量，并影响关系、忠诚、积怨、劳役压力、状态、技能、需求和声望。
- 玩家可以在 UI 中看懂“谁给谁下了什么任务、为什么能下、为什么没做、下一步该怎么做”。
- 测试黛玉时可以通过任务面板和埋点还原“一整天做了什么、为什么这样做”。

## 3. 核心概念

### 3.1 任务来源

| 来源 | `questType` / 机制 | 说明 |
| --- | --- | --- |
| 日常任务 | `daily` | 固定时间或职责契约自动生成。 |
| 职责任务 | `daily` + `servantConfig.dutyRoutines` | 由主仆职责契约生成，如晨起伺候、洒扫、陪读。 |
| 随侍轮值 | `daily` + `servantConfig.followRotations` | 当日指定仆从跟随主子。 |
| 单人差遣 | `directive` + `targetScope !== group` | 玩家或 AI 向单个目标下发。 |
| 群体传令 | `directive` + `targetScope: group` | 向符合筛选条件的一组人下发。 |
| 剧情任务 | `event` | 由人生路径、故事节点、事件触发。 |
| 人生路径任务 | `daily/directive/event` + lifePath | 根据角色路径与阶段解锁。 |
| AI 自主下发 | issue AI | 上位者或管事按冷却与权重自动派任务。 |

### 3.2 任务生命周期

当前已实现口径：

```text
生成任务
  ↓
PENDING 待接受
  ├─ 接受 → ACCEPTED → 满足条件推进 → COMPLETED / FAILED
  ├─ 拒绝 → DECLINED
  ├─ 发布者撤回 → REVOKED
  └─ 超时未处理 → EXPIRED
```

建议扩展状态：

```text
IN_PROGRESS   明确进行中
PAUSED        被高优先级任务、需求或状态打断
BLOCKED       家具/路径/目标/技能/身份权限/状态不满足
TRANSFERRED   被转派
DELAYED       被批准延期
```

### 5.2 群体字段

```js
groupFilter: {
  rankRange: [min, max],
  gender: 'male' | 'female' | 'any',
  familyId: 0,
  sceneId: 0 | 'issuer',
  excludeRanks: [],
  maxTargets: 0
}
```

说明：

- `rankRange`：目标身份等级。
- `gender`：筛选性别，`any` 不限。
- `familyId`：指定家庭，`0` 不限。
- `sceneId`：指定场景，`0` 不限，`issuer` 表示下发者所在场景。
- `excludeRanks`：排除身份等级。
- `maxTargets`：最多目标数，`0` 表示不限。

### 5.3 完成条件

| 条件 | 含义 | 最新口径 |
| --- | --- | --- |
| `USE_FURNITURE_DURATION` | 使用指定家具累计时间。 | 按游戏分钟累计。 |
| `USE_FURNITURE` | 完成一次指定家具行为。 | 家具类别可匹配。 |
| `INTERACT_WITH` | 与指定人物互动一次。 | 目标需匹配 `issuer/assignee/具体 ID`。 |

| `STAY_IN_SCENE` | 在指定场景停留。 | 支持 `issuer`、`assignee`、`issuer_residence`、`assignee_residence`。 |

### 5.4 失败条件

| 条件 | 含义 |
| --- | --- |
| `AVOID_SCENE` | 进入指定场景则失败。 |
| `AVOID_CHARACTER` | 接近指定人物则失败。 |

### 5.5 截止时间

任务统一用 `getGameTimestamp()`，单位为游戏分钟。

```text
gameTimestamp = gameDay × 1440 + gameHour × 60 + gameMinute
```

| 模式 | 含义 |
| --- | --- |
| `GAME_HOURS` | 接受时间 + 参数小时。 |
| `GAME_DAYS` | 接受时间 + 参数天数。 |
| `BY_TIME_OF_DAY` | 当日指定小时，错过则顺延一天。 |
| `BY_PHASE` | 当前固定为接受后 180 游戏分钟。 |
| `BEFORE_SLEEP` | 当日 22:00。 |
| `NO_DEADLINE` | 无期限。 |

默认时间配置：

| 项 | 值 |
| --- | --- |
| 待回应过期 | 180 游戏分钟。 |
| AI 尝试下发任务间隔 | 60 游戏分钟。 |
| 默认重复冷却 | 1440 游戏分钟。 |
| 群体任务默认冷却 | 1440 游戏分钟。 |

## 7. 单人差遣流程

### 7.1 玩家下发

```text
玩家选择控制角色
  ↓
点击目标人物
  ↓
系统判断身份关系与权限
  ↓
若可下发：互动菜单出现“差遣/吩咐”
  ↓
展示通过过滤的 QuestTemplate
  ↓
玩家点击任务
  ↓
生成 QuestInstance
  ↓
目标自动接受或进入待回应
```

过滤条件：

- 任务是 `directive`。
- 不是群体任务。
- 下发者有任务系统权限。
- `category` 在下发者权限允许列表中。
- 当前双方身份关系满足模板要求。
- 目标 rank 在模板范围内。
- 若模板是特殊任务，则下发者有对应 `specialQuestKey`。
- 若是主仆差遣，还要通过 `ServantRelationSystem.checkQuestAuthority()`。
- 冷却未结束，没有同类重复任务。

### 7.2 当值仆从快速差遣

主子视角底部家庭栏显示“今日当值”。

```text
点击当值仆从头像
  ↓
直接打开“吩咐”任务环
  ↓
选择任务
```

当前可见效果：

- 黛玉视角可看到紫鹃/雪雁当值。
- 点击当值头像直接出现任务环。
- `唤来报到`、`服侍更衣`、`抄写经文`、`替人顶班`等会根据权限显示。

### 7.3 下发成功反馈

下发成功后：

- 日志记录：`任务下发：任务名 → 执行者`。
- 若自动接受，日志记录：`执行者 接受「任务名」`。
- 仆从头上气泡：
  - 同场景普通差遣：“这就去办。”
  - `唤来报到` 或跨场景：“这就来。”

## 9. 仆从职责契约

### 9.1 契约字段

```js
{
  id: 'daiyu_zijuan',
  masterId: 'daiyu',
  servantId: 'zijuan',
  role: '贴身侍女',
  scope: 'personal',
  authorityLevel: 86,
  dutyCategories: ['侍奉', '陪伴', '传话', '照看', '跑腿', '跟随'],
  workScenes: [1],
  monthlyWage: 180
}
```

| 字段 | 含义 |
| --- | --- |
| `masterId` | 直接主子。 |
| `servantId` | 仆从。 |
| `role` | 角色说明，如贴身侍女、房中丫鬟、心腹管事。 |
| `scope` | 职责范围。 |
| `authorityLevel` | 主子对该仆从的权威强度。 |
| `dutyCategories` | 职责内任务类别。 |
| `workScenes` | 工作场景。 |
| `monthlyWage` | 月银。 |

### 9.2 scope 含义

| scope | 含义 |
| --- | --- |
| `personal` | 贴身服务，只对直接主子负责。 |
| `household` | 家庭/院落事务，可被家主、主母、管事调动。 |
| `yard` | 院落专属。 |
| `outer` | 外院、采办、传递等。 |

| 主子 | 仆从 | 角色 | 职责 |
| --- | --- | --- | --- |

### 9.4 权限边界

- 主子可对直属仆从下达职责内任务。
- 主子调动其他房贴身仆从时，需要更高家内权限，或其主子允许。
- 管事仆从可安排低等仆从做洒扫、传话、跑腿、采办、照看、粗使。
- 普通仆从不能向主子下任务。
- 职责外任务会降低接受概率，或被拒绝。

- AI 优先解决需求。
- 任务仍在，解决后因任务权重回归。

- `solving_need`

## 12. 任务接受与拒绝

### 12.1 自动接受

`autoAccept: true` 的任务直接进入 `ACCEPTED`。

适用：

- 家法、禁足、请安等强制任务。
- 大多数仆从职责。
- 随侍轮值。
- 唤来报到。

### 12.2 可拒绝任务

`autoAccept: false` 需要判断接受概率。

当前接受参考：

```text
接受概率 =
  基础概率
  + 职责内加成
  + 服从修正
  + 信任修正
  + 好感修正
  - 当前任务负担
  - 疲倦/心绪惩罚
  - 越权或职责外惩罚
```

当前涉及：

- `作诗陪吟`
- `告假探亲`
- `替人顶班`
- `隐瞒过失`
- 部分故事线任务

### 12.3 性格影响

来自性格系统 PRD 的任务相关口径：

| 性格/模块 | 影响 |
| --- | --- |
| 守时 | 日常任务准时率提高，提前赶往任务地点。 |
| 拖延 | 普通任务权重降低，截止前两小时急剧提高，易逾期。 |
| 多疑 | 更易拒绝陌生人任务、邀请、传情。 |
| 轻信 | 任务与邀请接受率提高，受骗后损失更大。 |
| 任务紧迫标签 | 任务越临近截止，AI 候选权重越高。 |

## 20. 与其他系统关系

### 20.1 时间系统

- 所有任务时间使用游戏分钟。
- 现实时间加速/减速时，任务相对游戏时间保持一致。
- 页面暂停时，任务不会离线推进。

### 20.2 需求系统

- 洒扫任务完成恢复洁净。
- 跟随中基础需求过低可打断。
- 任务失败可影响心绪。
- 任务完成可离散恢复需求。

### 20.3 性格系统

任务相关性格效果：

- 接受率。
- 准时率。
- 截止前权重。
- 失败概率。
- 任务紧迫度感知。
- 多疑/轻信对任务和邀请接受的影响。

### 20.4 关系系统

任务影响：

- 好感。
- 信任。
- 友谊。
- 服从。
- 忠诚。
- 积怨。

### 20.5 经济系统

已有关联：

- 契约月银字段。
- 家庭公账。
- 饮食消费。
- 工资收入。

待完善：

- 月银按月发放。
- 任务奖励/罚金入账。
- 采办类任务消耗和报销。

### 20.6 场景权限

- 仆从进入私宅可能需要主子、任务或跟随授权。
- 跟随任务可让仆从追随主子进入相关场景。
- 跟丢主子会导致跟随任务阻塞。
