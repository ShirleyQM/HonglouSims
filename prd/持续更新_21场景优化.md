# 场景优化 PRD

> 更新时间：2026-06-23
> 文档定位：场景视觉、场景资产、地板/地面材质、未完成场景占位规则。
> 主要代码：`js/config.js`、`js/world.js`、`js/draw.js`、`js/assets.js`、`assets/manifest.json`。

## 1. 总目标

场景优化要把当前“色块地图 + 少量房间背景”升级为可持续扩展的场景资产系统：

- 已完成场景有明确地面、边界、家具热区和氛围资产。
- 未完成场景统一显示为黑色，避免用临时色块误导玩家以为已完成。
- 室内、室外、游廊使用不同地板/地面素材，但仍保持俯视 2D/像素风可读性。
- 场景资产可按场景逐步替换，不阻塞 AI、路径、家具和权限系统。

## 3. P0 规则：未做场景用黑色

### 3.1 场景完成状态

新增场景视觉状态字段，建议挂在 `assets/manifest.json`，避免污染玩法配置：

```json
{
  "sceneVisuals": {
    "1": { "status": "done", "groundSet": "xiaoxiang_wood", "background": "roomBackgrounds.1" },
    "2": { "status": "todo" }
  }
}
```

状态枚举：

| 状态 | 绘制规则 | 用途 |
|---|---|---|
| `done` | 使用真实背景、地面素材或程序化地砖 | 已完成视觉验收的场景。 |
| `wip` | 使用灰黑半透明遮罩 + 场景名 | 正在制作，可测试路径和热区。 |
| `todo` | 整块黑色，仅显示场景名/边框 | 未做场景，玩家一眼知道不是最终效果。 |

### 3.2 默认策略

- 没有 `sceneVisuals[sceneId]` 的场景默认视为 `todo`。
- 已有 `roomBackgrounds` 的场景可默认视为 `done`，但最终仍应补 `sceneVisuals` 明确记录。
- 黑色占位不影响寻路、任务、互动、家具占位，只影响视觉层。
- 场景名仍保留，方便调试和点击定位。

## 4. 场景资产分层

| 层级 | 资产类型 | 当前建议 |
|---|---|---|
| 地面层 | 可平铺地板/地面纹理 | 优先 32x32 或 64x64 PNG，支持程序化回退。 |
| 背景层 | 整张房间/院落底图 | 只给重点场景使用，例如潇湘馆、怡红院、荣禧堂。 |
| 边界层 | 墙、围栏、廊柱、水岸、门洞 | 可用 tile 或小 PNG 组合。 |
| 家具热区层 | 家具图片或可点击占位 | 继续复用 `furnitureInstances`。 |
| 氛围层 | 竹影、花丛、水面、灯影 | 后置，先不影响基础可玩性。 |

## 5. 免费素材调研

### 5.1 优先候选

| 来源 | 适合内容 | 授权判断 | 备注 |
|---|---|---|---|
| [Kenney - Retro Textures Fantasy](https://kenney.nl/assets/retro-textures-fantasy) | 像素/复古地面纹理、幻想室内外铺地 | 页面标注 Creative Commons CC0，115 个文件 | 和当前俯视 2D 风格最容易接。 |
| [Kenney - Development Essentials](https://kenney.nl/assets/development-essentials) | 原型地面、调试占位 | 页面标注 Creative Commons CC0，15 个文件 | 适合做黑场/灰场之外的开发占位。 |
| [ambientCG - Wood Floor](https://ambientcg.com/list?category=WoodFloor) | 室内木地板 PBR/贴图 | 官方 license 文档说明全部资产 CC0，可商用、可改、可放进游戏 | 画风偏写实，需要降采样/像素化处理。 |
| [ambientCG - Ground / Tiles / Paving Stones](https://ambientcg.com/list) | 室外泥地、草地、石径、铺石 | 同上，ambientCG 全站资产 CC0 | 可作为生成 2D tile 的源素材。 |
| [Poly Haven Textures](https://polyhaven.com/textures) | 高质量石板、木地板、砖、泥地 | 官方 license 页面说明资产均为 CC0，可商用且无需署名 | 偏写实，适合 3D 预览或处理成低分辨率地砖。 |
| [OpenGameArt - Tiny 16: Basic](https://opengameart.org/content/tiny-16-basic) | 16x16 RPG 地面、草地、桥、树、基础 tileset | 页面标注 CC-BY 4.0 / CC-BY 3.0 / OGA-BY，需要署名 Lanea Zimmerman | 好用但非 CC0，若使用需在项目 credit 中署名。 |

### 5.2 授权结论

- P0/P1 优先使用 CC0：Kenney、ambientCG、Poly Haven。
- OpenGameArt 资源质量参差且多为 CC-BY/OGA-BY，能用，但需要建立 `assets/CREDITS.md`。
- 写实 PBR 素材不要直接塞进当前 2D 主画面；建议先做“裁切 -> 降到 32/64px -> 调色 -> 像素化/低频化 -> tile seam 检查”。
- 若后续做 `three-preview` 或 3D 沙盘，ambientCG / Poly Haven 的 PBR 法线、粗糙度图更有价值。

## 6. 场景地面规格

| 场景类型 | 推荐地面 | 尺寸 | 风格约束 |
|---|---|---:|---|
| 私密室内 | 木地板、砖、席纹 | 32x32 / 64x64 | 暗色、低对比，不抢人物。 |
| 礼仪厅堂 | 石板、方砖、汉白玉 | 32x32 / 64x64 | 规整、略亮，可体现等级。 |
| 院落庭园 | 草地、泥地、碎石、花径 | 32x32 / 64x64 | 变化丰富，但格线不要过强。 |
| 游廊 | 横向木板或石径 | 32x32 | 方向性明确，帮助玩家识别通道。 |
| 水边/洲渚 | 湿泥、青苔石、浅水边 | 32x32 / 64x64 | 与可行走边界区分清楚。 |

## 7. 实施计划

### P0：视觉状态与黑色占位

- 在 `assets/manifest.json` 增加 `sceneVisuals`。
- 在 `AssetSystem` 增加 `sceneVisualDef(sceneId)`。
- 在 `drawWorld()` 中先判断状态：`todo` 画黑色，`wip` 画灰黑占位，`done` 走原有背景/地砖逻辑。
- 当前仅潇湘馆可标为 `done`，其余场景先标 `todo` 或 `wip`。

### P1：地板素材接入

- 新增 `assets/tiles/` 目录，存放处理后的 32/64px tile。
- `manifest.tiles.types` 支持 `src`，没有图片时回退程序化地砖。
- 先做 4 套基础地面：室内木地板、礼仪石板、室外草地、游廊木板。
- 逐个场景替换 `ground` 或在 `sceneVisuals` 里指定 `groundSet`。

### P2：重点场景美术化

- 潇湘馆：保留当前背景，补竹影/窗纱/水声感的地面与边界。
- 怡红院：做锦绣木地板、花木院落、暖色调。
- 荣禧堂：做石板/玉阶、礼仪厅堂氛围。
- 大观楼·沁芳庭：做庭院、甬道、中心牌坊感。

## 8. 验收标准

- 未做场景在主画面中为黑色，不再出现“看起来像已完成”的临时色块。
- 黑色占位场景仍可走路、交互、执行任务，功能不被视觉状态破坏。
- 已完成场景的地面平铺无明显接缝，人物、家具、气泡可读。
- 地面素材授权来源可追溯；非 CC0 必须进入 credits。
- 低端浏览器下地砖缓存生效，不因每帧重建 canvas 或图片造成卡顿。
