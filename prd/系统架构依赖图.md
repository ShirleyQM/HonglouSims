# 大观园整体系统架构依赖图

> 本文档单独维护系统之间的架构依赖图。业务 PRD 只保留局部规则，并引用本文，避免同一张图在多份文档里分叉。

## 1. 当前核心依赖图

```mermaid
flowchart TD
  Character["人物 Character"]
  Career["职业/人生路径\nLifePath · Profession · Role"]
  Contract["职责契约\nServantRelationSystem"]
  Command["传令/上下级管理\n点名 · 群体"]
  Routine["起居 Profile\n需求/技能/职业"]
  Quest["任务系统 QuestSystem"]
  Utility["Utility AI\n候选池与权重"]
  Furniture["家具互动"]
  Social["社交互动"]
  Needs["六需求"]
  Scene["场景权限"]
  Skill["技能系统"]
  Relation["关系系统"]
  Economy["金钱/家庭经济"]
  Result["行为结果\n质量/关系/声望/忠诚"]

  Character --> Career
  Character --> Contract
  Character --> Skill
  Character --> Needs
  Character --> Relation
  Career -->|"dailyTasks / 职业班次"| Routine
  Career -->|"职业上下级 / 职位权限"| Command
  Contract -->|"dutyRoutines / followRotations"| Routine
  Contract -->|"直属 / 当值 / 职责内"| Command
  Relation -->|"服从 submission\n信任 trust"| Command
  Command -->|"点名传令 / 群体传令"| Quest
  Routine -->|"routineAnchor"| Utility
  Routine -->|"职业日课投影"| Quest
  Quest -->|"questFactor / 任务候选"| Utility
  Utility --> Furniture
  Utility --> Social
  Utility --> Scene
  Furniture --> Needs
  Social --> Relation
  Needs --> Utility
  Scene --> Utility
  Scene --> Quest
  Skill --> Utility
  Skill --> Quest
  Economy --> Quest
  Quest --> Result
  Relation --> Result
  Result --> Career
  Result --> Contract
  Result --> Relation
```

## 2. 职业、起居、任务、传令局部链路

```mermaid
flowchart LR
  Profession["职业身份\n丫鬟/小厮/管事/先生/商人"]
  Duty["每日职责\n日课/班次/随侍轮值"]
  Relation["关系轴\n服从/信任/好感"]
  Command["传令面板\n点名传令/群体传令"]
  Routine["起居职业行\n可视化时间块"]
  Quest["QuestSystem\n生成 daily 任务实例"]
  AI["Utility AI\nroutineFactor + questFactor"]
  Action["家具/社交/跟随/场景停留"]
  Result["结果反馈\n质量/服从/忠诚/积怨"]

  Profession --> Duty
  Profession --> Command
  Duty --> Routine
  Duty --> Command
  Relation --> Command
  Command --> Quest
  Routine --> Quest
  Routine --> AI
  Quest --> AI
  AI --> Action
  Action --> Result
  Result --> Relation
```

## 3. 传令与关系接入

```mermaid
flowchart TD
  Issuer["发布者"]
  Target["目标人物"]
  Relation["关系系统\nsubmission: 目标对发布者服从\ntrust: 发布者对目标信任"]
  Authority["管理权限判断\n身份/职业/契约/临时授权"]
  Command["传令面板\n展示服从、信任、预计接受"]
  QuestIssue["QuestIssueSystem\n过滤模板与目标"]
  Quest["QuestSystem\n任务实例"]
  Settlement["任务结算\n接受/拒绝/质量"]
  RelationResult["关系反哺\n服从/信任/好感"]

  Issuer --> Authority
  Target --> Authority
  Relation --> Command
  Authority --> Command
  Command --> QuestIssue
  QuestIssue --> Quest
  Quest --> Settlement
  Settlement --> RelationResult
  RelationResult --> Relation
```

## 4. 当前优先级口径

```text
严重需求危机 / 安全
  > 强制剧情 / 高优先临时任务
  > 随侍轮值 / 职业日课
  > 普通起居活动
  > 闲逛 / 普通社交 / 随机家具
```

## 5. 架构维护规则

- 系统级依赖图统一放在本文。
- 业务 PRD 可描述自己的局部流程，但不要复制全局图。
- 新系统接入时先补本文，再回到对应业务 PRD 写字段和验收。
