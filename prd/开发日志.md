## 2026-06-17 · Utility AI 任务候选与长模拟复验

- 修复场景权限软锁：角色若已经身处无权场景，路径检查允许其离开当前场景，避免“越无权越出不来”。

## 2026-06-12 · 对话/气泡/情绪/状态配置表梳理 + 批量生成指南

### 概述

梳理对话面板、叙事气泡、情绪、状态四套内容的配置来源与字段规范，并产出 **AI 批量生成用的空白模板 + Prompt**，便于扩写互动台词、内心气泡、状态机制等。

**完整文档：** [`配置表批量生成指南.md`](配置表批量生成指南.md)
**可复制 Prompt：** [`prompts/batch-gen/`](prompts/batch-gen/)（12 张表各一个 `.txt` + `00-master-all-in-one.txt` 整包）

### 四套系统对应关系

| 系统 | 主要配置位置 | 玩家可见 |
|------|--------------|----------|
| 对话面板 | `interactionTemplates` + `prompts/interaction-topics.txt` | LLM 全屏对白、关系/状态 HUD |
| 叙事气泡 | `narrativeBubble.demandBubbles` / `conflictTheaters` / `interactionStateBubbles` / `memorySurfaces` | 头顶内心/说话气泡 |
| 情绪 | `stateDefs`（情绪类）+ `emotionContagion` + `statusTags` | 状态标签、传染、旁白映射 |
| 状态 | `stateDefs` + `状态总表_结构化版.csv` | 禁技能、需求衰减、AI 权重、自动触发 |

### 配置表清单（12 张）

1. **互动模板** `interactionTemplates` — 菜单项、兜底台词、效果、软锁 `onLowScore`
2. **LLM 话题** `prompts/interaction-topics.txt` — 每类互动的写作说明
3. **需求气泡** `demandBubbles` — 饿/闷/累/脏 + 性格分支
4. **冲突小剧场** `conflictTheaters` — 双人性格冲突连播
5. **互动后气泡** `interactionStateBubbles` — 获状态后的内心话（`stateId` 须对齐 `stateDefs`）
6. **记忆浮现** `memorySurfaces` — 进场景按记忆标签回想
7. **状态定义** `stateDefs` + CSV 设计稿 — 完整机制版含 `needMods` / `aiModifiers` / `trigger`
8. **情绪传染** `emotionContagion` — 近距离状态传播 + 气泡
9. **旁观反应** `observerReactions` — 宝钗劝架、雪雁打听等
10. **关系阈值** `relationThresholdEvents` — 四轴 crossing 旁白与效果
11. **状态中文标签** `statusTags` — 旁白中文 → runtime id
12. **对话 HUD** — 无内容表（读运行时关系/状态）

### 批量生成推荐顺序

```
stateDefs → interactionStateBubbles → demandBubbles → emotionContagion
→ interactionTemplates + interaction-topics → conflictTheaters / memorySurfaces
→ relationThresholdEvents / observerReactions
```

### 关键约束（给 AI 时务必带上）

- 文风：红楼梦大观园，古风白话；气泡 **≤30 字**
- `duration` / `cooldown` 单位：**游戏分钟**
- 状态 id：**camelCase 运行时 id**（`awkward`）与 **CSV 编号**（`S003`）勿混为同一状态
- `interactionStateBubbles.stateId` 必须已有于 `stateDefs`
- `emotionContagion` 的 source/result 用 camelCase（`joyful`、`ganshang`），不是 `S024`

### 当前内容缺口

| 缺口 | 说明 |
|------|------|
| CSV 情绪缺机制 | `S001`–`S030` 注入后多数只有 name/desc，无 `needMods` |
| id 双轨 | `S101` 尴尬 vs `awkward`；气泡只认 `awkward` |
| 互动后气泡 | 仅 14 条，情绪类远未覆盖 |
| 段情绪 mood | LLM 段内 `开心/不欢/平平` 无配置表，代码写死倍率 |
| 群聊原型 | `dialogue.js` 内 `TOPIC_LIBRARY` 未进 config |

## 2026-06-12 · 故事任务面板显示修复

### 问题
玩家反馈右下角任务栏看不到袭人故事任务。根因有三：

1. **初始化顺序**：`LifePathSystem.init()` 在 `loadGameFromStorage()` 之前下发任务，读档后任务实例被存档覆盖清空。
2. **任务未自动下发**：event/directive 类故事任务（4002-4013）原先 `issueBaseChance: 0`，没有程序化下发逻辑。
3. **面板只显示当前角色**：右下角 `#col-quest` 仅渲染 `selectedIdx` 的任务；操控宝玉时看不到袭人的任务。

### 修复

**`life-path.js`**
- 新增 `tryIssueStoryQuests(c)` / `tryIssueStoryQuestsAll()`：按 `storyNodes` 的 `prevNode`、`minReputation`、`minSkill` 条件，调用 `QuestSystem.issueOne` 自动下发故事任务。
- 在路径开启、节点完成、阶段晋升、每小时 tick 时检查并下发。
- 每个 storyNode 补充 `prevNode` 链式条件。

**`main.js`**
- `QuestSystem.init()` 先于 `LifePathSystem.init()`。
- 读档/新局结束后统一调用 `tryIssueStoryQuestsAll()`。

**`ui.js` / `quest.js`**
- 右下角迷你任务卡：除当前角色外，也显示其他路径角色的故事任务（标注「袭人·进行中」）。
- 展开任务面板新增「故事线任务」分区。

### 玩家怎么看任务

1. 右下角「任务」列 — 开局应出现 **晨起理妆**（袭人·进行中），即使当前操控的是宝玉。
2. 点「展开」可看完整进度与完成条件。
3. 切换选中袭人后，任务归属更清晰；完成方式：让袭人跟宝玉同场景、跟随宝玉 20 分钟（每日卯时也会自动再发）。

## 2025-06-12 互动综合分软锁统一（0609_02）

### 问题

`0609_02_对话引起状态变化.md` 设计与实现不一致：

- 文档要求「不硬锁、综合分不足给状态」，代码里 `axisReq` / `unlock_conditions` 会直接禁用互动
- 阈值字段文档写 `minScore`，代码只用 `relMin`
- 逾矩失败只 `applyState`，不触发 `interaction:state`，头顶内心气泡不出现
- `selfDemeaning` / `unfilial` / `doted` 等状态有定义但未接线
- `config.js` 末尾 patch 重复覆盖 401–404，部分 `onLowScore` 丢失

### 方案：新建 `interaction-score.js`

统一软锁入口，与 `action-queue.js` / `interaction-social.js` 分工：

| 模块 | 职责 |
|------|------|
| `interaction-score.js` | `getMinScore`（`minScore` → `relMin` → 类别默认）、`checkSoftLock`、`resolveOnLowScore`、逾矩成败状态解析 |
| `action-queue.js` | `checkInteractionAvailable` 改调软锁；`applyLowScoreEffects` 用解析后的 `onLowScore` |
| `interaction-social.js` | 逾矩 `applyRiskOutcome` 发 `interaction:state`；解锁不足不再 `locked` 隐藏 |
| `config.js` | `interactionSocialConfig.categoryMinScore` / `categoryOnLowScore` / `hierarchyOnLowScore` |

### 行为变化

1. **综合分、三轴、unlock 不足** → 仍可发起，走近后走 `runLowScoreInteraction`（尴尬/恼怒/受责等），不再菜单灰掉
2. **类别默认阈值**（可被模板 `minScore`/`relMin` 覆盖）：传情 30、调笑 20、争执 -20、慰藉 0、叙旧 -50
3. **礼法逾矩成功**：传情 → `secretCrush`；长辈接受晚辈逾矩调笑/传情 → `doted`
4. **礼法逾矩失败**：晚辈顶撞长辈争执 → `unfilial`；仆对主争执 → `punished`；默认 `awkward`
5. **主子对仆过于殷勤（慰藉）** → 仆人 `selfDemeaning`；仆对主争执且服从极低 → `punished` + 主 `offended`
6. `showLockedInteractions: true`，互动菜单显示全部项，不足时标「情分欠·或尴尬」等

### 自测

- 对陌生人发起「凝眸」→ 尴尬状态 + 内心气泡 + 左侧状态标签
- 信任不足的「私语」→ 软锁尴尬（不再「需信任≥50」硬挡）
- 仆对主逾矩传情失败 → 尴尬气泡；晚辈顶撞长辈失败 → 不孝

## 2025-06-12 试探/尴尬收场接入本地 LLM

### 需求

综合分不足的软锁、逾矩失败等「试探性」收场，原先只有固定气泡「神色淡淡，不接这话」；改为走 Ollama 生成短对白。

### 实现

- `interaction-llm.js`：新增 `tryBeginAwkward` / `fetchAwkwardLines`
  - 专用 prompt：情分不足/逾矩未成、欲言又止、对方冷淡推拒
  - 默认 4 句、仅 1 段，播完自动收场（`awkwardMode`）
- `action-queue.js`：`runAwkwardDialogue` 统一软锁与逾矩失败；`finishInteractionAction` 识别 `lowScoreMode` 不再重复结算效果
- `config.js`：`interactionLowScoreEnabled`、`interactionLowScoreLines`；试探(502) 补 `llmPrompt`

### 配置

`narrativeBubble.settings.llm.interactionLowScoreEnabled`（默认 true）可关闭，回退固定台词。

### 自测

玩家控制角色，对关系差的人「凝眸/试探」→ 生成尴尬短对白 → 播放后左侧仍有「尴尬」状态。
