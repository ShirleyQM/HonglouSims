# UI：环形互动菜单 PRD

> 更新时间：2026-06-25
> 主要代码：`js/ui.js`、`js/systems/interaction-social.js`、`js/systems/quest-issue.js`。

## 1. 定位

环形互动菜单是玩家点击人物或家具后的第一层操作入口。

它负责：

- 在目标附近展示可执行互动。
- 把人物社交、家具使用、传令入口和查看入口分清楚。
- 不超出屏幕边缘，不遮挡关键 HUD。
- 展示禁用原因、关系意愿、礼法风险和 LLM 状态。

## 2. 当前口径

| 点击对象 | 菜单内容 |
| --- | --- |
| 地面 | 移动或加入行动队列。 |
| 家具 | 家具动作，恢复需求或触发效果。 |
| 人物 | 社交互动、传令、查看人物/关系等入口。 |

人物互动里的任务下发入口统一叫“传令”，不再混用“差遣”。点击后打开统一传令面板，并预选目标人物。

## 3. 视觉和交互规则

- 菜单以目标为中心，但必须根据屏幕边缘自动偏移。
- 第一层展示类别，第二层展示具体互动。
- 中心区展示目标名、返回和关闭。
- 禁用项不直接消失，要展示不可用原因。
- 高风险互动要有风险提示，而不是只靠颜色。
- 触屏和小屏优先保证可点击面积，不追求完整圆形。

## 4. 后续待办

- 增加菜单打开/关闭的轻动画，但不能影响响应速度。
- 给每个互动项展示“为什么推荐/为什么不推荐”的解释入口。
