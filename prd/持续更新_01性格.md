# 性格系统 PRD

> 本文维护【性格系统】的当前实现、配置协议、运行时接口和接入边界。系统级依赖图见 `prd/系统架构依赖图.md`。

## 1. 系统定位

性格系统回答两个问题：

- `这个人平常怎么反应`：由通用性格 `traitMetadata` 表达，例如乐天、敏感、孤僻、守时。
- `这个人独有的梗和执念是什么`：由人物专属性格 `specialtyMetadata` 表达，例如宝玉“见女儿痴语”、黛玉“临风洒泪”、凤姐“辣语压人”。

梦想系统已经独立为长期目标系统。梦想回答“这个人长期想把人生带到哪里”，不再混入性格效果解释。历史上梦想配置仍放在 `charSpecialtyConfig` 下，后续可迁到独立命名空间，但运行口径按梦想系统处理。

## 2. 当前实施状态

当前已经完成：

- 标准性格全部进入 `charSpecialtyConfig.traitMetadata`，并按 `性情 / 习惯 / 处世` 三类维护。
- 性格对偶关系使用 `oppositeTrait` 双向维护，后台【性格系统】按对偶分组展示。
- `TraitEffectSystem` 已实现统一解释层，读取 `traitMetadata.effects`。
- `TraitEffectSystem` 现已同时读取 `profiles[charId].specialties -> specialtyMetadata.effects`，人物专属性格也进入同一套底层解释。
- 后台【人物设定】使用三列性格配置，可按人物维护通用性格；人物专属性格作为只读标签展示，不再在该页暴露 JSON。
- 后台【性格系统】使用表格维护 `traitMetadata`，左侧 `ID / 标签` 冻结；高级 JSON 只保留低频维护入口。
- 基础需求、状态、关系、任务、社交、记忆、消费、竞赛、叙事气泡均已有性格接入点。
- `TraitBehaviorSystem` 已负责长期行为统计、记忆衰减、自主消费、竞赛结算和性格气泡。
- `CharacterLogicSystem` 已提供人物档案聚合接口，前后台可统一读取性格、需求、状态、职业、家庭和诊断信息。

仍需注意：

- `effects.schedule` 是预留/局部字段，尚未作为 `TraitEffectSystem` 的通用公开接口。
- `specialtyMetadata.effects.social` 中的 `targetCharId / observedCharId / watchedState` 等特殊字段是具体系统参数，不属于通用数值解释器；只有对应系统读取时才生效。
- `traitModifiers` 是旧 AI 权重兼容层，新配置优先写入 `traitMetadata.effects.actionWeights`。
- 不是所有配置字段都会自动影响行为。只有被本文接口或具体系统读取的字段才算运行时生效。

## 3. 配置协议

### 3.1 顶层结构

```js
charSpecialtyConfig: {
  traitLabels: { id: '中文名' },
  traitMetadata: { id: TraitMetadata },
  traitNarratives: { id: TraitNarrative },
  traitModifiers: { id: { boost: {}, cut: {} } },
  specialtyMetadata: { id: SpecialtyMetadata },
  profiles: { charId: CharacterTraitProfile },
  dreamMetadata: {},
  dreamProfiles: {}
}
```

### 3.2 通用性格 TraitMetadata

```js
{
  label: '敏感',
  oppositeTrait: 'chidun',
  category: '性情',
  description: '心思细腻，容易受伤。',
  effectExamples: ['关系值变化效果增强', '记忆更深且更难淡忘'],
  effects: {
    actionWeights: { comfort: 1.2, solitude: 1.15 },
    relation: { positiveMultiplier: 1.25, negativeMultiplier: 1.6 },
    stateDuration: { awkward: 1.35, melancholy: 1.2 },
    memory: { chanceMultiplier: 1.3, strengthMultiplier: 1.25, decayMultiplier: 0.85 }
  }
}
```

字段约定：

| 字段 | 说明 |
|---|---|
| `label` | 后台、前台展示名。 |
| `oppositeTrait` | 对偶性格 ID；后台修改时维护双方回指。 |
| `category` | `性情 / 习惯 / 处世`。旧名 `脾性 / 习性 / 社交` 会迁移到新名。 |
| `description` | 策划描述。 |
| `effectExamples` | 给策划看的效果摘要，不直接运行。 |
| `effects` | 运行时读取的结构化效果。 |

### 3.3 人物专属性格 SpecialtyMetadata

```js
{
  ownerId: 'baoyu',
  label: '见女儿痴语',
  category: '专属性格',
  description: '宝玉遇到女性社交对象时更容易主动开口。',
  systems: ['AI权重'],
  effectExamples: ['女性社交大类权重 x1.12'],
  effects: {
    actionWeights: { female_social: 1.12 },
    social: { femaleCategoryWeight: 1.12, maxMultiplier: 1.15 }
  }
}
```

ID 约定：

- 全局唯一专属性格可直接使用 `female_charm`。
- 多人物复用同名语义时使用 `charId.specialtyId`，例如 `baochai.mediate`、`sheyue.mediate`。
- `profiles[charId].specialties` 可以写 `mediate`，运行时会优先解析为 `${charId}.mediate`，再回退到全局 ID。

### 3.4 人物分配 CharacterTraitProfile

```js
profiles: {
  daiyu: {
    aiTraits: ['mingan', 'duochou', 'gupi', 'xijing'],
    displayTraits: ['敏感', '多愁', '孤僻', '喜静'],
    specialties: ['solitude_poetry', 'wind_tear', 'frail'],
    checks: {}
  }
}
```

字段约定：

| 字段 | 说明 |
|---|---|
| `aiTraits` | 运行时通用性格 ID。 |
| `displayTraits` | 兼容展示字段；后台可从 metadata 同步。 |
| `specialties` | 人物专属性格分配。 |
| `checks` | 旧专长激活条件兼容字段。 |

## 4. effects 支持字段

| 字段 | 当前口径 | 运行接口 |
|---|---|---|
| `actionWeights` | AI 候选标签倍率。 | `TraitEffectSystem.modifyActionWeight` |
| `needCoeffs` | 需求增长、衰退、上下限修正。 | `TraitEffectSystem.modifyNeedCoeffs` |
| `needThresholds` | 需求开始变得重要的个体阈值。 | `TraitEffectSystem.needAttentionMultiplier` |
| `stateChance` | 状态触发概率倍率。 | `TraitEffectSystem.stateApplyProbability` |
| `stateDuration` | 状态持续时间倍率。 | `TraitEffectSystem.modifyStateDuration` |
| `stateRecovery` | 状态恢复速度倍率。 | `TraitEffectSystem.stateRecoveryMultiplier` |
| `relation` | 关系变化倍率、关系轴倍率、背叛/败露倍率。 | `TraitEffectSystem.modifyRelationDelta` |
| `social` | 群聊加入、人群惩罚、邀约接受等社交修正。 | `socialJoinMultiplier` / `crowdPenaltyMultiplier` / `invitationAcceptanceChance` |
| `quest` | 任务权重、紧急任务权重、接受率、提前准备。 | `questWeightMultiplier` / `questAcceptanceChance` / `questEarlyPrepareMinutes` |
| `movement` | 移动速度倍率。 | `TraitEffectSystem.movementMultiplier` |
| `memory` | 记忆生成率、强度、负面强度、遗忘速度。 | `memoryChanceMultiplier` / `memoryStrengthMultiplier` / `memoryDecayMultiplier` |
| `money` | 自主消费概率、金额倍率、消费文案。 | `TraitEffectSystem.spendingProfile` |
| `competition` | 论道/争执等竞赛加成。 | `TraitEffectSystem.competitionBonus` |
| `furnitureNeeds` | 家具满足需求时的上限、拒绝、恢复区间等配置。 | 家具/需求适配层读取 |
| `schedule` | 起居、睡眠、截止节奏预留字段。 | 预留；不作为统一公开接口 |

通用规则：

- 倍率类效果采用相乘合并，并在接口内做上下限夹取。
- 概率类效果最终限制在合法概率区间。
- 正负关系变化分开处理。
- `effectExamples` 只做后台说明，不参与计算。
- 未被接口读取的字段可以作为策划参数保留，但不能宣称已运行生效。

## 5. 运行时接口

### 5.1 TraitEffectSystem

`TraitEffectSystem` 是性格效果解释器，不拥有业务流程，只返回倍率、概率或修正后的值。

```js
TraitEffectSystem.has(c, traitId)
TraitEffectSystem.traitsOf(c)
TraitEffectSystem.specialtiesOf(c)
TraitEffectSystem.metadata(id)
TraitEffectSystem.effectsOf(c)
```

说明：

- `traitsOf(c)` 返回 `profiles[c.id].aiTraits`。
- `specialtiesOf(c)` 返回解析后的专属性格 ID。
- `metadata(id)` 会先查 `traitMetadata`，再查 `specialtyMetadata`。
- `effectsOf(c)` 同时合并通用性格和人物专属性格。

AI / 行动：

```js
TraitEffectSystem.modifyActionWeight(c, cand, base)
TraitEffectSystem.decisionTrace(cand)
```

需求：

```js
TraitEffectSystem.modifyNeedCoeffs(c, coeffs)
TraitEffectSystem.needAttentionMultiplier(c, needKey, ratio)
```

状态：

```js
TraitEffectSystem.stateApplyProbability(c, stateId)
TraitEffectSystem.modifyStateDuration(c, stateId, duration)
TraitEffectSystem.stateRecoveryMultiplier(c, stateId)
```

关系：

```js
TraitEffectSystem.modifyRelationDelta(idA, idB, axis, delta, context)
```

移动 / 社交 / 任务：

```js
TraitEffectSystem.movementMultiplier(c)
TraitEffectSystem.socialJoinMultiplier(c)
TraitEffectSystem.crowdPenaltyMultiplier(c)
TraitEffectSystem.invitationAcceptanceChance(c, base)
TraitEffectSystem.questWeightMultiplier(c, urgent)
TraitEffectSystem.questAcceptanceChance(c, base)
TraitEffectSystem.questEarlyPrepareMinutes(c)
```

记忆 / 金钱 / 竞赛：

```js
TraitEffectSystem.memoryChanceMultiplier(c)
TraitEffectSystem.memoryStrengthMultiplier(c, valence)
TraitEffectSystem.memoryDecayMultiplier(c)
TraitEffectSystem.spendingProfile(c)
TraitEffectSystem.competitionBonus(c, category)
```

### 5.2 TraitBehaviorSystem

`TraitBehaviorSystem` 负责事件型和长期型行为，不直接解释配置表格。

```js
TraitBehaviorSystem.init()
TraitBehaviorSystem.addMemory(c, data, context)
TraitBehaviorSystem.decayMemories()
TraitBehaviorSystem.getStats(c)
TraitBehaviorSystem.resetStats(c)
TraitBehaviorSystem.showTraitBubble(c, event, context, preferredTraits)
```

当前职责：

- 按性格倍率创建和衰减记忆。
- 按 `money` 配置触发自主消费。
- 按 `competition` 配置结算论道/争执胜负。
- 监听 AI、互动、任务、邀约、记忆事件，累计长期统计。
- 按 `traitNarratives` 展示性格气泡。

### 5.3 CharacterLogicSystem

人物聚合读入口，供前台人物面板和后台人物设定使用。

```js
CharacterLogicSystem.profile(charId)
CharacterLogicSystem.exportManagedConfig(charId)
CharacterLogicSystem.charDef(charId)
CharacterLogicSystem.runtimeChar(charId)
```

`profile(charId)` 内含：

- `traits.ids / labels / specialties`
- `needs[].coeffs`
- `states[]`
- `skills[]`
- `family / identity / path / scene`
- `diagnostics[]`

## 6. 当前接入链路

### 6.1 AI 候选权重

```text
profiles.aiTraits + profiles.specialties
  -> TraitEffectSystem.effectsOf(c)
  -> actionWeights 匹配候选 cand.tags / kind / category
  -> modifyActionWeight()
  -> Utility AI finalWeight
  -> ai:decision 事件携带 traitEffectTrace
```

候选标签常用值：

```text
social / solitude / lively / comfort / conflict / movement / waiting / task /
taskUrgent / meal / kitchen / bath / wash / sleep / energy / outdoor /
indoor / desk / seat / lundao / hunger / hygiene
```

### 6.2 需求与基础生活

```text
人物 baseNeedCoeffs
  -> calcNeedCoeffs(c)
  -> TraitEffectSystem.modifyNeedCoeffs(c, coeffs)
  -> 需求衰退 / 恢复 / Utility AI 需求压力
```

后台【人物设定】已展示：

- 人物配置系数：`baseNeedCoeffs` 初始设定。
- 实际生效系数：按当前性格、状态和系统快照实时计算。

### 6.3 状态与情绪

```text
状态触发 / 状态应用 / 状态更新
  -> stateChance / stateDuration / stateRecovery
  -> TraitEffectSystem 对概率、持续时间、恢复速度做修正
```

### 6.4 关系与社交

```text
互动结果 / 任务结果 / 逾矩败露
  -> CharacterEffectSystem / RelationSystem
  -> TraitEffectSystem.modifyRelationDelta()
  -> 关系轴变化
```

```text
群聊 / 邀约 / 人群密度
  -> socialJoinMultiplier / crowdPenaltyMultiplier / invitationAcceptanceChance
  -> 对加入、拒绝、拥挤惩罚做个体化修正
```

### 6.5 任务与职业管理

```text
任务候选 / 任务接受 / 截止临近
  -> questWeightMultiplier / questAcceptanceChance / questEarlyPrepareMinutes
  -> 守时、拖延、多疑、轻信、周全等性格产生差异
```

### 6.6 记忆、金钱、竞赛和叙事

```text
事件记忆
  -> TraitBehaviorSystem.addMemory()
  -> memoryChance / memoryStrength / memoryDecay
```

```text
每小时自主行为
  -> TraitBehaviorSystem.trySpending()
  -> money.spendChance / amountMultiplier
```

```text
论道 / 争执结果
  -> TraitBehaviorSystem.settleCompetition()
  -> competition.bonus / categories
```

```text
行为、任务、记忆事件
  -> TraitBehaviorSystem.showTraitBubble()
  -> traitNarratives[event]
```

## 7. 后台维护口径

### 7.1 【性格系统】

用途：维护通用性格。

当前 UI：

- 表格化维护 `traitMetadata`。
- 左侧 `ID / 标签` 冻结。
- 支持搜索、新增、删除、ID 改名、对偶维护。
- 不展示完整 `traitMetadata` JSON。
- `traitNarratives`、`traitModifiers`、`specialtyMetadata` 作为高级 JSON，默认收起。

维护原则：

- 新性格必须有稳定 ASCII ID。
- 新性格必须归入 `性情 / 习惯 / 处世`。
- 新运行效果优先写入 `effects`，不要再新增 `traitModifiers`。
- 如果写了新 effect key，必须同时实现运行时解释器，否则只算配置备注。

### 7.2 【人物设定】

用途：维护单人物配置。

当前 UI：

- 可编辑性别、年龄、身份位阶、职业/阶段。
- 三列维护通用性格，按对偶 pair 二选一。
- 展示人物专属性格标签，但不在此页暴露 JSON。
- 基础需求表展示配置系数和实际生效系数。
- 梦想面板展示和维护人物长期志向。

人物专属性格维护位置：

- 定义表：后台【性格系统】高级 JSON `specialtyMetadata`。
- 分配表：当前仍在 `profiles[charId].specialties`，不建议在人物设定页直接改 JSON。

## 8. 新增性格流程

1. 选择稳定 ID。
2. 填写 `label / category / oppositeTrait / description / effectExamples`。
3. 只使用本文列出的 `effects` 支持字段。
4. 如果有对偶性格，维护双方 `oppositeTrait`。
5. 保存后在【人物设定】给人物分配。
6. 观察 AI 决策、需求系数、状态、关系和任务表现。
7. 如需上线给别人玩，使用后台“写回本地”把配置写入 `js/config.local.js` 并提交。
8. 批量导入或从表格生成时，使用 Codex skill `batch-generate-traits`；该 skill 会按当前 `TraitEffectSystem` 支持字段自动补齐 ID、分类、对偶、效果摘要和运行参数。

## 10. 调试与验收

后台和调试入口：

- 【性格系统】：确认性格配置、对偶、效果字段。
- 【人物设定】：确认人物分配、基础需求配置系数、实际生效系数。
- `CharacterLogicSystem.profile(charId)`：查看聚合档案和诊断。
- `TraitEffectSystem.effectsOf(c)`：查看当前人物通用性格 + 专属性格的运行 effects。
- `TraitBehaviorSystem.getStats(c)`：查看长期行为统计。
- `BehaviorTelemetry.watch(charId, true)`：观察候选池、权重和决策。

验收口径：

- 同一场景模拟一天，不同性格的行动分布应有可见差异。
- 生存需求危机仍高于性格偏好。
- 新 effect key 如果没有运行接口，不得在 PRD 中写成已生效。
- 人物专属性格的标准 effects 应进入 `TraitEffectSystem.effectsOf(c)`。
- 删除未知性格或专属性格时，AI 不应报错，只是不产生额外倍率。
