# 故事系统、人生路径与故事节点 PRD

> 更新时间：2026-06-25
> 主要代码：`js/systems/life-path.js`、`js/systems/fortune.js`、`js/systems/quest.js`、`js/admin.js`。

## 1. 总目标

故事系统负责人物的长期路径、阶段节点、剧情选择、失败风险和结局伏线。

它要支持：

- 每个人物可自动或手动进入一条路径。
- 路径由阶段、要求、日常任务、特殊行动、场景权限和故事节点组成。
- 完成任务、选择剧情、积累技能和声望后晋升阶段。
- 晋升改变身份展示、任务池、关系奖励、金钱或家族权限。

## 2. 与职业系统的边界

当前后台已拆为：

| 系统 | 职责 |
| --- | --- |
| 职业系统 | 职业身份、职责、日课、班次、随侍、上下级权限。 |
| 故事系统 | 人生路径、阶段节点、剧情选择、转线、失败和结局。 |

`lifePathConfig` 仍是现有代码承载路径/阶段/节点的配置容器，但后台展示上“职业”和“故事”分开，避免把丫鬟、小厮这类职业身份与长期人生故事混成一个入口。

## 4. 核心数据

### 4.1 路径 `paths`

| 字段 | 含义 |
| --- | --- |
| `id`、`name` | 路径 ID 与名称。 |
| `gender`、`rankRange`、`roleTags` | 可进入条件。 |
| `startStageId` | 初始阶段。 |
| `description` | 后台说明。 |

### 4.2 阶段 `stages`

| 字段 | 含义 |
| --- | --- |
| `title` | 当前称号。 |
| `requirements` | 技能、声望、关系、任务、故事节点要求。 |
| `dailyTaskIds` / `dailyTasks` | 阶段日常任务池，可投影到职业/起居。 |
| `specialActionIds` | 阶段特殊行动。 |
| `scenePermissions` | 阶段临时或永久通行权。 |
| `reputationDelta` | 晋升或事件声望变化。 |
| `nextStageId` / `nextStages` | 下一阶段。 |

### 4.3 故事节点 `storyNodes`

| 字段 | 含义 |
| --- | --- |
| `id`、`name` | 节点标识。 |
| `pathId`、`stageId` | 所属路径阶段。 |
| `trigger` | 触发条件：任务完成、关系阈值、状态、日期等。 |
| `choices` | 玩家或 AI 可选结果。 |
| `effects` | 关系、状态、记忆、金钱、路径标记、任务等奖励或代价。 |

## 5. 推进流程

```text
路径初始化
  -> 读取当前阶段
  -> 解锁日常任务 / 特殊行动 / 场景权限
  -> 任务或互动产生事件
  -> 更新技能、声望、关系、故事节点
  -> tryPromote 检查晋升
  -> 晋升后写历史、发奖励、刷新 UI
```
