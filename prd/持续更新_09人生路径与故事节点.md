## 1. 总目标

人生路径系统负责把人物从“日常模拟角色”推进成有长期目标、阶段称号、声望变化和关键故事节点的角色。

它要支持：

- 每个人物可自动或手动进入一条路径。
- 路径由阶段、要求、日常任务、特殊行动、场景权限和故事节点组成。
- 完成任务、选择剧情、积累技能和声望后晋升阶段。
- 晋升改变身份展示、任务池、关系奖励、金钱或家族权限。

## 3. 核心数据

### 3.1 路径 `paths`

| 字段 | 含义 |
|---|---|
| `id`、`name` | 路径 ID 与名称。 |
| `gender`、`rankRange`、`roleTags` | 可进入条件。 |
| `startStageId` | 初始阶段。 |
| `description` | 后台说明。 |

### 3.2 阶段 `stages`

| 字段 | 含义 |
|---|---|
| `title` | 当前称号。 |
| `requirements` | 技能、声望、关系、任务、故事节点要求。 |
| `dailyTaskIds` | 阶段日常任务池。 |
| `specialActionIds` | 阶段特殊行动。 |
| `scenePermissions` | 阶段临时或永久通行权。 |
| `reputationDelta` | 晋升或事件声望变化。 |
| `nextStageId` | 下一阶段。 |

### 3.3 故事节点 `storyNodes`

| 字段 | 含义 |
|---|---|
| `id`、`name` | 节点标识。 |
| `pathId`、`stageId` | 所属路径阶段。 |
| `trigger` | 触发条件：任务完成、关系阈值、状态、日期等。 |
| `choices` | 玩家或 AI 可选结果。 |
| `effects` | 关系、状态、记忆、金钱、路径标记、任务等奖励或代价。 |

## 4. 推进流程

```text
路径初始化
  -> 读取当前阶段
  -> 解锁日常任务 / 特殊行动 / 场景权限
  -> 任务或互动产生事件
  -> 更新技能、声望、关系、故事节点
  -> tryPromote 检查晋升
  -> 晋升后写历史、发奖励、刷新 UI
```
