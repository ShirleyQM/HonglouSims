# 秩序管理与第四轴玩法 PRD（讨论稿）

> 日期：2026-07-14
> 文档定位：基于现有关系 PRD、项目目标和“贾府秩序”原型，讨论新增秩序管理模块的玩法边界。  
> 重要边界：本模块不改现有 `友谊 friendship`、`姻缘 affection`、`信任 trust` 三轴，只围绕第四象限关系做玩法解释和管理反馈。

## 0. 一页结论

秩序管理模块要解决的是：

1. 看清秩序：谁是什么身份，谁和谁是什么血缘和上下级关系。
2. 管理归口：某类府内事务默认归哪个身份层级负责。
3. 解释后果：这个归口会影响传令、任务接受、礼法风险、关系第四轴变化。

主游戏入口建议放在顶部状态栏，按钮名为【秩序】。原因是秩序册面向“当前家庭 / 当前府邸”的全局制度视角，不是当前选中人物的个人面板。底部【人物】【关系】【技能】仍保留个人视角；顶部【秩序】负责打开全府视角。

它不直接做：

- 不直接发布具体任务。
- 不在这里批量勾选权限动作。
- 不替代传令面板。
- 不改写友谊、姻缘、信任三轴。

当前落地边界：

- `jiafu-order.html` 是秩序册视觉和信息架构原型。
- `js/jiafu-order-metadata.js` 已有秩序元数据：族谱、身份谱、权利定义、职位定义、事务定义。
- 主游戏已通过顶部【秩序】入口嵌入 `index.html` 和 `panel-overlay`，快捷键为 `Y`。
- `js/systems/order-book.js` 已接入第一版只读运行时：读取秩序元数据、人物身份、关系第四轴、技能值和任务归口，生成秩序册面板。
- 技能系统已在 `CONFIG.skillDefs` 中混合存在旧技能和新 9 项技能；秩序册只读取其中“管理事务相关技能”的子集。
- 当前版本已把第四轴接入第一版主仆日常互动合礼判定：贴身照拂类互动会读取服从/体恤，达标时视为合礼。
- 当前版本不直接改秩序全局属性，也不让综合关系标签产生 AI 影响；任务接受、关系结算和处罚仍以传令/任务系统为主。

## 1. 现有系统边界

### 1.1 关系四轴

当前关系系统已有四轴底座：

| 轴 | ID | 方向 | 本文处理方式 |
|---|---|---|---|
| 友谊 | `friendship` | 双向共享 | 不改。继续表示共同经历、平辈亲近、可玩笑可陪伴的程度。 |
| 姻缘 / 好感 | `affection` | 双向共享 | 不改。继续表示情感倾向、吸引、偏爱、厌恶。 |
| 信任 | `trust` | 双向共享 | 不改。继续表示可靠、嘴严、公平、是否值得托付。 |
| 第四象限 | `submission` / 派生显示 | 有方向 | 本模块重点。用于表达名分、上下、亲缘、主仆、客居、人情义理下的正负关系。 |

现有 PRD 已规定：

- `submission` 不进入综合关系分。
- 主仆中，仆从看主子显示“服从”；主子看仆从显示“体恤”。
- 亲缘中，晚辈看长辈显示“孝道”；长辈看晚辈显示“慈爱”。
- 其他关系第四象限暂时可留空。

本 PRD 的新增方向是：让“其他关系”也能在第四象限上有可玩语义，但不覆盖前三轴。

### 1.2 第四轴显示口径

第四轴不叫统一的“权力”或“职能”。它是一个底层方向性数值，但前台根据名分关系显示不同名字。

建议底层仍复用第四象限方向性轴，前台显示名由 `nominalRelation + 方向` 决定。

| 名分语境 | A 看 B 的第四象限显示 | 正向含义 | 负向含义 |
|---|---|---|---|
| 仆从 -> 主子 / 下级 -> 上级 | 服从 | 听命、恭顺、愿意执行 | 违逆、怠慢、顶撞、拖延 |
| 主子 -> 仆从 / 上级 -> 下级 | 体恤 | 宽待、照拂、护短、赏识 | 苛待、冷落、疑忌、压榨 |
| 晚辈 -> 长辈 | 孝道 | 敬重、守礼、愿受训诫 | 忤逆、敷衍、阳奉阴违 |
| 长辈 -> 晚辈 | 慈爱 | 疼惜、庇护、成全 | 冷待、嫌弃、压制、放弃 |
| 客居 / 亲戚 / 无明确上下 / 平辈 | 情分 | 念旧、顾面子、肯通融、互相撑场、同盟、敢共事 | 薄情、亏欠、疏离、不给面子、拆台、猜忌、互相甩锅 |

## 2. 秩序管理模块定位

“贾府秩序 / 皇宫秩序”是一个族谱面板，外加身份权限管理。

模块关系：

```text
秩序管理
  -> 定义身份层级与事务归口
  -> 影响传令候选和礼法解释
  -> 任务执行后反哺关系四轴

传令
  -> 发布具体命令
  -> 读取身份、事务归口、服从/信任/状态
  -> 生成任务实例

任务
  -> 执行、拖延、拒绝、完成
  -> 结算关系、状态、声望、故事条件
```

## 3. 管理分层

### 3.1 身份 role

身份回答：谁在制度上有资格管理。

| 身份 | 管理意味 |
|---|---|
| 至尊 | 可以裁断全府大事，越级发话。 |
| 长辈 | 可以训诫、安排晚辈、维护礼法。 |
| 平辈主子 | 可以管理自己院中起居和随侍。 |
| 晚辈主子 | 有轻量差遣权，但常需长辈背书。 |
| 体面仆从 | 可代主传话、调度职责内下人。 |
| 普通仆从 | 执行具体差事，不承担归口管理。 |
| 外人 | 需要邀请和通报，默认无府内管理权。 |

### 3.2 事务 transactions

事务回答：这类事默认谁负责。

- 责任：可以干嘛。
- 权利：享受什么。
- 通用权利：下级向上级，请安、贿赂、讨好等。

| 事务类型 | 默认归口 | 责任（具体职权） | 权利（享受/支配） | 决策效果（分配给不同人的博弈点） |
|:---|:---|:---|:---|:---|
| 礼法 | 至尊 / 长辈（贾母/王夫人） | 1. 定节庆规模、戏目、座次<br>2. 裁决“失礼”行为（如冲撞）<br>3. 批准外男/外眷觐见 | 通用：受晚辈请安、节礼<br>专属：最终解释权（说谁有理谁就有理） | 长辈亲管：法度 +5，体面 +3，但压制活力，发起活动需消耗“体面”。<br>交由平辈（如凤姐代管）：法度 -3，腐败 +5，易产生“拿着鸡毛当令箭”的怨气（人心 -2）。 |
| 功课 | 长辈（贾政/王夫人） | 1. 布置课业、请先生<br>2. 行使“禁足/罚跪”惩戒权<br>3. 检查读书进度 | 通用：质询权（随叫随到）<br>专属：“为你好”的支配权 | 严父/嫡母管：法度 +5，但情义 -5，宝玉类角色叛逆度上升。<br>放任给大丫鬟（如袭人）：法度 -3，人心 +2（宽松），但当贾政抽查时，体面 -5（责问主子失职）。 |
| 随侍 | 平辈主子 / 体面仆从（宝玉/凤姐/李纨） | 1. 安排谁值夜、梳洗、端茶<br>2. 调动小丫头和粗使婆子<br>3. 决定主子饮食偏好 | 通用：被服侍的享受<br>专属：随意打赏下人的额度 | 平辈主子（如宝玉）自管：情义 +5，人心 +3，但法度 -5（没规矩，作息混乱）。<br>交给体面仆从（如袭人）：法度 +2，但极易形成“垄断”，腐败 +5（贿赂才能见到主子）。 |
| 雅集 | 平辈主子（探春/黛玉/宝玉） | 1. 定诗社日期、主题、韵脚<br>2. 邀请谁参与、罚谁请客<br>3. 评判诗词优劣 | 通用：社交娱乐权<br>专属：风雅名声（体面）的增减 | 公中嫡女（探春）主持：法度 +3，体面 +5，庶出角色易借此提升地位。<br>才情高但孤僻者（黛玉）主持：情义 +3，但圈子窄，未受邀者人心 -3（觉得被冷落）。 |
| 账房 | 平辈主子 / 管事身份（凤姐/平儿/吴新登家的） | 1. 批对牌、发月钱<br>2. 采买大件、园中修缮<br>3. 执行“扣罚月钱” | 通用：经手银钱的油水<br>专属：拖延支付、克扣成色的隐性权力 | 嫡派管事（凤姐）管：法度 +3，腐败 +5（凤姐放贷），但人心 -5（怨声载道）。<br>交给旁支/厚道者（如李纨）：腐败 -3，人心 +5，但体面 -3（被下人认为好糊弄，拿不到足额采买）。 |
| 洒扫 | 体面仆从（林之孝家的/赖大家的） | 1. 排班次、换岗<br>2. 领取笤帚、灯笼等杂物<br>3. 催办外围杂役 | 通用：支配粗使丫头/婆子<br>专属：推荐自家亲戚进园子当差 | 交给“地头蛇”老仆：法度 -3，腐败 +5（吃空饷），但园内卫生极好（熟悉套路）。<br>交给年轻无根基者（如小红）：法度 +5，但人心 -3（老仆不服，背后使绊子）。 |
| 门禁 | 体面仆从 / 长辈授权（周瑞家的/林之孝家的） | 1. 通报何人来访、递送书信<br>2. 阻拦无牌之人出入<br>3. 传递内宅口信 | 通用：第一手信息的知情权<br>专属：“有选择地忘记通报”的截流权 | 交给太太陪房（周瑞家的）：法度 +3，但情报单向透明（王夫人势力 +5）。<br>交给中立/老太太多年的老人：法度 +5，但极其死板，走正规流程耗时 +2 天，遇急事易误事（体面 -3）。 |
| 人事（进/出） | 长辈 / 管家的 | 买丫头、配小厮、放人出去、升迁大丫鬟 | 推荐权、一票否决权 | 凤姐管：腐败 +5（收钱办事），但人员质量不稳。<br>贾母亲管：体面 +5，人心 +5，但凤姐积极性 -5（觉得被架空）。 |

### 3.3 全局属性与家族综合分

分配不同的人管事，最终会影响这些数值。它们彼此牵制，形成动态平衡。展示在秩序面板上。

| 属性 | 含义 |
|---|---|
| 法度（Order） | 规矩是否森严，流程是否合规。高则稳定，低则混乱。 |
| 人心（Morale） | 下人及旁支的满意度与忠诚度。高则执行力强，低则怨声载道、偷奸耍滑。 |
| 体面（Dignity） | 主子的威望和社交资本。高则压得住场，低则被轻视。 |
| 腐败（Purse） | 中饱腐败或人情往来的空间。高则灵活机动，低则财务严苛。 |

另外，4 个全局属性和整个家的人物综合分，计算家族综合分：

- 家族温情（Favor）：亲情与个人关系的黏合度。高则家族温情，低则冷漠算计。

基于这套表产生的策略博弈：

1. 信任与腐败的对立：把“账房”和“门禁”同时交给周瑞家的，短期法度 +6，但长期会形成“王夫人闭环”，主子的任何动向都被监控，且腐败会持续膨胀（雁过拔毛）。
2. 效率与规矩的对立：把“随侍”和“功课”都给宝玉自己管，人心 +5，情义 +5，但法度 -10，这会触发贾政的“突击检查”事件，导致体面巨幅崩塌。
3. 制衡策略：聪明的玩家会把“账房”给凤姐（为了效率），但把“人事”给贾母（为了品质），并安排“雅集”给探春（为了监视凤姐克扣银两时的借口）。
4. 特殊角色策略（林黛玉）：玩家若让黛玉掌管“礼法”，虽然她懂规矩，但因为性格孤高，法度 +3，但人心 -8（下人觉得伺候她太累，规矩太细碎）。若让她管“雅集”，则体面 +10，直接提升整个家族的文学格调。

### 3.4 人物属性与技能

生成一套人物技能，作为不同人管理的效果。

重要口径：秩序册里的技能不是另一套独立技能系统，而是全技能系统的一个子集。主游戏的全技能可以包含移动、交谈、诗词、女红、医药、茶道、账房等更细的技能；秩序册只拿“能解释府内管事能力”的管理子集来计算和展示。

秩序技能子集暂定为：

| 秩序技能 | ID | 是否来自全技能 | 在秩序册里的用途 |
|---|---|---|---|
| 规矩 | `ritual` | 是 | 礼法、门禁、座次、训诫是否合规。 |
| 文墨 | `literacy` | 是 | 账册、文书、功课、雅集的理解和判断。 |
| 察言 | `insight` | 是 | 判断上意、来客亲疏、任务接受度。 |
| 办事 | `affairs` | 是 | 排班、采买、洒扫、筹备、落地执行。 |
| 心计 | `scheming` | 是 | 制衡、压怨气、账房和人事博弈。 |
| 口才 | `eloquence` | 是 | 传话、催办、圆场、训诫和请托。 |
| 勤勉 | `diligence` | 是 | 当值、盯工、长时任务、细活稳定性。 |
| 藏锋 | `subtlety` | 是 | 暗中传话、避嫌、账房和人事不露痕迹。 |
| 通变 | `adaptability` | 是 | 临时改派、危机救场、任务条件变化。 |

不进入秩序技能子集的例子：

- `move` 行走：基础能力，只影响能不能抵达，不代表管事能力。
- `talk` 交谈：基础社交能力，后续可迁入口才，但秩序册不单独展示。
- `poetry` 诗词、`craft` 女红、医药、茶道等：是全技能或二级专精，可在具体事务里加成，但不作为秩序册主轴。
- `finance` 金融：后续可作为账房专精，秩序册主公式仍优先读取文墨、办事、心计、藏锋。

底层人物（如小红、刘姥姥）技能点满后，确实能把事办得漂亮，但身份带来的体面 Debuff 和行动合法性限制会一直伴随。相反，主子（如宝玉）哪怕技能稀烂，只要身份在那，仆从也不敢明面违抗，但实际执行效果会大打折扣。

所有人物都共享这 9 项基础技能，决定其办事效率和质量：

| 技能名称 | 核心能力 | 对应管理事务（匹配表） | 高技能表现 | 低技能表现 |
|:---|:---|:---|:---|:---|
| 规矩（Ritual） | 懂礼数、知旧例、仪式流程 | 礼法、门禁 | 安排座次滴水不漏，通报流程合规 | 闹出“坐错席面”的笑话，通报丢三落四 |
| 文墨（Literacy） | 识字、诗词、书写、算学 | 功课、雅集、账房 | 诗社判词令人信服，账目清晰 | 把账簿念成流水账，批复写错别字 |
| 察言（Insight） | 看眼色、听弦外音、揣摩上意 | 随侍、门禁 | 主子一个眼神就知道递茶，能判断来访者亲疏 | 撞枪口上，传话传错意思 |
| 事务（Affairs） | 动手能力、排班、采买、杂务 | 洒扫、医药、人事 | 采买东西物美价廉，人手调配井井有条 | 买个烛台被坑三倍价，排班乱成一锅粥 |
| 心计（Scheming） | 博弈、算计、利益权衡 | 账房、人事、礼法 | 能用最少银两办最大事，借力打力 | 实心眼，被人卖了还帮着数钱 |
| 口才（Eloquence） | 说话讨喜、辩解、说服、吵架 | 门禁、账房（讨债）、随侍 | 把人哄得团团转，说辞滴水不漏 | 一开口就得罪人，要钱反被骂回来 |
| 勤勉（Diligence） | 不偷懒、执行力、盯梢 | 洒扫、医药、随侍 | 凌晨就起来盯婆子扫雪，药煎好亲自试温 | 能躲就躲，煎药都能睡着了烧干锅 |
| 藏锋（Subtlety） | 不露富、不招摇、低调做事 | 人事、账房、随侍 | 经手大笔银两却不招人眼红，提拔人而不结党 | 四处显摆，被同僚举报 |
| 通变（Adaptability） | 临场应变、救急、破例 | 所有事务（危机时） | 宴席突降大雨，能立刻改成室内曲水流觞 | 原地干瞪眼，等主子发话 |

### 3.5 身份特权与 Debuff

技能决定“能不能办好”，身份决定“别人认不认”和“有多大的自主权”。

| 身份层级 | 代表人物 | 特权（Buff） | 身份枷锁（Debuff） |
|:---|:---|:---|:---|
| 至尊 / 长辈 | 贾母、贾政、王夫人 | 一言九鼎：即使技能值为 0，发话也无人敢违抗，可否决任何归口安排。 | 高坐堂：远离一线，对琐事细节判定需消耗“体面”值才能获知真相。 |
| 平辈主子 | 宝玉、黛玉、探春、凤姐、李纨 | 生杀予夺：有罚跪、扣月钱、撵人的合法权力。无需解释。 | 影子博弈：任何管理行为都会被视为“揽权”或“越位”，引发其他主子暗中掣肘。 |
| 体面仆从 | 袭人、平儿、鸳鸯、周瑞家的 | 近水楼台：信息最灵通，可“选择性传达”，实际执行空间极大。 | 僭越风险：若技能不足以服众（如袭人管账），老仆会在背后骂“你算什么东西”。 |
| 底层 / 外围 | 小红、刘姥姥、坠儿、老嬷嬷们 | 隐形人优势：没人盯着，反而能做成主子不便出面的事（如打听消息）。 | 人微言轻：办成事功劳归上面，办砸了锅全自己背；需消耗双倍“口才”才能让主子听见真话。 |

### 3.6 综合管理效果公式

```text
最终执行效果 = 人物技能对应值 × 身份信任系数 - 身份 Debuff 阻力
```

- 身份信任系数：主子管琐事（系数 0.8，因为不接地气），仆从管礼法（系数 0.5，因为不够格）。
- 简易判定：同一件事，主子用 20 点技能就能办，仆从可能需要 60 点技能才办到同样效果，但仆从做油水事务（账房、洒扫）有天然系数加成。

2. 探春 + 李纨：让探春管账房（改革心切），但配李纨管人事（藏锋 + 勤勉）。李纨的“藏锋”会中和探春的锋芒，避免老仆集体反弹。组合效果：账目清理顺利，且无人告黑状。
3. 黛玉 + 刘姥姥：让黛玉管雅集（文墨 100），但让刘姥姥负责“洒扫”场地布置（事务 80）。刘姥姥会把园子布置得野趣横生，反而激发黛玉诗兴。效果：诗社质量 +30%，且花费极低。唯一的坏处是：宝玉会笑“把大观园变农家乐了”。

### 3.10 失败处置与赏罚权限

秩序册不直接执行处罚，但要回答“谁有资格处罚、处罚是否合规、钱项是否需要账房背书”。

任务失败后的处置从传令面板发起；秩序册提供判断依据：

| 判断项 | 秩序册提供什么 |
|---|---|
| 上下级关系 | 是否直属主仆、职业上下级、家族长幼、跨院平辈。 |
| 事务归口 | 这件事按规矩归谁管，如账房采买、起居侍奉、功课礼法。 |
| 钱项权利 | 发布者是否有 `account_control`、`steward_dispatch` 等可执行扣月银/赔偿的权利。 |
| 礼法权威 | 发布者是否有训诫、禁足、罚抄等非钱项处罚资格。 |
| 越权风险 | 若强行处罚，会不会降低体面、人心或提高腐败/风险。 |

当前实现口径：

- 普通任务失败的自动数值反馈已调轻，避免一次失败导致主仆关系断崖式下降。
- 更重的处罚必须来自传令失败处置：宽恕、训斥、扣月银、补做、禁足、罚抄等。
- 秩序册负责解释“谁有资格这样罚、钱项是否需要账房背书、是否越权”，不直接执行处罚。
- 个人月银账户完整上线前，“扣月银”优先作为处置策略和埋点/待处理记录，不应伪装成已经完成的个人账户扣款。

### 3.11 第四轴与主仆合礼互动

第四轴已经接入第一版社交礼法判定，主要用于解决“贴身仆从合理照拂主子却被系统当作逾矩”的问题。

规则：

```text
仆从 -> 主子：读取 submission，前台解释为服从。
主子 -> 仆从：读取反向 submission 的负值，前台解释为体恤。
服从和体恤同时达标，且互动属于日常照拂范围，才可豁免主仆礼法风险。
```

第一版阈值：

| 接触 | 服从 | 体恤 | 互动类型 |
|---|---:|---:|---|
| 无接触 | 25 | 15 | 倾听、安慰、陪谈、评文。 |
| 低风险接触 | 35 | 25 | 照拂、拍背、按揉。 |

边界：

- 只影响主仆日常照拂，不影响情爱、调笑、争执。
- 只降低礼法风险，不直接改变关系结果。
- 综合关系标签如“心腹忠仆”“离心父子”暂不产生 AI 影响，避免策划改名后牵动行为逻辑。

示例：

- 凤姐处理采买失败：合规，可训斥、扣月银、要求赔偿；但频繁扣罚会提高苛待/腐败风险。
- 宝玉处理袭人侍奉失败：可宽恕、训斥、要求补做；扣月银需要看怡红院月银是否由宝玉掌管，否则应交给王夫人/凤姐执行。
- 黛玉处理紫鹃传话失败：可训斥或宽恕；扣月银暂应视为需要管账人背书。
- 平辈主子跨院处罚他院仆从：通常越权，可记录不满或请求管事，不应直接扣钱。

因此，赏罚权限应输出给传令面板，而不是在秩序册里新增一套操作表单。

## 4. 第四轴如何参与玩法

第四轴的价值不是“多一个好感条”，而是让秩序管理产生社会后果。

## 5. 接入主游戏的位置

### 5.1 顶部入口

主游戏顶部 `#status-bar #top-actions` 当前已有：

- 日志
- 设置
- 帮助
- 切换家庭
- 存档

已在【日志】之后、【设置】之前新增：

```html
<button type="button" class="top-btn" id="btn-order">秩序<span class="menu-key">Y</span></button>
```

快捷键使用 `Y`，避免和已有 `F/J/P/Q/R/K/M/B/S/H/V` 冲突。

入口命名：

- 前台玩家：`秩序`
- 面板标题：`贾府秩序`
- 后续切换家庭时：`{家庭名}秩序` 或 `{府邸名}秩序`

### 5.2 面板层级

秩序册应该复用主游戏已有 `panel-overlay`，而不是新开一个全屏独立页面。

原因：

1. 玩家打开秩序册时仍在同一局游戏中，需要读当前家庭、人物、关系和任务状态。
2. `jiafu-order.html` 的完整页面结构适合做原型，但接入主游戏后应拆成 `buildOrderPanel()` 这种 HTML 片段。
3. 后续秩序调整会产生游戏内影响，必须和 `CONFIG`、`CHARS`、`relations`、任务系统共享运行时。

当前前台模块：

```text
js/systems/order-book.js
  - OrderBookSystem.openPanel()
  - OrderBookSystem.buildPanel()
  - OrderBookSystem.metadata()
  - OrderBookSystem.getAffairAssignee(affairId)
  - OrderBookSystem.getAffairControllers(affairId)
  - OrderBookSystem.estimateAffairFit(charId, affairId)
  - OrderBookSystem.describeOrderFit(issuerId, targetId, templateId)
  - OrderBookSystem.inferAffairFromTemplate(template)
```

原型资源复用：

- 保留 `jiafu-order.html` 作为视觉试验场。
- 迁移 `js/jiafu-order-metadata.js` 为运行时数据源；短期可直接引入，长期并入 `CONFIG.orderBookConfig`。
- 把原型里的三个主 tab 收敛成前台秩序册 tab，不和后台配置混在一起。

### 5.3 推荐面板结构

秩序册前台建议 4 个 tab：

| Tab | 玩家问题 | 读取系统 |
|---|---|---|
| 血缘谱 | 谁是谁的亲戚、长辈、晚辈、姻亲？ | `relationPanelConfig.familyRelations`、`familyConfig`、`jiafu-order-metadata.sourceGenealogy` |
| 身份谱 | 谁在府里压得住谁？谁能管谁？ | `identityProtocolConfig`、`peoplePositions`、`positionDefs` |
| 事务归口 | 礼法、账房、洒扫、门禁归谁管？ | `affairDefs`、`rightDefs`、任务系统 |
| 管事评估 | 某人管某事合不合适？会有什么后果？ | 技能子集、身份、信任、第四轴、状态 |

其中【事务归口】是未来玩法核心，但第一阶段可以只读展示，不允许玩家改派，避免系统一次性过重。

## 6. 和现有系统的结合

### 6.1 与关系系统

秩序册读取关系系统，但不替代关系面板：

- 血缘谱读取家庭关系和名分关系。
- 身份谱读取身份位阶和家庭角色。
- 第四轴用于解释主仆、亲缘、客居、平辈情分下的服从/体恤/孝道/慈爱。
- 友谊、姻缘、信任仍在关系面板里表达人物之间的情感和信任，不由秩序册改写。

秩序册可以提供跳转：

- 点击人物：打开人物面板。
- 点击两人连线：打开当前两人的关系面板或关系详情。
- 点击第四轴解释：展示“为什么这个任务可能被拖延/拒绝/高质量完成”。

### 6.2 与传令和任务系统

秩序册不直接发布具体任务，但影响传令和任务解释。它可以提供“去传令”的跳转按钮，但点击后必须打开传令面板，由传令面板完成选人、选任务、确认发布。

| 秩序数据 | 传令/任务中的用途 |
|---|---|
| 事务归口 | 决定“谁名义上该管这件事”。 |
| 权利 `rightDefs` | 决定命令是否越权、是否需要背书。 |
| 身份 `positionDefs` | 决定命令的合法性和对方默认接受程度。 |
| 技能子集 | 决定任务质量、耗时、失败概率。 |
| 第四轴 | 决定执行意愿、拖延、顶撞、阳奉阴违。 |

三层边界：

| 层 | 前台入口 | 核心动作 | 不能做什么 |
|---|---|---|---|
| 秩序 | 顶部【秩序】 | 查看制度、事务归口、身份谱、管事适配、越权解释 | 不直接创建任务实例，不批量勾人发布。 |
| 传令 | 底部【传令】/人物互动【传令】/当值【令】 | 选择目标、选择任务、确认发布、展示接受意愿和风险 | 不维护事务归口，不改职位制度。 |
| 任务 | 【任务】面板和 NPC 行动队列 | 接受、拒绝、执行、完成、失败、结算 | 不负责判断谁名义上该管。 |

最小闭环：

1. 玩家打开【秩序】只读查看事务归口。
2. 玩家在秩序册里看到“账房采买：凤姐负责”，可点击“去传令”。
3. 系统打开【传令】并预选事务类型或建议目标，但不自动发布。
4. 玩家通过【传令】确认发布任务。
5. 传令系统读取事务归口和身份规则，给出“合理 / 越权 / 需背书 / 易结怨”的解释。
6. 任务完成后结算关系轴、状态、声望和日志，并把长期影响反馈到秩序指标。

推荐数据流：

```text
OrderBookSystem
  -> getAffairOwner(affairId)
  -> describeOrderFit(issuerId, targetId, templateId)
  -> estimateAffairFit(charId, affairId)

CommandPanel
  -> 读取 OrderBookSystem 的解释
  -> 调用 ManagementAuthoritySystem / QuestIssueSystem 过滤候选
  -> 调用 QuestIssueSystem.issueTo / issueGroupTo

QuestSystem
  -> 创建任务实例
  -> 执行与结算
  -> EventBus 反哺关系、声望、秩序指标
```

需要避免的混乱：

- 不要在秩序册里也做一套“发布任务”表单。
- 不要让传令面板维护职位、权利和事务归口配置。
- 不要让任务模板自己解释全部礼法；任务模板只标明 `category/affairId/requiredRelation` 等字段，礼法解释交给管理权限层。

### 6.3 与技能系统

秩序册只展示“事务相关技能匹配度”，不展示完整技能面板。

展示方式建议：

```text
账房采买：凤姐
匹配：心计 95 / 办事 85 / 口才 90 / 藏锋 60
风险：藏锋偏低，容易招怨；身份足够，执行强。
```

计算时读取全技能系统：

```text
事务匹配度 =
  Σ(秩序技能值 × 事务权重)
× 身份合法性系数
× 信任/第四轴意愿系数
× 当前状态修正
- 越权/僭越/客居阻力
```

注意：全技能里非秩序子集的技能可以做专项加成。例如诗词高可加成雅集，女红高可加成针线差事，医药高可加成侍药，但它们不是秩序册固定 9 项主轴。

### 6.4 与后台配置

后台后续应新增【秩序系统】配置，不放在前台秩序册里。

后台配置项：

- 权利定义 `rightDefs`
- 职位/身份定义 `positionDefs`
- 人物职位映射 `peoplePositions`
- 事务定义 `affairDefs`
- 事务技能权重 `affairSkillWeights`
- 事务默认归口 `affairAssignees`
- 身份合法性系数 `positionAffairCoeffs`

前台只展示结果和解释；后台负责策划编辑。
