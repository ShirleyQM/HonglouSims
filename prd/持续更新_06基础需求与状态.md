# 基础需求与状态 PRD

## 2026-06-21 置顶补充：基础需求进入作息层

基础需求不再只作为“数值低了就救火”的压力源，也要参与日常节律。AI 作息层读取六项需求并给候选动作加权：

- 饥饿：早餐、午餐、晚餐三段锚点；正餐优先，点心兜底。
- 精力：夜间睡眠强锚点，白天短休弱锚点。
- 洁净：晨起梳洗锚点，爱洁/邋遢调整权重但不取消需求。
- 意趣：上午、下午、晚上通过读书、园景、乐器、闲谈、任务等柔性满足。
- 交游：下午与晚上是自然社交窗口，低交游可额外触发主动寻人。
- 心绪：由睡眠、梳洗、进食、慰藉、独处、读书、园景等间接维护。

验收时不只看最终需求值，也要看一天内行为分布：三餐、晨起洁净、夜间睡眠、白天意趣/社交是否形成可读循环。

## 1. 总目标

基础需求系统是人物日常行为的底层压力源。它要回答三件事：

- 人物现在缺什么：饥、倦、闷、洁、交游、心绪。
- 缺到什么程度会改变行为：需求衰退、AI 候选加权、紧急抢占、状态标签。
- 满足需求后留下什么长期差异：习惯适应、性格阈值、状态恢复、叙事气泡和行为记忆。

## 3. 数据口径

### 3.1 需求定义

需求配置维护在 `CONFIG.needDefs`、`needDecayPerGameDay` 与人物 `baseNeedCoeffs`。

| 字段 | 含义 |
|---|---|
| `key` | 需求 ID。 |
| `name` | 前台展示名。 |
| `defaultGrow` / `defaultDecay` | 默认恢复和衰退系数。 |
| `stateBands` | 需求分档状态，通常为 `<20`、`20-39`、`80-99`、`>=100`。 |
| `baseNeedCoeffs` | 人物个体差异，含 `min/max/grow/decay`。 |

### 3.2 状态定义

状态统一在 `CONFIG.stateDefs` 中维护，CSV 注入由 `js/systems/state-defs.js` 完成。

| 字段 | 含义 |
|---|---|
| `name` / `desc` | 展示名和说明。 |
| `duration` | 持续游戏分钟。 |
| `category` | mood、need、social、task 等分类。 |
| `polarity` | positive、negative、mixed、neutral。 |
| `conflictGroup` | 同组状态互斥。 |
| `needMods` | 需求系数修正。 |
| `aiModifiers` | AI 标签权重修正。 |
| `blockedSkills` | 阻塞技能或行动能力。 |
| `traitEffects` | 特定性格追加效果。 |

## 4. 运行流程

```text
游戏时间推进
  -> 自然衰退需求
  -> 需求变化事件
  -> CoreNeedSystem 计算交游/心绪牵引
  -> NeedStateSystem 同步分档和组合状态
  -> StateSystem 更新状态时长、冲突和容量
  -> Utility AI 读取需求、状态、性格、家具恢复
```

## 5. AI 接入规则

- 低需求通过 `demandFactor`、`CoreNeedSystem.actionFactor` 和性格阈值提高相关候选。
- 极低需求可以触发紧急行为，优先寻找可恢复家具。
- 状态通过 `stateDefs.aiModifiers` 影响标签候选，例如心绪低时提高独处、求慰藉或争执概率。
- 满足或超额状态不是纯奖励，可能抑制继续使用同类家具，避免人物一直吃饭或一直洗漱。

## 6. 后台入口

- 【人物 / 需求】：维护需求定义、默认系数和分档状态。
- 【人物 / 状态】：维护 `stateDefs`、需求状态矩阵、组合状态和高级 JSON。
- 【人物 / 性格系统】：维护性格对需求系数、状态、阈值的影响。
