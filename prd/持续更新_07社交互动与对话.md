> 更新时间：2026-06-21
> 来源：`0609_02_对话引起状态变化.md`、`0609_03_对话轮次和群聊.md`、`0612_05_新增社交互动.md`、当前代码。
> 主要代码：`js/systems/interaction-social.js`、`js/systems/interaction-score.js`、`js/systems/dialogue.js`、`js/systems/interaction-llm.js`、`js/systems/ai-drama.js`、`js/ui.js`。

## 1. 总目标

社交互动系统负责把“点一个人”变成可解释、有风险、有后果的互动链：

- 互动能不能做：由对象、身份礼法、冷却、技能、场景和硬条件判断。
- 人物愿不愿意做：由关系四轴、状态、性格、心情和戏剧意图影响。
- 做完发生什么：关系轴、状态、需求、记忆、叙事气泡和后续 AI 意图更新。
- 文本如何表现：模板对白优先，可切换本地 LLM 对话。

## 3. 互动模板协议

`CONFIG.interactionTemplates` 是社交内容的主表。

| 字段 | 含义 |
|---|---|
| `id`、`name`、`category` | 模板标识、展示名、互动类别。 |
| `type` | `dialogue` 或 `action`。 |
| `duration`、`cooldown`、`once` | 时长、冷却、一次性限制。 |
| `lines` | 模板对白。 |
| `llmPrompt` | 连接 `prompts/interaction-topics.txt` 或模型提示标签。 |
| `effects` | 需求、状态、关系、记忆等效果。 |
| `axisEffects` | 关系四轴变化。 |
| `axisReq` | 关系软门槛。 |
| `onLowScore` | 意愿低或软锁失败时的收场。 |
| `unlock_conditions` | 硬条件，例如状态、家具、性别、年龄关系。 |
| `is_risky`、`risky_details` | 逾矩风险和成功/失败状态。 |

## 4. 判定流程

```text
发起互动
  -> InteractionSocialSystem.evaluate 硬条件
  -> IdentityProtocolSystem 计算身份礼法与接触风险
  -> InteractionScoreSystem 计算关系意愿和低分提示
  -> 菜单展示意愿强弱
  -> 执行互动
  -> DialogueSystem 或 InteractionLlmSystem 生成对白
  -> ActionQueue 结算效果
  -> Relation / State / Need / Memory 更新
  -> AiDrama / MultiInteract / Narrative 接收事件
```

## 5. 关系意愿口径

互动不再因关系值低而简单消失。当前口径：

- 世界规则仍然硬拦：无效对象、不能说话、硬禁忌、技能不足、冷却、场景条件。
- 关系值决定“自然程度”和“风险收场”，可显示为意愿高、中、低、冷。
- `onLowScore` 负责尴尬、难过、拒绝、关系下降等后果。

## 6. LLM 对话口径

LLM 是表现层，不改变底层规则：

- 开关和配置维护在 `narrativeBubble.settings.llm`。
- 默认 provider 为本地 Ollama，可通过 `serve.mjs` 代理。
- 模型生成失败时回退模板对白。
- 模型段落可以按轮次应用轻量关系效果，但不直接绕过互动模板的主效果。
- `prompts/interaction-global.txt` 和 `prompts/interaction-topics.txt` 是文风主入口。

## 7. 后续待办

- 群聊的“分裂成小圈子”和“私语化”仍需正式状态。
- LLM 对话需要更清晰的安全字段：可输出情绪、关系倾向，但不能直接执行任意效果。
- 互动模板后台应增加批量查重、ID 分段、冷却合理性检查。
- 争执、安慰、拒绝、心动等结果链可沉淀为可复用 Outcome 表。
