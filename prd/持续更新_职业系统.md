## 古代人生路径与声望体系 PRD（通用框架）

### 1. 设计理念
构建一个**开放式、多路径**的古代身份晋升系统，让角色不仅在大观园内“升职”，还可选择走出府邸，踏上士农工商、选秀入宫等截然不同的人生道路。
核心原则：
- **路径多样**：丫鬟可成为女官、商人、贵妃；公子可科举、从军、归隐。
- **条件明确**：每条路径有独立的声望、技能、关系、事件门槛。
- **自由切换**：玩家可随时改变目标（但有一定成本），路径间并非完全互斥。
- **与现有系统深度集成**：声望影响关系、任务、对话；身份决定场景权限与互动选项。
- **配置驱动**：所有路径、晋升条件、奖励均由数据表定义，易于扩展新职业。

### 2026-06-25 补充：职业、起居与任务的架构关系

职业系统负责回答“这个人是什么身份、处在什么人生阶段、每天天然承担什么职责”。任务系统负责把职责实例化并推进。起居系统负责把稳定职责可视化成一天的时间结构。三者不互相替代。

核心口径：

- `丫鬟`、`小厮`、`管事`、`贴身侍女`、`房中丫鬟` 都属于职业/职责身份。
- 职业身份可以来自多处：人生路径阶段、家族角色、仆从职责契约、后续职业配置。
- 职业阶段可以提供 `dailyTasks`、`directiveTasks`、`eventTasks`，其中 `dailyTasks` 会投影到起居职业行。
- 仆从职责契约中的 `dutyRoutines` 与 `followRotations` 也视为职业日课来源。
- 临时任务不是职业日课，不进入起居；它只作为执行层的插队任务。
- 随侍是职业日课中的目标跟随型职责，属于职业行起居块，而不是普通社交活动。
- 整体系统依赖图单独维护在 `prd/系统架构依赖图.md`。

职业日课的推荐数据口径：

```js
{
  professionId: 'servant_personal',
  roleTags: ['丫鬟', '贴身侍女'],
  dailyTasks: [3001, 3003, 3004],
  followRotations: [3005],
  routineProjection: {
    rowId: 'profession',
    sourceType: 'lifePathDaily',
    lockedByDefault: true
  }
}
```

优先级原则：

```text
严重需求危机 / 安全
  > 强制剧情 / 高优先临时任务
  > 随侍轮值 / 职业日课
  > 普通起居活动
  > 闲逛 / 普通社交 / 随机家具
```

### 2. 核心概念定义

#### 2.1 声望值（Reputation）
- **全局声望**：角色在整个游戏世界中的名气，0-1000，分为五阶（默默无闻→名满天下）。
- **领域声望**（可选扩展）：如“文名”“商誉”“武勋”，用于特定路径，初期可统一用全局声望。

#### 2.2 人生路径（LifePath）
- 一个角色当前追求的长期目标，决定其任务类型、技能倾向、解锁场景。
- 路径可随时更换，但更换时声望会受轻微损失（“半途而废”）。

#### 2.3 身份等级（SocialRank）
- 沿用原有 `SocialRank`（0-6），但路径晋升时可以**动态改变**该值，且不同路径的等级含义不同。
- 例如：丫鬟（5级）通过“女官路径”可晋升至2级（宫廷女官），地位相当于主子。

#### 2.4 职业/行业（Profession）
- 路径下的具体阶段，每个阶段有头衔、任务池、特权。
- 在大观园内部，职业也包括职责身份，例如丫鬟、小厮、管事、贴身侍女、房中丫鬟。它们可以不改变人生路径，但会决定每日职责、任务权限、起居职业块和 Utility AI 倾向。

### 3. 数据模型

#### 3.1 人生路径配置表（LifePathConfig）
```json
{
  "path_id": "merchant_path",
  "name": "商贾之路",
  "description": "从商贩到巨贾",
  "required_initial_rank": [5,6],   // 谁可以开始此路径（社会底层或外人）
  "stages": ["stage_1", "stage_2", "stage_3"],  // 阶段链
  "default_skills": ["口才","算学"], // 倾向技能
  "ending_scenes": ["own_shop"],    // 成功后解锁的场景
  "quitting_penalty": -50           // 退出路径声望损失
}
```

#### 3.2 阶段配置表（PathStageConfig）
```json
{
  "stage_id": "stage_1",
  "title": "货郎",
  "rank_override": 5,              // 此阶段对应的 SocialRank
  "required_reputation": 50,
  "required_skills": {"口才": 2},
  "required_items": {"货担": 1},   // 需要拥有特定道具
  "daily_tasks": ["叫卖","进货"],   // 解锁的日常任务池
  "special_actions": ["讨价还价"], // 解锁的互动
  "scene_permissions": ["集市"],    // 解锁场景
  "next_stages": ["stage_2"],
  "abandon_conditions": "破产"     // 若金钱归零则失败
}
```

#### 3.3 角色运行时数据扩展
在 `CharacterData` 中增加：
```js
{
  life_path: 'merchant_path',      // 当前路径
  current_stage: 'stage_1',        // 当前阶段
  reputation: 120,                 // 全局声望
  profession_history: []           // 历史职业记录
}
```

### 4. 路径设计：士农工商 + 特殊

#### 4.1 士（科举/官员）
- **起点**：公子（2-3级）、读书人（4级）
- **阶段**：童生→秀才→举人→进士→翰林/外放
- **条件**：文学、经义技能，乡试/会试事件（每年触发），需特定 NPC 推荐
- **特权**：免除部分劳役，可进入官场场景，影响家族声望

#### 4.2 农（田庄/地主）
- **起点**：普通仆从（5级）、平民（6级）
- **阶段**：佃农→自耕农→小地主→大地主
- **条件**：种植/农事技能，积累银两购置田地，与官府关系
- **特权**：产出粮食可供给家族或出售，影响家族食物储备

#### 4.3 工（工匠/手艺）
- **起点**：仆从（5级）、平民（6级）
- **阶段**：学徒→匠人→名匠→御用工匠
- **条件**：对应技艺技能（女红、木工等），完成作品任务，声望达标
- **特权**：可制作高级物品、承接官府订单，解锁特殊工坊场景

#### 4.4 商（行商/富商）
- **起点**：平民（6级）、丫鬟（5级赎身后）
- **阶段**：货郎→店主→豪商→皇商
- **条件**：口才、算学技能，本金，完成一定数量贸易任务
- **特权**：可开设店铺，影响家族收入，可资助其他角色

#### 4.5 特殊路径
- **选秀入宫**：女性角色，需容貌、才艺，经选拔事件入宫，路径为宫女→女官→嫔妃（风险极高）
- **江湖游侠**：任意角色，需武艺技能，路径为游侠→侠客→掌门，可触发特殊江湖事件
- **方外之人**：出家为僧/道，解除大部分身份礼法限制，但社交受限
- **姨娘/侧室**：丫鬟专属，通过感情与主子的关系晋升，改变身份等级但仍在府内

### 5. 声望体系详细设计

#### 5.1 声望获取（通用）
| 行为类型 | 声望增减 | 说明 |
|----------|----------|------|
| 完成本路径任务 | +5~20 | 路径专属任务 |
| 诗文/作品受好评 | +15~30 | 才艺展示 |
| 施舍/资助他人 | +5~10 | 善行 |
| 逾矩失败 | -20~50 | 丑闻 |
| 刑狱/官司 | -100 | 重创 |
| 长期无作为 | -1/天 | 逐渐淡出视野 |

#### 5.2 声望等级与影响
| 等级 | 称号 | 社交影响 | 场景权限 | 任务解锁 |
|------|------|----------|----------|----------|
| 0-199 | 默默无闻 | 无 | 基础 | 新手任务 |
| 200-399 | 小有名气 | 初始好感+5 | 可进入低级社交圈 | 中级任务 |
| 400-599 | 远近皆知 | 陌生人物主动社交概率+30% | 可进入大部分场景 | 进阶任务 |
| 600-799 | 声名显赫 | 对方服从+10 | 可进入上流场景 | 高级任务 |
| 800-1000 | 名满天下 | 对方初始关系大幅提升，可能被嫉妒 | 所有场景（除极秘） | 传奇任务 |

### 6. 与现有系统集成

#### 6.1 任务系统
- 根据角色当前路径与阶段，从任务模板池中筛选专属日常/指令任务。
- 完成路径任务可获得额外声望奖励，加速晋升。
- 路径转换时，旧路径任务池关闭，新任务池开启。
- 职业阶段的 `dailyTasks` 与仆从契约的 `dutyRoutines/followRotations` 会投影到起居职业行，再由任务系统生成对应 `daily` 任务实例。
- 临时差遣和剧情突发任务不进入起居，只在任务系统中作为插队/打断处理。

#### 6.1.1 起居系统
- 起居面板展示职业身份、职业日课和随侍轮值的时间投影。
- 职业日课块不是任务实例本身，而是“这个时段应生成或推进某类日课”的可视化窗口。
- 随侍、值守、门房、巡查等没有固定家具的职责，作为目标跟随型或场景值守型起居块处理。
- 起居通过 `routineFactor` 影响 Utility AI，任务通过 `questFactor` 影响 Utility AI，两者共享同一候选池。

#### 6.2 关系系统
- 声望影响初始好感、信任基础值。
- 某些路径要求与特定 NPC 关系达标（如商人需与官员交好）。
- 路径成功可解锁新的关系类型（如合作伙伴、师徒）。

#### 6.3 对话与互动
- 对话中根据头衔改变称呼（如“张秀才”“李掌柜”）。
- 高声望角色解锁“名人轶事”话题。
- 不同路径解锁专属互动（商人可“谈生意”，工匠可“教授手艺”）。

#### 6.4 场景与移动
- 路径晋升后解锁新场景（店铺、衙门、学府等），这些场景对旧身份不可见。
- 场景权限随 `SocialRank` 动态变化，而 `SocialRank` 随路径阶段调整。

#### 6.5 观察-反应
- 高声望角色的逾矩行为更易被关注，流言传播更快。
- 某些路径（如官员）的不当行为会引发弹劾事件。

### 7. 多态示例：丫鬟的人生选择
- **留守府内**：普通仆从 → 体面仆从 → 执事丫鬟 → 管家 → 老嬷嬷（院内体系）
- **出府经商**：赎身（需银两或主子恩典）→ 货郎 → 开店 → 豪商
- **选秀入宫**：通过采选事件 → 宫女 → 女官 → 才人 → 嫔妃（高风险，但可影响整个家族）
- **成为侧室**：与主子关系极高 + 贾母默认 → 姨娘 → 侧室夫人（身份提升但被正妻压制）
- **匠艺传承**：若有一技之长（刺绣、烹饪），可走工匠路径，成为御用绣娘等

### 8. 扩展性与开发计划

#### 8.1 配置化架构
- 所有路径、阶段、条件、奖励均存储于 JSON 配置，策划可随时添加新路径（如“海外贸易”“军功”）。
- 代码只维护一套通用的晋升判断逻辑：检查条件 → 执行晋升 → 更新角色数据。

#### 8.2 分阶段开发
1. **框架搭建**：实现路径/阶段配置读取、角色运行时数据、晋升判定引擎。
2. **初始路径**：先落地“丫鬟晋升”和“出府开店”两条完整路径，验证框架。
3. **声望系统**：整合全局声望与UI。
4. **多路径扩展**：逐步添加科举、选秀、江湖等路径。
5. **场景与任务**：为每条路径设计专属场景和任务。

#### 8.3 支持 Modding
- 将路径配置外置为可读文件，玩家社区可自定义新职业，丰富游戏生态。

---

这个框架为你提供了一个可演进的古代人生模拟基础，不绑定具体内容，但预留了士农工商、选秀、江湖等所有可能性。你可以从大观园开始，逐步让角色走向更广阔的世界。
