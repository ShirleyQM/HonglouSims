# 金钱系统 PRD（通用框架）

## 1. 概述

金钱系统是支撑角色经济行为的基础模块，驱动大观园内外的一切交易、赏赐、投资与贪贿行为。本系统设计为**通用框架**，支持个人私库与家族公库双线并行，同时预留放贷、经营、投资等扩展接口，后续可接入商贾路径、科举贿赂、选秀打点等场景。

**核心原则：**
- **双线并行**：个人账户（私房钱）与家族账户（府库公银）独立运作，各有收支规则。
- **配置驱动**：所有价格、薪水、利率、罚金均由数据表定义，易于调整经济平衡。
- **来源多样**：月钱、赏赐、任务报酬、放贷利息、贪贿所得、经营收入等。
- **消耗明确**：购物、打赏、投资、贿赂、罚金、日常消费等。
- **风险机制**：借贷有坏账风险，贪贿有被查风险，过度消费有亏空风险。

## 2. 数据结构

### 2.1 账户类型

| 类型 | 说明 | 持有人 | 收支来源 |
|------|------|--------|----------|
| 个人私库 | 角色的私人积蓄 | 每个角色独立 | 月钱、赏赐、任务报酬、放贷、贪贿 |
| 家族公库 | 贾府公共银两 | 家庭为单位 | 家族收入（田庄、俸禄）、支出（月钱发放、宴席、修缮） |
| 店铺账户 | 商贾路径角色拥有 | 店铺主人 | 经营收入、进货支出 |

### 2.2 个人账户数据结构

```js
PersonalAccount {
  character_id: int,
  balance: float,              // 当前余额
  total_income: float,         // 累计收入
  total_expense: float,        // 累计支出
  monthly_salary: float,       // 月钱（由身份决定）
  loans_out: [Loan],           // 放出的贷款
  debts: [Debt],               // 自己的欠债
  transaction_log: [Transaction],  // 最近流水
}
```

### 2.3 家族账户数据结构

```js
FamilyAccount {
  family_id: int,
  balance: float,
  monthly_income: float,       // 固定月收入（田庄、房租、俸禄）
  monthly_expense: float,      // 固定月支出（月钱、饮食、修缮）
  emergency_fund: float,       // 应急储备（低于此值触发警告）
  investments: [Investment],   // 投资记录
  transaction_log: [Transaction],
}
```

### 2.4 交易记录

```js
Transaction {
  id: string,
  timestamp: float,
  type: enum,           // income/expense/loan/debt/transfer
  amount: float,
  source: string,       // 来源描述（如“月钱发放”“赏赐-贾母”）
  target_account: enum, // personal/family/shop
  related_character_id: int,
  note: string,
}
```

## 5. 核心机制

### 5.1 月钱发放

每月初一自动触发，由系统从家族公库中扣除总额，分别打入各角色个人账户。若公库余额不足，优先发放主子，仆从可能被拖欠（引发不满状态）。

### 5.2 交易系统（购买/贩卖）

玩家进入商店场景（预留）或与货郎 NPC 互动时，打开交易界面。物品价格由 `ItemTemplate.price` 决定，支持议价（需口才技能判定）。购买后物品进入角色背包（预留物品系统）。

### 5.3 放贷机制

- **放出贷款**：通过心腹 NPC（如来旺）将银两借给他人，约定利息和期限。
- **收回**：到期后借款人需归还本息。若对方无力偿还，可延期、逼债（降低关系/声望）或坏账。
- **风险**：借款人可能赖账（由其信任值和关系决定），无抵押品。

### 5.4 贪贿与风险

- **收受贿赂**：在特定事件中接受他人金钱，同时增加“把柄”计数。
- **风险累积**：把柄≥3时，御史弹劾概率大幅增加，触发抄家事件链。
- **打点**：可通过贿赂官员降低风险，形成“贪→打点→再贪”循环。

### 5.5 家族危机机制

- 家族公库余额低于警戒线时，触发“财务危机”状态。
- 危机状态下，全府仆从服从下降，部分任务无法执行。
- 需通过削减开支、增加收入（如变卖家产）来缓解。

## 6. 与现有系统集成

| 系统 | 集成方式 |
|------|----------|
| 任务系统 | 任务报酬自动打入个人账户；任务可能要求花费金钱 |
| 关系系统 | 金钱往来影响信任（借债、送礼）、服从（克扣月钱） |
| 身份系统 | 身份决定月钱基准和消费水平；晋升后月钱提升 |
| 声望系统 | 挥霍/吝啬影响声望；救济穷人增加声望 |
| 物品系统 | 购买/贩卖物品需要金钱；物品有价格属性 |
| 场景系统 | 商店、当铺、银号等经济场景 |

## 7. UI 设计

### 7.1 账本界面

角色面板新增“账本”标签，显示：
- 当前余额
- 月钱数额
- 近期收支流水（最近10条）
- 贷款/债务列表

### 7.2 交易界面

购买物品时弹出交易界面：
- 物品图标、名称、价格
- 持有金额
- 购买/取消按钮
- 议价选项（需技能判定）

## 9. 扩展接口预留

| 扩展方向 | 预留接口 |
|----------|----------|
| 商贾路径经营 | `BusinessAccount`，`Inventory` 批量交易 |
| 科举贿赂 | 考试事件中调用金钱打点，影响考官关系 |
| 选秀打点 | 选秀事件中调用金钱送礼，影响入选概率 |
| 田庄管理 | 家族账户中增加田产收益模块 |
| 多家族经济 | 家族账户独立，支持不同家族间贸易 |

此系统为通用经济底座，王熙凤的贪贿经营、袭人的月钱积攒、贾府的财政危机皆可基于此运转。
