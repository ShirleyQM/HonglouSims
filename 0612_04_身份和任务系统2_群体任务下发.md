# 任务下发系统：群体与单人完整设计

## 1. 扩展现有下发模型

在原有的单人任务下发基础上，增加**群体任务下发**能力。  
一个任务模板可同时支持单人下发和群体下发，由下发者选择或由模板定义。

### 1.1 任务模板扩展字段
在 `QuestTemplate` 中新增：
```js
{
  target_scope: 'single' | 'group',   // 单人/群体
  group_filter: {                      // 仅当 scope='group' 时有效
    rank_range: [min, max],            // 目标身份等级范围
    gender: 'male' | 'female' | 'any',
    family_id: 0,                      // 特定家庭，0=不限
    scene_id: 0,                       // 当前所在场景，0=不限
    exclude_roles: ['长辈', '至尊'],   // 排除特定身份（如不能要求贾母）
    max_targets: 0,                    // 最大下发人数，0=无限制
  },
  single_target_relations: ['parent_to_child', 'master_to_servant'],  // 单人的身份关系要求
}
```
原有的 `issuer_relation_required` 和 `target_rank_range` 被整合到此结构。

### 1.2 群体任务示例
| 模板ID | 名称 | 类别 | 目标范围 | 下发权限 |
|--------|------|------|----------|----------|
| Q011 | 晨昏定省（全员） | 侍奉 | 所有小姐、晚辈（等级2-3，女） | 至尊、长辈 |
| Q012 | 洒扫庭院（全院） | 洒扫 | 本场景所有仆从（等级4-5） | 主子、体面仆从 |
| Q013 | 抄写家规 | 功课 | 所有晚辈主子（等级3） | 长辈 |
| Q014 | 备宴 | 特殊 | 所有厨房相关仆从 | 至尊、长辈 |

## 2. 下发流程

### 2.1 单人下发（保持不变）
玩家点击某人 → 系统显示可下发任务列表 → 生成一个 `QuestInstance`。

### 2.2 群体下发
**触发入口：**  
- 玩家操控具有群体下发权限的角色时，界面底部或菜单出现“传令”按钮。  
- 点击后弹出群体任务面板，列出所有可用的群体任务模板。

**下发步骤：**
1. 系统筛选 `QuestTemplate` 中 `target_scope='group'` 且当前角色在 `allowed_issuer_relations` 中有权下发的模板。
2. 对每个模板，检查 `group_filter` 条件是否满足（至少有一个合法目标）。
3. 玩家选择一个模板，可预览受影响的角色列表（例如“将向黛玉、宝钗、探春3人下发晨昏定省”）。
4. 确认后，系统遍历当前游戏世界中所有满足 `group_filter` 的角色，逐一检查：
   - 下发者对该目标是否拥有至少一种 `single_target_relations` 中列出的身份关系？
   - 目标当前是否可以被下发任务（无锁、非禁足等）？
5. 为每个通过检查的目标生成独立的 `QuestInstance`。
6. 所有实例共享同一个 `batch_id`，便于后续统一管理（如撤销、查看进度）。

### 2.3 AI 自主群体下发
当具有权限的 AI 角色检测到需要大规模行动时（如贾母觉得晚辈们疏于礼数），可自主使用群体模板下发任务。权重计算类似，但受全局冷却影响更大（一天内不会反复下发同一群体任务）。

## 3. 身份与群体下发权限

沿用 `QuestIssuePermissionConfig`，但增加 `allow_group_quests: true/false` 字段：

```js
{
  issuer_rank: 0,   // 贾母
  allow_group_quests: true,  // 可群体下发
  allowed_categories: ['侍奉','功课','禁足','家宴',...],
}
{
  issuer_rank: 1,   // 长辈
  allow_group_quests: true,  // 也可群体下发
  allowed_categories: ['功课','禁足','侍奉'],
}
{
  issuer_rank: 2,   // 平辈主子
  allow_group_quests: false, // 一般不可群体下发（只能差遣自家仆从）
}
```

体面仆从若有权限（如袭人），也可向全院普通仆从下发洒扫类群体任务。

## 4. 群体任务 UI 与交互

### 4.1 “传令”面板
点击“传令”后弹出：
```
┌─ 传令全府 ────────────────────────┐
│ 选择任务：                        │
│ [ ] 晨昏定省 (侍奉)               │
│     目标: 黛玉、宝钗、探春… (3人) │
│ [ ] 抄写家规 (功课)               │
│     目标: 探春、惜春 (2人)        │
│ [ ] 洒扫庭院 (洒扫)               │
│     目标: 晴雯、麝月、雪雁… (5人) │
│                                   │
│         [取消] [传令]             │
└───────────────────────────────────┘
```

### 4.2 批量管理
下发者可在任务面板中看到“我下发的群体任务”聚合卡片，显示完成进度（如“3/5人完成”），可对未开始的任务一键撤销。

## 5. 数据库与性能

- 群体下发时，一次性生成多个 `QuestInstance`，但总人数受大观园角色数限制（<50人），批量操作性能无虞。
- 每个任务实例独立追踪进度，互不干扰。

---

## 6. 与现有系统的融合

- **身份系统**：群体筛选基于 `SocialRank`、性别、家庭等身份字段。
- **关系系统**：群体任务不改变下发者与目标的关系，但完成或失败会走正常的奖惩（关系值、服从等）。
- **日常任务**：群体下发任务可打断仆从的日常任务，但主子、小姐的日常不会自动生成，因此群体任务成为她们为数不多的强制性指令。
- **记忆系统**：若群体任务涉及重要事件（如家宴），结束后可能生成集体记忆。

---

至此，任务下发系统完善为**单人精确差遣**与**群体批量传令**的双模式，完全贴合古代大家族管理方式，扩展性强且易于配置。
