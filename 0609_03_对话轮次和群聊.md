# 对话与群聊系统 PRD（完整版）

## 1. 概述

本系统为双人互动与多人场景提供**可变长度、多模式生成、群聊扩散**的对话内容层。它嵌入社交互动，生成符合角色性格、状态和关系的文本，并实时反馈情绪与关系变化。核心特征：
- **双人对话**：2~10轮，由互动类别、关系、场景等因素动态决定。
- **群聊机制**：同场景≥3人自动形成对话圈，话题在成员间传播，支持观察者插话、多人情绪传染。
- **生成模式可选**：默认模板驱动，预留本地AI小模型接口，可无缝切换或混合。
- **状态闭环**：对话 ↔ 情绪状态实时双向影响。

---

## 2. 双人对话机制（核心）

### 2.1 对话长度动态决定
总轮次数范围 `[2, 10]`，由互动开始时根据以下因素计算：

| 影响因素 | 对轮次的影响 | 说明 |
|----------|--------------|------|
| 互动类别 | 基础轮次 | 寒暄2~4，叙旧3~6，论道4~8，慰藉3~6，传情4~8，争执2~6 |
| 双方综合分 | +0~2 | 每30分+1轮 |
| 场景私密性 | +0~1 | 私密场景+1轮 |
| 双方情绪匹配度 | +0~1 | 同为积极情绪+1，一方负面-1 |
| 近期美好记忆 | +0~1 | 3天内有正面记忆+1 |
| 身份礼法约束 | -0~2 | 仆对主正式场合-1，逾矩情境-1 |

最终轮次 = CLAMP(基础+修正, 2, 10)

### 2.2 对话生成模式
| 模式 | 说明 | 使用场景 |
|------|------|----------|
| 模板模式 | 基于配置表随机生成 | 默认 |
| 混合模式 | 日常话题模板，高情感话题调用本地模型 | 平衡性能与丰富性 |
| 模型模式 | 全由本地模型生成，失败回退模板 | AI驱动 |

### 2.3 对话链结构
- 开场白（Speaker发起，1轮）
- 主体对话（交替发言，1~8轮）
- 结束语（任一方触发，1轮）

### 2.4 对话效果累积
每轮携带 `micro_effect`（如愉悦+3、好感+1），通过状态系统累积进度，触发状态切换（如“愉悦”→“心动”）。

## 3. 群聊机制

### 3.1 群聊触发条件
同场景内 ≥3 个角色，且任意两人正在进行社交互动时：
1. 互动双方进入双人对话流程。
2. 系统检查场景内其他角色的**观察意愿**，决定是否“加入群聊”。
3. 若至少1人加入，则对话圈扩展为**群聊模式**。

### 3.2 观察意愿判定
对每个旁观者，计算加入概率：
```
P = base + relation_bonus + trait_bonus + status_bonus - distance_penalty
```
- base: 0.2
- relation_bonus: 与互动中任一方的综合分每20分 +0.1
- trait_bonus: 好客/乐天/风流 +0.3，隐士/清高 -0.2
- status_bonus: 愉悦/微醺 +0.2，忧伤/愤怒 -0.1
- distance_penalty: 距离超过5格每格 -0.05

若 P > 0.5，则该旁观者**主动靠近**并加入群聊。

### 3.3 群聊结构
群聊仍以**话题**为单元，由当前持有“发言权”的角色发言，其他人可回应。发言权传递规则：
- 初始发言权归互动发起者。
- 每次发言后，系统从**有意回应者**中按权重选一人接话。若无人回应，则发起者可继续发言或结束话题。
- 群聊总轮次上限为 12 轮（超过自动衰减）。

### 3.4 话题扩散与分裂
- 群聊中可能出现**子话题**：当回应者的发言与主话题偏离较远时，自动生成新话题标签。
- 若某两人连续3轮互相回应，则他们可能“私语化”，降低音量，旁人发言权重降低。但这不脱离群聊，只是形成临时焦点。
- 若群聊人数≥5，可能自动分裂为两个并行对话圈（逻辑分簇），但UI仍表现为同一场景内的多人气泡交替。

### 3.5 群聊效果
- 所有参与者的社交需求得到少量恢复。
- 话题中的情绪效果会传播给**所有聆听者**（情绪传染系统加成）。
- 群聊结束后，参与者之间可能产生集体记忆（如“那日在潇湘馆众人说笑”）。
- 身份礼法在群聊中仍然生效：仆从不会在主子面前插科打诨，逾矩行为若被多人目睹则风险加倍。

### 3.6 群聊UI表现
- 同一场景内的人物头顶轮流冒泡，显示发言内容。
- 发言者高亮，听者表情/动作微反应（微笑、皱眉、点头）。
- 左侧消息日志记录群聊关键对话，格式：“【群聊】宝玉：…… | 黛玉：…… | 宝钗掩嘴轻笑。”

## 4. 上下文构建（扩展为多人）

### 4.1 双人上下文 (DialogueContext)
```js
{
  speaker, listener,
  relation, interaction,
  scene: { privacy, atmosphere },
  speaker_statuses, listener_statuses,
  recent_memories,
  topic_history
}
```

### 4.2 群聊上下文 (GroupChatContext)
继承自双人上下文，扩展：
```js
{
  primary_speaker,            // 当前主发言人
  participants: [             // 所有参与者
    { char, relation_to_primary, statuses }
  ],
  active_topic: Topic,
  sub_topics: [Topic],
  discussion_round: 0,
  max_rounds: 12,
  private_mode: false         // 是否已私语化
}
```

## 5. 对话决策（轻量AI + 本地模型）

### 5.1 双人策略表（不变）
根据性格和状态计算整体姿态（主动热情、谨慎恭顺等），影响话题选择和回应。

### 5.2 群聊话题管理
- 群聊开始时，从主互动的话题库中抽取一个“群聊适宜”标签的话题（如闲聊、趣事、议论）。
- 当有人回应时，根据回应内容可能延续或切换话题。切换判断由当前话题的 `continue_probability` 和回应者的话题偏移决定。
- 若调用模型，上下文需包含前3轮对话历史和所有参与者简述。

### 5.3 本地模型接口扩展
群聊生成时，向模型提供的上下文包含：
- 主发言人信息
- 所有参与者列表（名字+身份）
- 关系矩阵（关键人物间）
- 最近3轮发言记录
- 当前话题和氛围

## 6. 配置表设计（整合）

### 6.1 话题库（双人+群聊标签）
- `topic_id`, `category`, `tone_tags`
- `group_suitable`: bool  // 是否适合群聊
- `min_score_suggested`
- `speaker_conditions`
- `templates`: [string]
- `model_prompt_hint`

### 6.2 回应库
- `response_id`
- `trigger_topic_category`
- `listener_strategy` (或状态条件)
- `tone` (positive/neutral/negative/deflect)
- `text_templates`
- `micro_effect`
- `continue_topic`

### 6.3 群聊配置
```json
{
  "group_chat_base_join_prob": 0.2,
  "max_group_rounds": 12,
  "private_mode_rounds": 3,
  "split_threshold": 5
}
```

### 6.4 互动类别基础轮次与群聊修正
在原有基础上增加 `group_round_bonus`: 群聊时额外增加2~4轮。

## 7. 与现有系统的集成

| 系统 | 集成点 |
|------|--------|
| 双人互动 | 互动开始时构建上下文，调用对话引擎 |
| 关系系统 | 每轮对话微调四轴 |
| 状态系统 | 累积情绪进度，触发状态切换 |
| 叙事气泡 | 所有对话文本通过气泡展示 |
| 观察-反应 | 旁观者加入群聊即视为“反应”，群聊中逾矩触发观察者检举 |
| 记忆系统 | 双人重要对话、群聊欢乐/冲突生成记忆 |
| 场景系统 | 提供场景私密性、距离、氛围数据 |
| 身份礼法 | 群聊中逾矩风险加倍，仆人发言权重受压制 |
| 本地模型接口 | 可选生成对话内容 |

## 8. 流程示例：群聊触发

1. 宝玉和黛玉在沁芳亭“叙旧·闲谈”，双人对话已进行2轮。
2. 宝钗路过，距离3格，判定加入概率：关系高（+0.4），性格好客（+0.3），总计0.9 → 加入。
3. 群聊形成，对话轮次上限提升为8轮（原基础4 + 群聊额外4）。
4. 话题转为“园中春色”，三人轮流发言，每轮产生社交需求和情绪微调。
5. 第6轮，宝钗提到“宝兄弟近日功课如何”，话题转向读书，引发宝玉不悦（微效果）。
6. 群聊结束时，所有参与者获得“热络”状态，并记录集体记忆“春日亭中闲话”。

## 9. 本地模型接入策略（不变）
- 超时2秒回退模板。
- 连续失败3次本互动禁用模型。
- 每个角色每分钟最多6次调用。

---

## 10. 总结

本PRD将双人对话与群聊机制统一在同一个轻量框架下，通过配置表驱动、可变长度、情绪反馈，实现了沉浸式社交模拟。群聊机制让场景中的社交动态更自然，玩家的旁观体验更丰富。系统保留了对本地AI小模型的扩展能力，可在未来提供更动态的对话内容。
