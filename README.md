# HonglouSims

早期的模拟人生网页版。

人物会有需求、性格、关系、起居和日常任务，玩家可以观察他们怎么过一天，也可以通过互动和传令去影响他们。

目前首期背景放在《红楼梦》风格的大观园/贾府里。不是正经复刻原著，也不是战斗、抽卡或者经营放置；更像是在试一套“古代生活 + 人物关系 + 府内秩序”的模拟系统。

## 现在能玩什么
- 人物会根据饥、倦、趣、洁、交游、心绪等需求自行行动。
- 性格会影响人物偏好，比如黛玉更容易被文墨、独处和心绪牵动。
- 家具可以被使用，用来恢复需求、触发状态或推进技能。
- 点人物可以进行社交互动，关系会变化。
- 起居面板可以查看/调整人物一天的安排。
- 传令面板可以给别人下发任务，任务接受和执行会受到身份、职责、关系影响。
- 后台 v2 用来配置人物、关系、任务、行为规则和世界数据。

## 怎么玩
1. 先切到一个人物，比如林黛玉，看看她的当前状态和所在位置。
2. 打开“人物”，看她的需求、性格、状态和关系。很多行动不是纯随机的，会被这些东西影响。
3. 打开“起居”，看这个人今天本来该做什么。起居不是任务清单，更像一天里的生活节奏。
4. 点场景里的家具，比如书案、饭桌、茶案、琴台，看看人物能不能使用，以及用了以后需求或状态有没有变化。
5. 点另一个人物，试试社交互动。互动可能成功，也可能因为关系、身份、礼法不合适而失败。
6. 打开“传令”，给能被差遣的人下一件事。任务接不接受、做得好不好，会看身份、职责、信任、服从和当前负担。
7. 如果人物突然做了奇怪的事，可以先看叙事气泡、状态和行动队列。有时是需求太低，有时是起居或任务把她拉走了。

#### 体验优化
- 当前选中的人物就是玩家操控对象，不会自己跑完整的自主 AI，防止随意打断用户进程。
- 没有被选中的人物才会自己行动，会根据需求、起居、关系、任务和性格去找事做。
- 当前选中的人物仍然可能被别人拉进事件，比如被 NPC 主动找来说话、互动，或者成为任务目标。
- 互动开始后双方会被锁进对话/互动状态。非选中人物会暂停自己的 AI，选中人物本来就没有自主 AI，所以只是被卷入这次事件。

比较推荐的试玩目标：观察黛玉和紫鹃从早上到中午的行动。黛玉应该能体现出心绪、才艺和喜静倾向；紫鹃则更像贴身侍女，会围绕主子、随侍和下发任务行动。


## 项目结构
```text
assets/      角色、场景、家具、UI 等资源
js/          游戏主体逻辑和系统代码
prd/         持续更新的系统设计文档
tests/       配置检查、冒烟测试和场景测试
scripts/     资源处理、模拟和辅助脚本
reports/     自动化或手动验证留下的报告
outputs/     对外展示或临时导出的材料
```

主要系统文档可以先看 `prd/README.md`，里面有每个系统对应的代码入口。

## 本地跑

需要 Node.js 18+。

```bash
npm install
npm run start:local
```

```text
http://127.0.0.1:8765/
```


## 入口

| 文件 | 说明 |
| --- | --- |
| `index.html` | 游戏主入口 |
| `jiafu-order.html` | 贾府秩序 / 族谱 / 身份权力相关原型 |
| `memory-panel.html` | 记忆面板原型 |
| `relation-test.html` | 关系系统测试页 |
| `prd/` | 需求和系统设计文档 |
| `reports/` | 模拟日志、测试报告、临时验证结果 |
| `outputs/` | 导出的 PPT、截图、表格等产物 |

## 测试

```bash
npm run test:release
```

它会依次检查配置、前台冒烟、后台冒烟、传令面板和场景测试注册表。

单项命令：

```bash
npm run test:config
npm run test:smoke
npm run test:admin
npm run test:command
npm run test:scenario
```

部分冒烟测试需要先启动本地服务。


## 限制

- 美术和动画基本没做。
- 人物 AI 还需要继续调权重和补反馈。
- 群体传令、秩序管理、经济等系统还在迭代。
- 精细的对话只有本地 LLM。
- 存档主要依赖浏览器本地数据，没做后端数据库，清缓存会丢。
