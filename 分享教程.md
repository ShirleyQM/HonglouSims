# HonglouSims · 分享给朋友玩 — 每次使用教程

> 你当「房主」开服务，朋友用浏览器打开链接即可。**不用安装游戏**，存档在各自浏览器里。

---

## 一、你要做什么（房主）

### 第一次准备（只做一次）

1. 确认已安装 **Node.js 18+**（终端执行 `node -v` 有版本号即可）
2. 进入游戏目录：
   ```bash
   cd /Users/bytedance/Desktop/myCursor/game
   ```
3. **外网分享**还需安装 cloudflared（只做一次）：
   ```bash
   brew install cloudflared
   ```
4. **可选 · AI 对话**：想和朋友一起用 AI 互动，先装好 Ollama 并拉模型：
   ```bash
   ollama serve          # 另开终端常驻
   ollama pull qwen2.5:7b-instruct
   ```

---

### 每次开玩（必做）

#### 第 1 步：启动游戏服务

打开**终端 1**，执行：

```bash
cd /Users/bytedance/Desktop/myCursor/game
npm start
```

看到类似输出即成功：

```
HonglouSims  已启动
本机访问:   http://127.0.0.1:8765
局域网分享: http://192.168.x.x:8765
```

**不要关这个终端窗口。**

---

#### 第 2 步：选一种方式发给朋友

| 场景 | 你要做的 | 发给朋友的链接 |
|------|----------|----------------|
| **同一 WiFi / 同一局域网** | 第 1 步就够了 | 终端里 `局域网分享` 那一行，如 `http://192.168.0.107:8765` |
| **不在同一网络（外网）** | 再开**终端 2**，执行下面命令 | 终端里出现的 `https://xxx.trycloudflare.com` |

外网隧道命令（**终端 2**）：

```bash
cd /Users/bytedance/Desktop/myCursor/game
npm run tunnel
```

等几秒，找到这一行：

```
https://xxxxxxxx.trycloudflare.com
```

复制整段 **https** 链接，微信/飞书发给朋友。

**两个终端都要保持开着。**

---

#### 第 3 步：自己先试一下

- 本机浏览器打开 `http://127.0.0.1:8765` 确认能玩
- 外网分享时，自己也用那个 `https://...trycloudflare.com` 打开一次，确认不是 530 错误再发给朋友

---

### 玩完后怎么关

1. 两个终端里分别按 `Ctrl + C` 停掉服务
2. 或直接关终端窗口

下次再玩，从「每次开玩 → 第 1 步」重新来一遍。

> **注意**：外网链接每次重启 tunnel 都会变，要重新发给朋友。

---

## 二、朋友要做什么

### 必做（就 1 件事）

1. 收到你发的链接（`http://192.168...` 或 `https://...trycloudflare.com`）
2. 用 **Chrome / Safari / Edge** 等现代浏览器打开
3. 开始玩

### 朋友不需要

- 不用安装 Node、不用下载游戏文件
- 不用登录账号
- 不用和你连同一个 WiFi（若你发的是 https 外网链接）

### 朋友侧建议

| 建议 | 说明 |
|------|------|
| 用电脑或平板 | 屏幕更大，地图和 UI 更舒服 |
| 横屏 / 宽窗口 | 游戏最小宽度约 1280px |
| 别用无痕模式长期玩 | 存档存在浏览器本地，清缓存会丢档 |
| 卡住就刷新 | `Cmd+R` / `F5` 重新加载 |

## 三、快速对照表

```
房主（你）                         朋友
─────────────────────────────────────────────────
终端1: npm start                   收到链接
       ↓                           ↓
终端2: npm run tunnel（仅外网）     浏览器打开链接
       ↓                           ↓
复制 https 链接发过去               直接玩
       ↓
电脑保持开机、终端别关
```

## 四、常见问题

### 朋友打不开 / 显示 Cloudflare 错误

- 确认你的**两个终端都还在跑**
- 外网链接刚生成时等 10～30 秒再让朋友试
- 你本地用同一 https 链接能打开，朋友才能打开

### 朋友能进游戏，但 AI 对话报错

- 你这边需要另开终端运行 `ollama serve`
- 确认已拉取模型：`ollama pull qwen2.5:7b-instruct`
- 没有 AI 也能正常玩，只是互动对白会用预设文案

### 同一 WiFi 的朋友连不上

- 发 **局域网** 地址（`192.168.x.x:8765`），不要发 `127.0.0.1`
- 检查 Mac 防火墙是否拦了 8765 端口
- 确认手机和电脑连的是同一个 WiFi

### 链接以前能用，现在不行了

- 你重启过电脑或关了终端 → 需要重新 `npm start` + `npm run tunnel`
- 外网链接会变，**要发新链接**

### 只想自己玩、不分享

```bash
npm run start:local
```

只监听本机 `127.0.0.1`，别人访问不到。

## 五、命令速查

```bash
# 启动游戏（局域网 + 本机）
npm start

# 外网隧道（另开终端，且 npm start 已在跑）
npm run tunnel

# 仅本机
npm run start:local

# 可选：AI
ollama serve
```

## 六、Docker 方式（可选）

若不想每次手动 `npm start`：

```bash
docker compose up
```

外网分享仍需另开终端执行 `npm run tunnel`（或 `cloudflared tunnel --url http://127.0.0.1:8765`）。
