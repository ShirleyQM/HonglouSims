# 大观园模拟 · GitHub Pages 上线指南

## 1. 推荐发布方式

第一版推荐使用 GitHub Pages。

原因：

- 项目当前是静态前端游戏。
- 不需要后端服务器。
- 免费。
- 适合给朋友和早期玩家试玩。
- URL 可以直接分享。

## 2. GitHub Pages 是什么

GitHub Pages 是 GitHub 自带的静态网页托管服务。

开启后，仓库里的 `index.html`、`js/`、`assets/` 等文件会被发布成一个公网网页。

网址一般是：

```text
https://你的用户名.github.io/仓库名/
```

## 4. GitHub Pages 路径兼容性

当前项目适合部署到 GitHub Pages 子路径。

已经确认：

- `index.html` 使用相对路径加载图标和脚本。
- `js/`、`assets/`、`prd/`、`tests/` 都在仓库内。
- favicon 使用 `assets/icons/favicon.svg` 相对路径。
- 页面不依赖后端 API。

注意：

- 不要把资源路径改成以 `/` 开头的根路径，例如 `/assets/xxx.png`。
- GitHub Pages 项目页通常部署在 `/仓库名/` 子路径下，绝对根路径会找错位置。
- 如果后续加入 API、数据库、本地 LLM，需要另行部署后端。

## 5. 开启 GitHub Pages

步骤：

1. 创建 GitHub 仓库。
2. 推送当前项目代码。
3. 打开仓库页面。
4. 进入 `Settings`。
5. 进入 `Pages`。
6. Source 选择 `Deploy from a branch`。
7. Branch 选择 `main`。
8. Folder 选择 `/root`。
9. 保存。
10. 等待 GitHub 构建完成。

完成后会得到网址：

```text
https://你的用户名.github.io/仓库名/
```

## 9. 发布后维护建议

- 每次发版前跑 `npm run test:release`。
- 每次修复后更新 `CHANGELOG.md`。
