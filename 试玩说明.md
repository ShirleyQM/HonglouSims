# 大观园模拟 · 试玩说明

> 当前版本：v0.1.0-alpha  
> 版本主题：起居、传令、人物 AI、家具互动、社交互动第一版试玩。

## 1. 这是什么

《大观园模拟》是一个以大观园人物日常为核心的模拟游戏原型。

你可以观察人物在园中的行动，也可以通过起居、传令、家具和社交系统影响他们的一天。

这不是最终正式版，而是给朋友和早期玩家体验用的试玩版。

## 2. 推荐试玩目标

建议试玩时先做这几件事：

1. 观察人物半天到一天，看他们会不会按需求、起居、任务自行行动。
2. 打开“起居”，看看人物当天有哪些需求、技能、职业安排。
3. 点击“传令”，给可传令对象发布一个点名任务。
4. 点击家具，试试人物能不能使用家具恢复需求。
5. 点击其他人物，试试社交互动和关系变化。

## 3. 主要入口

| 入口 | 说明 |
| --- | --- |
| 人物 | 查看当前人物信息、性格、身份和状态。 |
| 起居 | 查看和编辑人物当天作息。 |
| 传令 | 点名或群体下发任务。 |
| 关系 | 查看人物之间的关系轴，包括服从、信任等。 |
| 技能 | 查看人物技能。 |
| 背包 | 查看持有物。 |
| 存档 | 保存、读取或清除当前浏览器本地进度。 |
| 后台 | 配置入口，普通试玩玩家可以先不用。 |

## 4. 当前重点系统

### 起居

- 一天按 48 刻划分，每刻 30 分钟。
- 起居会影响人物 Utility AI 的行动权重。
- 职业日课和随侍轮值会投影到起居里。
- 临时任务不会写入起居。

### 传令

- 传令统一为任务下发入口。
- 左侧是点名传令，选择明确人物。
- 右侧是群体传令，选择符合条件的一组人。
- 当值人物头像上的“令”会打开传令面板并预选此人。
- 服从、信任、职责内和任务负载会影响任务接受。

### 人物 AI

- 人物会根据需求、起居、任务、距离、关系、状态来选择行为。
- 严重需求危机会优先于普通作息。
- 任务和随侍会影响人物行动。

## 5. 已知限制

- 这是 alpha 版本，人物行为可能还会有不自然之处。
- 群体传令不是每个角色都有权限，某些角色打开后可能只显示点名传令。
- 本地 LLM / Ollama 相关能力在线试玩版默认不可用。
- 存档主要依赖浏览器本地，不同浏览器或清缓存后可能丢失。
- 后台配置入口仍保留给开发和策划使用，试玩玩家不建议随意修改。

## 6. 如何反馈

反馈时最好包含：

- 你做了什么操作。
- 当前人物是谁。
- 游戏内大概时辰。
- 你期望发生什么。
- 实际发生了什么。
- 如果页面报错，截图控制台或页面状态。

推荐反馈类别：

- 页面打不开。
- 人物卡住或走到空白地图。
- 起居没有生效。
- 传令看不懂或任务没执行。
- 家具/社交不可用但原因不清楚。
- 哪个瞬间最有趣。
