# 潇湘馆多日模拟使用手册

> 脚本：`scripts/simulate_xiaoxiang_7days.cjs`
> 默认报告：`reports/潇湘馆_7天模拟日志.md`
> 默认人物：黛玉、紫鹃、雪雁

## 1. 这个脚本用来做什么

用于离线模拟潇湘馆人物多日 AI 行为，方便看：

- 每个人每天到底做了什么。
- 是否反复吃饭、喝茶、弹琴、找家具。
- 是否有任务下发、任务阻塞、路径失败。
- 需求、社交、任务、家具、AI 候选是否长期平衡。

它不需要打开浏览器，也不依赖当前浏览器里的存档。脚本会在 Node 里加载一套新的默认游戏配置来跑。

## 2. 最常用命令

默认跑 7 天：

```bash
node scripts/simulate_xiaoxiang_7days.cjs
```

只跑 1 天，适合快速冒烟：

```bash
SIM_DAYS=1 node scripts/simulate_xiaoxiang_7days.cjs
```

跑 3 天，适合改完 AI 或配置后复验：

```bash
SIM_DAYS=3 node scripts/simulate_xiaoxiang_7days.cjs
```

跑 7 天并更频繁打印进度：

```bash
SIM_DAYS=7 SIM_PROGRESS_EVERY_GAME_MIN=120 node scripts/simulate_xiaoxiang_7days.cjs
```

## 3. 参数说明

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `SIM_DAYS` | `7` | 模拟游戏天数。 |
| `SIM_PROGRESS_EVERY_GAME_MIN` | `720` | 每隔多少游戏分钟打印一次进度。`720` 约等于半天一次。 |

脚本内部目前固定：

| 项 | 当前值 | 说明 |
|---|---|---|
| 目标人物 | `daiyu`、`zijuan`、`xueyan` | 只汇总这三个人的详细行为。 |
| 随机种子 | `61617` | 保持同一版本下结果可复现。 |
| 模拟步长 | `1` 秒 | 每次推进 1 秒游戏逻辑。 |
| LLM | 关闭 | 避免本地模型影响离线复现。 |

如果之后要换测试家庭，需要改脚本顶部的 `TARGET_IDS`。

## 4. 怎么看运行进度

运行时终端会输出类似：

```text
[sim]  42.9% 第04日 08:00 rows=1234 elapsed=140.2s eta=186.9s
```

字段含义：

| 字段 | 含义 |
|---|---|
| 百分比 | 当前模拟完成度。 |
| `第04日 08:00` | 当前游戏时间。 |
| `rows` | 已记录进报告的日志行数。 |
| `elapsed` | 已消耗现实时间。 |
| `eta` | 预计还要等多久。 |

当前机器上，完整 7 天通常约 5 到 6 分钟；1 天大约半分钟上下。具体会随代码和机器负载波动。

## 5. 报告怎么看

报告固定写到：

```text
reports/潇湘馆_7天模拟日志.md
```

每次运行会覆盖旧报告。如果要对比多轮结果，先把旧文件另存一份再跑。

报告主要看这些章节：

| 章节 | 用途 |
|---|---|
| 自动问题摘要 | 快速看路径、任务、队列、社交等是否有明显异常。 |
| 高频重复行为 | 看 NPC 是否一直重复同一类选择，比如反复前往某地或重复使用同一家具。 |
| 每人每日汇总 | 按人物和天数看行为量、任务量、社交量、家具使用量。 |
| 最终状态 | 看模拟结束时人物在哪、需求如何、队列里是否残留动作。 |
| 按时间顺序详细日志 | 按游戏时间复盘每个动作；如果是任务，会展示下发者。 |
| 原始事件计数 | 看埋点量级是否正常，避免因为事件缺失误判。 |

常用搜索关键词：

```text
行动受阻
任务失败
任务阻塞
AI选择
开始用家具
完成用家具
主动社交
```

## 6. 推荐测试流程

1. 先跑 1 天，看有没有直接报错或严重卡住。

```bash
SIM_DAYS=1 node scripts/simulate_xiaoxiang_7days.cjs
```

2. 改 AI、任务、家具、需求配置后，跑 3 天看趋势。

```bash
SIM_DAYS=3 node scripts/simulate_xiaoxiang_7days.cjs
```

3. 比较稳定后再跑完整 7 天。

```bash
node scripts/simulate_xiaoxiang_7days.cjs
```

4. 优先看“自动问题摘要”和“高频重复行为”，再去详细日志里定位具体时间点。

## 7. 常见问题

### 服务要不要启动？

不用。这个脚本直接在 Node 里跑，不需要 `npm run start:local`，也不需要打开 `127.0.0.1:8765`。

### 为什么和我浏览器里看到的不一样？

脚本使用新的默认配置和固定随机种子，不读取浏览器当前存档、手动操作历史或 localStorage。

### 为什么报告很大？

这是故意的。它要保留按时间顺序的详细日志，方便追查“某一天某个人为什么突然一直做某事”。

### 跑完没有看到最新内容？

确认看的文件是：

```text
reports/潇湘馆_7天模拟日志.md
```

脚本最后也会在终端打印 `outFile`，以终端里的路径为准。

### 想减少等待时间怎么办？

先用 `SIM_DAYS=1` 或 `SIM_DAYS=3`。如果只是想知道它是不是还在跑，可以降低进度间隔：

```bash
SIM_PROGRESS_EVERY_GAME_MIN=60 SIM_DAYS=1 node scripts/simulate_xiaoxiang_7days.cjs
```
